Question 1: Full Star Diamond

Problem Statement

Print a diamond shape with stars (pyramid + inverted pyramid).

Sample Input and Output

Sample Input 1:
5

Sample Output 1:
*
***
*****
*******
*********
*******
*****
***
*
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class FullDiamond { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter N: "); int n = sn.nextInt(); // Upper pyramid for (int i = 1; i <= n; i++) { for (int j = 1; j <= n - i; j++) { System.out.print(" "); } for (int k = 1; k <= 2 * i - 1; k++) { System.out.print("*"); } System.out.println(); } // Lower inverted pyramid for (int i = n - 1; i >= 1; i--) { for (int j = 1; j <= n - i; j++) { System.out.print(" "); } for (int k = 1; k <= 2 * i - 1; k++) { System.out.print("*"); } System.out.println(); } sn.close(); } }

Step-by-Step Explanation

  1. Diamond = upper + lower halves: First loop creates upward pyramid (widening), second loop creates inverted pyramid (narrowing). Together they form complete diamond shape.
  2. Lower starts at n-1 not n: Starting lower half at n would repeat the widest row. Starting at n-1 creates smooth transition without duplication.
  3. Code duplication is acceptable here: Could extract to method, but for learning, seeing both halves explicitly shows the mirror structure clearly. Repetition aids understanding.
  4. Composite pattern technique: Many complex patterns are combinations of simpler ones. Diamond = 2 pyramids. Hourglass = inverted + normal. Learning to decompose complex shapes is a key skill.

Question 2: Hollow Diamond

Problem Statement

Print a hollow diamond (stars only on edges).

Sample Input and Output

Sample Input 1:
5

Sample Output 1:
*
* *
* *
* *
* *
* *
* *
* *
*
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class HollowDiamond { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter N: "); int n = sn.nextInt(); for (int i = 1; i <= n; i++) { for (int j = 1; j <= n - i; j++) { System.out.print(" "); } for (int k = 1; k <= 2 * i - 1; k++) { if (k == 1 || k == 2 * i - 1) { System.out.print("*"); } else { System.out.print(" "); } } System.out.println(); } for (int i = n - 1; i >= 1; i--) { for (int j = 1; j <= n - i; j++) { System.out.print(" "); } for (int k = 1; k <= 2 * i - 1; k++) { if (k == 1 || k == 2 * i - 1) { System.out.print("*"); } else { System.out.print(" "); } } System.out.println(); } sn.close(); } }

Step-by-Step Explanation

  1. Hollow logic in both halves: Same edge-detection condition (k==1 || k==2*i-1) used in both upper and lower parts. Consistency in hollowing logic across both sections.
  2. No special case for center row: Unlike hollow pyramid which fills base, diamond's widest row stays hollow. It's just another row with edges, not a special base.
  3. Symmetric hollowing: Upper and lower halves are mirrors, including the hollow spaces. This creates uniform outline all around the diamond.
  4. Visual debugging tip: If diamond looks wrong, test each half separately. Debug upper pyramid, then lower. Combining working parts is easier than debugging the whole at once.

Question 3: Number Diamond

Problem Statement

Print a diamond with numbers increasing to center then decreasing.

Sample Input and Output

Sample Input 1:
4

Sample Output 1:
1
121
12321
1234321
12321
121
1
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class NumberDiamond { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter N: "); int n = sn.nextInt(); for (int i = 1; i <= n; i++) { for (int j = 1; j <= n - i; j++) { System.out.print(" "); } for (int k = 1; k <= i; k++) { System.out.print(k); } for (int m = i - 1; m >= 1; m--) { System.out.print(m); } System.out.println(); } for (int i = n - 1; i >= 1; i--) { for (int j = 1; j <= n - i; j++) { System.out.print(" "); } for (int k = 1; k <= i; k++) { System.out.print(k); } for (int m = i - 1; m >= 1; m--) { System.out.print(m); } System.out.println(); } sn.close(); } }

Step-by-Step Explanation

  1. Palindrome in each row: Each row prints ascending numbers then descending (1, 121, 12321). This row-level palindrome creates the number pattern within diamond shape.
  2. Same row logic in both halves: Both upper and lower use identical ascending-descending number logic. The only difference is the outer loop direction (i increasing vs decreasing).
  3. Complexity from composition: Combining palindromic rows with diamond shape creates intricate end result. Yet code is just diamond structure + number pyramid rows. Complex patterns from simple components.
  4. Pattern design principle: Can mix and match pattern techniques - diamond shape + number palindromes, pyramid structure + hollow edges, etc. Understanding components lets you create infinite variations.

Question 4: Butterfly Pattern

Problem Statement

Print a butterfly pattern (diamond rotated horizontally).

Sample Input and Output

Sample Input 1:
4

Sample Output 1:
* *
** **
*** ***
********
*** ***
** **
* *
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class ButterflyPattern { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter N: "); int n = sn.nextInt(); for (int i = 1; i <= n; i++) { for (int j = 1; j <= i; j++) { System.out.print("*"); } for (int j = 1; j <= 2 * (n - i); j++) { System.out.print(" "); } for (int j = 1; j <= i; j++) { System.out.print("*"); } System.out.println(); } for (int i = n - 1; i >= 1; i--) { for (int j = 1; j <= i; j++) { System.out.print("*"); } for (int j = 1; j <= 2 * (n - i); j++) { System.out.print(" "); } for (int j = 1; j <= i; j++) { System.out.print("*"); } System.out.println(); } sn.close(); } }

Step-by-Step Explanation

  1. Left wing + middle gap + right wing: Three sections per row - left triangle, middle spaces, right triangle (mirror of left). This left-gap-right structure creates butterfly wings.
  2. Gap size formula 2*(n-i): As wings grow (i increases), gap s shrinks. Row 1 has i=1 stars per wing and 2*(n-1) gap. Last row has i=n stars per wing and 0 gap (wings meet).
  3. Mirror structure horizontally: Unlike diamond (vertical mirror), butterfly mirrors horizontally. Same pattern principles, different axis. Understanding mirror axes helps design varied patterns.
  4. Nature-inspired patterns: Butterfly, hourglass, bowtie - many patterns model real shapes. Seeing code as art/design, not just syntax, makes programming more creative and engaging.

Question 5: Hourglass Pattern

Problem Statement

Print an hourglass (inverted pyramid on top, pyramid on bottom).

Sample Input and Output

Sample Input 1:
5

Sample Output 1:
*********
*******
*****
***
*
***
*****
*******
*********
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class HourglassPattern { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter N: "); int n = sn.nextInt(); for (int i = n; i >= 1; i--) { for (int j = 1; j <= n - i; j++) { System.out.print(" "); } for (int k = 1; k <= 2 * i - 1; k++) { System.out.print("*"); } System.out.println(); } for (int i = 2; i <= n; i++) { for (int j = 1; j <= n - i; j++) { System.out.print(" "); } for (int k = 1; k <= 2 * i - 1; k++) { System.out.print("*"); } System.out.println(); } sn.close(); } }

Step-by-Step Explanation

  1. Hourglass = inverted pyramid + pyramid: Opposite of diamond! First narrows (inverted), then widens (normal). Reversing the order creates dramatically different visual.
  2. Lower half starts at 2 not 1: Starting at 2 avoids repeating the narrowest row (single star). This creates smooth pinch point without duplication.
  3. Same code, different order: Compare to diamond - literally same pyramid code but inverted first, normal second instead of normal first, inverted second. Order matters hugely in composite patterns.
  4. Pattern combination mastery: By now you've seen simple patterns combined in various orders and orientations to create complex shapes. This is the essence of pattern programming - master basics, combine creatively.
