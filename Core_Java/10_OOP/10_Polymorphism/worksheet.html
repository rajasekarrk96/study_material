Question 1: Constructor Overloading (Compile-Time Polymorphism)

Problem Statement

Create a Book class with two overloaded constructors: one that takes the title and author of the book, and another that takes the title, author, and year of publication. Demonstrate this in the main method.

Solution

class Book { String title; String author; int year; Book(String t, String a) { title = t; author = a; year = 0; } Book(String t, String a, int y) { title = t; author = a; year = y; } void display() { System.out.println("Title: " + title); System.out.println("Author: " + author); if (year != 0) System.out.println("Year: " + year); } } class _01_ { public static void main(String[] args) { Book b1 = new Book("Java", "James"); Book b2 = new Book("Python", "Guido", 1991); b1.display(); b2.display(); } }

Sample Output

Title: Java Author: James Title: Python Author: Guido Year: 1991
Key Concepts:
  • Constructor Overloading: Multiple constructors with different parameters
  • Compile-Time Polymorphism: Constructor selection happens at compile time
  • Default Values: First constructor sets year to 0 as default
  • Flexibility: Allows object creation with different levels of detail
Use Case: Providing multiple ways to initialize objects based on available data

Question 2: Method Overriding (Runtime Polymorphism)

Problem Statement

Create an Animal class with a method makeSound() that prints "Some sound". Create a Dog class that extends Animal and overrides the makeSound() method to print "Bark". Demonstrate this in the main method.

Solution

class Animal { void makeSound() { System.out.println("Some sound"); } } class Dog extends Animal { void makeSound() { System.out.println("Bark"); } } class _02_ { public static void main(String[] args) { Animal a = new Dog(); a.makeSound(); } }

Sample Output

Bark
Key Concepts:
  • Method Overriding: Child class provides specific implementation
  • Runtime Polymorphism: Method call resolved at runtime based on object type
  • Dynamic Binding: JVM determines which method to call at runtime
  • Parent Reference: Animal reference points to Dog object
Output Explanation: Dog's makeSound() is called, not Animal's

Question 3: Method Overloading (Compile-Time Polymorphism)

Problem Statement

Create a Calculator class with overloaded methods add() that can handle different numbers of parameters: one that adds two integers, one that adds three integers, and one that adds two double values. Demonstrate this in the main method.

Solution

class Calculator { int add(int a, int b) { return a + b; } int add(int a, int b, int c) { return a + b + c; } double add(double a, double b) { return a + b; } } class _03_ { public static void main(String[] args) { Calculator c = new Calculator(); System.out.println(c.add(2, 3)); System.out.println(c.add(1, 2, 3)); System.out.println(c.add(2.5, 3.5)); } }

Sample Output

5 6 6.0
Key Concepts:
  • Method Overloading: Same method name, different parameters
  • Parameter Variation: Different number or types of parameters
  • Compile-Time Resolution: Compiler determines which method to call
  • Return Types: Can vary (int vs double) based on parameters
Rules: Methods must differ in number, type, or order of parameters

Question 4: Runtime Polymorphism Using Shape Reference

Problem Statement

Create a Shape class with a method draw(). Create Circle and Rectangle classes that extend Shape and override the draw() method to print "Drawing Circle" and "Drawing Rectangle", respectively. Use a Shape reference to point to Circle and Rectangle objects and call the draw() method to demonstrate runtime polymorphism.

Solution

class Shape { void draw() { System.out.println("Drawing Shape"); } } class Circle extends Shape { void draw() { System.out.println("Drawing Circle"); } } class Rectangle extends Shape { void draw() { System.out.println("Drawing Rectangle"); } } class _04_ { public static void main(String[] args) { Shape s1 = new Circle(); Shape s2 = new Rectangle(); s1.draw(); s2.draw(); } }

Sample Output

Drawing Circle Drawing Rectangle
Key Concepts:
  • Upcasting: Child objects assigned to parent reference
  • Dynamic Method Dispatch: Correct method called based on actual object
  • Polymorphic Behavior: Same reference type, different behaviors
  • Flexibility: Can treat different shapes uniformly
Advantage: Write code that works with parent type but executes child-specific logic

Question 5: Method Overriding with Parent Reference

Problem Statement

Create a Vehicle class with a method start() that prints "Vehicle starting". Create a Car class that extends Vehicle and overrides the start() method to print "Car starting". Demonstrate this in the main method.

Solution

class Vehicle { void start() { System.out.println("Vehicle starting"); } } class Car extends Vehicle { void start() { System.out.println("Car starting"); } } class _05_ { public static void main(String[] args) { Vehicle v = new Car(); v.start(); } }

Sample Output

Car starting
Key Concepts:
  • Runtime Polymorphism: Method resolution happens at runtime
  • Object Type Matters: Actual object type (Car) determines method called
  • Reference Type: Vehicle reference can hold Car object
  • Override Annotation: @Override can be used for clarity (optional)
Key Point: Reference type is Vehicle, but object type is Car, so Car's start() executes
