Access Modifiers

Access Modifiers in Java

Control visibility and accessibility of classes, methods, and variables.

What are Access Modifiers?

Access modifiers in Java are keywords that set the accessibility (visibility) of classes, methods, constructors, and variables. They control which parts of your code can access specific members of a class.

Four Types of Access Modifiers:
  • public - Accessible from anywhere
  • private - Accessible only within the same class
  • protected - Accessible within the same package and subclasses
  • default (no modifier) - Accessible only within the same package
💡 Important: Access modifiers are fundamental to encapsulation, one of the four pillars of OOP.

1. Public Access Modifier

The public modifier makes a class, method, or variable accessible from anywhere in the program.

Example 1: Public Class and Methods
public class PublicDemo {
    public String name = "Public Variable";
    
    public void display() {
        System.out.println("This is a public method");
    }
}

class TestPublic {
    public static void main(String[] args) {
        PublicDemo obj = new PublicDemo();
        System.out.println(obj.name);  // Accessible
        obj.display();                  // Accessible
    }
}

This is a public method
Example 2: Public Constructor
public class Student {
    private String name;
    private int age;
    
    // Public constructor - can be called from anywhere
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    public void displayInfo() {
        System.out.println("Name: " + name + ", Age: " + age);
    }
}

public class TestStudent {
    public static void main(String[] args) {
        Student s = new Student("Alice", 20);
        s.displayInfo();
    }
}
Name: Alice, Age: 20
✅ Use Public When: You want the member to be accessible from any class in any package.

2. Private Access Modifier

The private modifier makes a member accessible only within the same class. It provides the highest level of encapsulation.

Example 1: Private Variables
class BankAccount {
    private String accountNumber;
    private double balance;
    
    public BankAccount(String accNum, double bal) {
        accountNumber = accNum;
        balance = bal;
    }
    
    // Public methods to access private variables
    public double getBalance() {
        return balance;
    }
    
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            System.out.println("Deposited: " + amount);
        }
    }
}

public class TestBank {
    public static void main(String[] args) {
        BankAccount acc = new BankAccount("ACC001", 1000);
        
        // acc.balance = 5000;  // ERROR: balance is private
        
        System.out.println("Balance: " + acc.getBalance());
        acc.deposit(500);
        System.out.println("New Balance: " + acc.getBalance());
    }
}
Balance: 1000.0
Deposited: 500.0
New Balance: 1500.0
Example 2: Private Methods
class Calculator {
    private int add(int a, int b) {
        return a + b;
    }
    
    private int multiply(int a, int b) {
        return a * b;
    }
    
    public int calculate(int a, int b) {
        int sum = add(a, b);           // Private method called internally
        int product = multiply(a, b);   // Private method called internally
        return sum + product;
    }
}

public class TestCalculator {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        
        // calc.add(5, 3);  // ERROR: add() is private
        
        System.out.println("Result: " + calc.calculate(5, 3));
    }
}
Result: 23
✅ Use Private When: You want to hide implementation details and protect data from external access.

3. Protected Access Modifier

The protected modifier makes a member accessible within the same package and in subclasses (even if they're in different packages).

Example 1: Protected Variables in Inheritance
class Animal {
    protected String species;
    protected int age;
    
    protected void displayInfo() {
        System.out.println("Species: " + species + ", Age: " + age);
    }
}

class Dog extends Animal {
    void setDetails(String species, int age) {
        this.species = species;  // Accessible - protected in parent
        this.age = age;          // Accessible - protected in parent
    }
    
    void show() {
        displayInfo();  // Accessible - protected method from parent
    }
}

public class TestProtected {
    public static void main(String[] args) {
        Dog dog = new Dog();
        dog.setDetails("Canine", 5);
        dog.show();
    }
}
Species: Canine, Age: 5
Example 2: Protected Methods
class Vehicle {
    protected void startEngine() {
        System.out.println("Engine started");
    }
    
    protected void stopEngine() {
        System.out.println("Engine stopped");
    }
}

class Car extends Vehicle {
    public void drive() {
        startEngine();  // Protected method accessible in subclass
        System.out.println("Car is driving");
        stopEngine();   // Protected method accessible in subclass
    }
}

public class TestVehicle {
    public static void main(String[] args) {
        Car car = new Car();
        car.drive();
    }
}
Engine started
Car is driving
Engine stopped
✅ Use Protected When: You want subclasses to access the member but keep it hidden from unrelated classes.

4. Default Access Modifier (Package-Private)

When no access modifier is specified, it's called default or package-private. Members are accessible only within the same package.

Example 1: Default Access
class DefaultDemo {
    String message = "Default access";  // No modifier = default
    
    void display() {  // No modifier = default
        System.out.println(message);
    }
}

class TestDefault {
    public static void main(String[] args) {
        DefaultDemo obj = new DefaultDemo();
        System.out.println(obj.message);  // Accessible - same package
        obj.display();                     // Accessible - same package
    }
}

Example 2: Package-Level Access
// File: Helper.java (in package com.example)
package com.example;

class Helper {  // Default class - accessible only in com.example package
    void assist() {
        System.out.println("Helping within package");
    }
}

// File: Main.java (in package com.example)
package com.example;

public class Main {
    public static void main(String[] args) {
        Helper helper = new Helper();  // Accessible - same package
        helper.assist();
    }
}
Helping within package
✅ Use Default When: You want to restrict access to classes within the same package only.

Access Modifiers Comparison

Accessibility Table:
Modifier Same Class Same Package Subclass Other Packages
public ✅ Yes ✅ Yes ✅ Yes ✅ Yes
protected ✅ Yes ✅ Yes ✅ Yes ❌ No
default ✅ Yes ✅ Yes ❌ No ❌ No
private ✅ Yes ❌ No ❌ No ❌ No
Complete Example - All Modifiers:
public class AccessModifierDemo {
    public String publicVar = "Public";
    protected String protectedVar = "Protected";
    String defaultVar = "Default";
    private String privateVar = "Private";
    
    public void testAccess() {
        // All accessible within the same class
        System.out.println(publicVar);
        System.out.println(protectedVar);
        System.out.println(defaultVar);
        System.out.println(privateVar);
    }
}

class SamePackageClass {
    void test() {
        AccessModifierDemo obj = new AccessModifierDemo();
        System.out.println(obj.publicVar);     // ✅ Accessible
        System.out.println(obj.protectedVar);  // ✅ Accessible
        System.out.println(obj.defaultVar);    // ✅ Accessible
        // System.out.println(obj.privateVar); // ❌ NOT Accessible
    }
}

class SubclassInSamePackage extends AccessModifierDemo {
    void test() {
        System.out.println(publicVar);     // ✅ Accessible
        System.out.println(protectedVar);  // ✅ Accessible
        System.out.println(defaultVar);    // ✅ Accessible
        // System.out.println(privateVar); // ❌ NOT Accessible
    }
}
Best Practices:
Scenario Recommended Modifier
Instance variables private
Getter/Setter methods public
Methods for subclasses protected
Helper methods (internal use) private
Package-level utilities default
API methods public
