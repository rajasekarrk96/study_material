Question 1: Single Inheritance with Method Overriding

Problem Statement

Create an Animal class with a method makeSound(). Then, create a Dog class that extends the Animal class and overrides the makeSound() method to print "Bark". Demonstrate the use of these classes in the main method.

Solution

class Animal { void makeSound() { System.out.println("Animal sound"); } } class Dog extends Animal { void makeSound() { System.out.println("Bark"); } } class _01_ { public static void main(String[] args) { Animal a = new Dog(); a.makeSound(); } }

Sample Output

Bark
Key Concepts:
  • Single Inheritance: Dog extends Animal (one parent, one child)
  • Method Overriding: Dog provides its own implementation of makeSound()
  • Polymorphism: Animal reference holds Dog object
  • Runtime Binding: Dog's method is called at runtime
Use Case: Specializing behavior in child classes

Question 2: Multilevel Inheritance

Problem Statement

Create a Vehicle class with a method move(). Create a Car class that extends Vehicle and adds a method honk(). Finally, create an ElectricCar class that extends Car and adds a method charge(). Demonstrate the use of these classes in the main method.

Solution

class Vehicle { void move() { System.out.println("Vehicle is moving"); } } class Car extends Vehicle { void honk() { System.out.println("Car is honking"); } } class ElectricCar extends Car { void charge() { System.out.println("Electric car is charging"); } } class _02_ { public static void main(String[] args) { ElectricCar ec = new ElectricCar(); ec.move(); ec.honk(); ec.charge(); } }

Sample Output

Vehicle is moving Car is honking Electric car is charging
Key Concepts:
  • Multilevel Inheritance: Vehicle → Car → ElectricCar chain
  • Method Accumulation: ElectricCar has all methods from ancestors
  • Inheritance Chain: Each level adds new functionality
  • Access: Child can access all public/protected members of ancestors
Hierarchy: ElectricCar inherits from Car, which inherits from Vehicle

Question 3: Hierarchical Inheritance

Problem Statement

Create a Shape class with a method draw(). Create Circle and Rectangle classes that both extend Shape and override the draw() method to print "Drawing Circle" and "Drawing Rectangle", respectively. Demonstrate the use of these classes in the main method.

Solution

class Shape { void draw() { System.out.println("Drawing Shape"); } } class Circle extends Shape { void draw() { System.out.println("Drawing Circle"); } } class Rectangle extends Shape { void draw() { System.out.println("Drawing Rectangle"); } } class _03_ { public static void main(String[] args) { Shape s1 = new Circle(); Shape s2 = new Rectangle(); s1.draw(); s2.draw(); } }

Sample Output

Drawing Circle Drawing Rectangle
Key Concepts:
  • Hierarchical Inheritance: Multiple children (Circle, Rectangle) from one parent (Shape)
  • Common Base: Both classes share Shape as parent
  • Polymorphic Behavior: Same reference type, different implementations
  • Code Reuse: Common functionality in parent, specific in children
Pattern: One parent class, multiple child classes

Question 4: Deep Multilevel Inheritance Chain

Problem Statement

Create an Employee class with a method work(). Create a Manager class that extends Employee and adds a method manage(). Create a TeamLead class that extends Manager and adds a method lead(). Finally, create a Developer class that extends TeamLead and adds a method code(). Demonstrate the use of these classes in the main method.

Solution

class Employee { void work() { System.out.println("Employee is working"); } } class Manager extends Employee { void manage() { System.out.println("Manager is managing"); } } class TeamLead extends Manager { void lead() { System.out.println("TeamLead is leading"); } } class Developer extends TeamLead { void code() { System.out.println("Developer is coding"); } } class _04_ { public static void main(String[] args) { Developer d = new Developer(); d.work(); d.manage(); d.lead(); d.code(); } }

Sample Output

Employee is working Manager is managing TeamLead is leading Developer is coding
Key Concepts:
  • 4-Level Hierarchy: Employee → Manager → TeamLead → Developer
  • Method Inheritance: Developer has access to all ancestor methods
  • Specialization: Each level adds specific functionality
  • IS-A Relationship: Developer IS-A TeamLead IS-A Manager IS-A Employee
Design: Each level represents increasing specialization

Question 5: Academic Hierarchy with Multilevel Inheritance

Problem Statement

Create a Person class with a method introduce(). Create a Student class that extends Person and adds a method study(). Create a PostgraduateStudent class that extends Student and adds a method research(). Finally, create a PhDStudent class that extends PostgraduateStudent and adds a method dissertation(). Demonstrate the use of these classes in the main method.

Solution

class Person { void introduce() { System.out.println("I am a person"); } } class Student extends Person { void study() { System.out.println("Student is studying"); } } class PostgraduateStudent extends Student { void research() { System.out.println("Postgraduate student is researching"); } } class PhDStudent extends PostgraduateStudent { void dissertation() { System.out.println("PhD student is writing dissertation"); } } class _05_ { public static void main(String[] args) { PhDStudent p = new PhDStudent(); p.introduce(); p.study(); p.research(); p.dissertation(); } }

Sample Output

I am a person Student is studying Postgraduate student is researching PhD student is writing dissertation
Key Concepts:
  • Academic Progression: Person → Student → Postgraduate → PhD
  • Cumulative Capabilities: Each level adds new methods
  • Real-World Modeling: Represents actual academic hierarchy
  • Method Access: PhDStudent can call all inherited methods
Hierarchy: Models progression from general person to specialized PhD student
