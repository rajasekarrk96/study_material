Question 1: Flyable Interface

Problem Statement

Write a Java program to create an interface Flyable with a method called fly_obj(). Create three classes Spacecraft, Airplane, and Helicopter that implement the Flyable interface. Implement the fly_obj() method for each of the three classes.

Solution

interface Flyable { void fly_obj(); } class Spacecraft implements Flyable { public void fly_obj() { System.out.println("Spacecraft is flying"); } } class Airplane implements Flyable { public void fly_obj() { System.out.println("Airplane is flying"); } } class Helicopter implements Flyable { public void fly_obj() { System.out.println("Helicopter is flying"); } } class _01_ { public static void main(String[] args) { Flyable f1 = new Spacecraft(); Flyable f2 = new Airplane(); Flyable f3 = new Helicopter(); f1.fly_obj(); f2.fly_obj(); f3.fly_obj(); } }

Sample Output

Spacecraft is flying Airplane is flying Helicopter is flying
Key Concepts:
  • Interface: Defines a contract that classes must implement
  • Polymorphism: All three classes treated as Flyable type
  • Abstract Method: fly_obj() has no body in interface
  • Multiple Implementations: Each class provides its own implementation
Real-World Use: Common behavior across different types of objects

Question 2: Encryptable Interface

Problem Statement

Write a Java program to create an interface Encryptable with methods encrypt(String data) and decrypt(String encryptedData) that define encryption and decryption operations. Create two classes AES and RSA that implement the Encryptable interface and provide their own encryption and decryption algorithms.

Solution

interface Encryptable { String encrypt(String data); String decrypt(String encryptedData); } class AES implements Encryptable { public String encrypt(String data) { return "AES_ENC(" + data + ")"; } public String decrypt(String encryptedData) { return "AES_DEC(" + encryptedData + ")"; } } class RSA implements Encryptable { public String encrypt(String data) { return "RSA_ENC(" + data + ")"; } public String decrypt(String encryptedData) { return "RSA_DEC(" + encryptedData + ")"; } } class _02_ { public static void main(String[] args) { Encryptable e1 = new AES(); Encryptable e2 = new RSA(); System.out.println(e1.encrypt("HELLO")); System.out.println(e2.encrypt("WORLD")); } }

Sample Output

AES_ENC(HELLO) RSA_ENC(WORLD)
Key Concepts:
  • Multiple Methods: Interface can declare multiple abstract methods
  • Return Types: Methods can return values (String in this case)
  • Strategy Pattern: Different encryption algorithms with same interface
  • Interchangeable: Can swap AES with RSA without changing client code
Real-World Use: Security systems, plugin architectures, algorithm selection

Question 3: Sortable Interface

Problem Statement

Write a Java program to create an interface Sortable with a method sort() that sorts an array of integers in ascending order. Create two classes BubbleSort and SelectionSort that implement the Sortable interface and provide their own implementations of the sort() method.

Solution

interface Sortable { void sort(int[] arr); } class BubbleSort implements Sortable { public void sort(int[] arr) { for (int i = 0; i < arr.length; i++) { for (int j = 0; j < arr.length - 1; j++) { if (arr[j] > arr[j + 1]) { int t = arr[j]; arr[j] = arr[j + 1]; arr[j + 1] = t; } } } } } class SelectionSort implements Sortable { public void sort(int[] arr) { for (int i = 0; i < arr.length; i++) { int min = i; for (int j = i + 1; j < arr.length; j++) { if (arr[j] < arr[min]) min = j; } int t = arr[i]; arr[i] = arr[min]; arr[min] = t; } } } class _03_ { public static void main(String[] args) { int[] arr = {5, 3, 1, 4, 2}; Sortable s = new BubbleSort(); s.sort(arr); for (int i : arr) System.out.print(i + " "); } }

Sample Output

1 2 3 4 5
Key Concepts:
  • Array Parameters: Interface methods can accept arrays
  • Different Algorithms: BubbleSort vs SelectionSort implementation
  • In-Place Modification: Array is modified directly
  • Algorithm Abstraction: Client doesn't need to know sorting details
Comparison: BubbleSort: O(n²), SelectionSort: O(n²) - both quadratic time

Question 4: Playable Interface

Problem Statement

Write a Java program to create an interface Playable with a method play() that takes no arguments and returns void. Create three classes Football, Volleyball, and Basketball that implement the Playable interface and override the play() method to play the respective sports.

Solution

interface Playable { void play(); } class Football implements Playable { public void play() { System.out.println("Playing Football"); } } class Volleyball implements Playable { public void play() { System.out.println("Playing Volleyball"); } } class Basketball implements Playable { public void play() { System.out.println("Playing Basketball"); } } class _04_ { public static void main(String[] args) { Playable p1 = new Football(); Playable p2 = new Volleyball(); Playable p3 = new Basketball(); p1.play(); p2.play(); p3.play(); } }

Sample Output

Playing Football Playing Volleyball Playing Basketball
Key Concepts:
  • Simple Interface: Single method with no parameters
  • Uniform Behavior: All sports have common play() action
  • Reference Type: Playable reference can hold any implementing class
  • Loose Coupling: Main method doesn't depend on specific sport classes
Design Benefit: Easy to add new sports without changing existing code

Question 5: Searchable Interface

Problem Statement

Write a Java program to create an interface Searchable with a method search(String keyword) that searches for a given keyword in a text document. Create two classes Document and WebPage that implement the Searchable interface and provide their own implementations of the search() method.

Solution

interface Searchable { void search(String keyword); } class Document implements Searchable { public void search(String keyword) { System.out.println("Searching \"" + keyword + "\" in Document"); } } class WebPage implements Searchable { public void search(String keyword) { System.out.println("Searching \"" + keyword + "\" in WebPage"); } } class _05_ { public static void main(String[] args) { Searchable s1 = new Document(); Searchable s2 = new WebPage(); s1.search("Java"); s2.search("OOPS"); } }

Sample Output

Searching "Java" in Document Searching "OOPS" in WebPage
Key Concepts:
  • String Parameters: Interface methods can accept String arguments
  • Common Functionality: Both Document and WebPage support searching
  • Different Contexts: Same operation in different environments
  • Extensibility: Easy to add new searchable types (PDF, Email, etc.)
Real-World Use: Search engines, content management systems, file explorers
