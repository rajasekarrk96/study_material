Question 1: Member Inner Class - Pet Owner System

Problem Statement

Imagine you are building a system to manage pets and their owners. Create an outer class Owner representing a pet owner, with attributes like name and age. Inside Owner, define an inner class Pet representing a pet, with attributes like name, age, and species. In the main() method, instantiate an Owner object and then create a Pet object inside it. Finally, display the owner's name and the pet's details.

Solution

class Owner { String name; int age; Owner(String n, int a) { name = n; age = a; } class Pet { String petName; int petAge; String species; Pet(String pn, int pa, String s) { petName = pn; petAge = pa; species = s; } void displayPet() { System.out.println("Pet Name: " + petName); System.out.println("Pet Age: " + petAge); System.out.println("Species: " + species); } } } class _01_ { public static void main(String[] args) { Owner o = new Owner("Ramesh", 35); Owner.Pet p = o.new Pet("Buddy", 3, "Dog"); System.out.println("Owner Name: " + o.name); p.displayPet(); } }

Sample Output

Owner Name: Ramesh Pet Name: Buddy Pet Age: 3 Species: Dog
Key Concepts:
  • Member Inner Class: Pet class is non-static and nested inside Owner
  • Object Creation: Syntax: Owner.Pet p = o.new Pet(...)
  • Relationship: Pet depends on Owner instance
  • Access: Inner class can access outer class members
Real-World Use: Modeling has-a relationships where inner entity depends on outer

Question 2: Method-Local Inner Class

Problem Statement

Write a Java program that demonstrates the concept of method local inner classes. In the Main class, define a method outerMethod() that declares a local variable outerVar initialized to 20. Inside outerMethod(), define a method-local inner class Inner that has a method display() which prints the value of outerVar. Finally, invoke outerMethod() and within it, instantiate the inner class Inner and call its display() method.

Solution

class _02_ { void outerMethod() { int outerVar = 20; class Inner { void display() { System.out.println("Outer Variable: " + outerVar); } } Inner i = new Inner(); i.display(); } public static void main(String[] args) { _02_ obj = new _02_(); obj.outerMethod(); } }

Sample Output

Outer Variable: 20
Key Concepts:
  • Local Inner Class: Defined inside a method
  • Variable Access: Can access final/effectively final local variables
  • Scope: Inner class only visible within outerMethod()
  • Effectively Final: outerVar is not modified, so it's accessible
Use Case: When helper class is needed only within a specific method

Question 3: Static Nested Class - Restaurant Menu System

Problem Statement

Design a system for managing restaurants and their menus. Define an outer class Restaurant representing a restaurant, with attributes like name and location. Inside Restaurant, define a static nested class Menu representing the menu offered by the restaurant, with attributes like name, description, and price of items. In the main() method, create a Restaurant object, and then create instances of Menu to represent different menus offered by the restaurant. Display their details.

Solution

class Restaurant { String name; String location; Restaurant(String n, String l) { name = n; location = l; } static class Menu { String itemName; String description; double price; Menu(String i, String d, double p) { itemName = i; description = d; price = p; } void displayMenu() { System.out.println("Item: " + itemName); System.out.println("Description: " + description); System.out.println("Price: " + price); } } } class _03_ { public static void main(String[] args) { Restaurant r = new Restaurant("Spice Hub", "Chennai"); Restaurant.Menu m1 = new Restaurant.Menu("Biryani", "Chicken Biryani", 250); Restaurant.Menu m2 = new Restaurant.Menu("Dosa", "Plain Dosa", 50); System.out.println("Restaurant: " + r.name); m1.displayMenu(); m2.displayMenu(); } }

Sample Output

Restaurant: Spice Hub Item: Biryani Description: Chicken Biryani Price: 250.0 Item: Dosa Description: Plain Dosa Price: 50.0
Key Concepts:
  • Static Nested Class: Menu doesn't need Restaurant instance
  • Object Creation: new Restaurant.Menu(...) - no outer instance needed
  • Independence: Menu can exist without specific Restaurant object
  • Logical Grouping: Menu is logically related to Restaurant
Advantages: Menu items can be created independently of restaurant instances

Question 4: Anonymous Inner Class - Shopping Cart Payment

Problem Statement

Create a Java program that models a shopping cart functionality. Define an interface Payment with a method processPayment(double amount) to simulate processing payments. In the ShoppingCart class, implement a method checkout(double totalAmount, Payment payment) that takes the total amount of the items in the cart and a payment object, and simulates the checkout process by processing the payment. Use an anonymous inner class to implement the Payment interface within the main() method to demonstrate a cash payment and a credit card payment.

Solution

interface Payment { void processPayment(double amount); } class ShoppingCart { void checkout(double totalAmount, Payment payment) { payment.processPayment(totalAmount); } } class _04_ { public static void main(String[] args) { ShoppingCart cart = new ShoppingCart(); cart.checkout(500, new Payment() { public void processPayment(double amount) { System.out.println("Cash Payment of Rs." + amount); } }); cart.checkout(1200, new Payment() { public void processPayment(double amount) { System.out.println("Credit Card Payment of Rs." + amount); } }); } }

Sample Output

Cash Payment of Rs.500.0 Credit Card Payment of Rs.1200.0
Key Concepts:
  • Anonymous Inner Class: Implements Payment interface inline
  • No Class Name: Defined and instantiated in one expression
  • Multiple Implementations: Different behavior for each payment type
  • Strategy Pattern: Different payment strategies passed as objects
Modern Alternative: Lambda: cart.checkout(500, amount -> System.out.println(...))

Question 5: Nested Inner Classes - Building Management System

Problem Statement

Design a program to manage apartment buildings. Create an outer class Building representing a building, with attributes like address and number of floors. Inside Building, define an inner class Floor representing a floor of the building, with attributes like number and number of apartments. Inside Floor, define another inner class Apartment representing an individual apartment, with attributes like number, size, and occupancy status. In the main() method, create a Building object, add some floors and apartments to it, and display their details.

Solution

class Building { String address; int floors; Building(String a, int f) { address = a; floors = f; } class Floor { int floorNumber; int apartments; Floor(int fn, int ap) { floorNumber = fn; apartments = ap; } class Apartment { int number; double size; boolean occupied; Apartment(int n, double s, boolean o) { number = n; size = s; occupied = o; } void displayApartment() { System.out.println("Apartment No: " + number); System.out.println("Size: " + size); System.out.println("Occupied: " + occupied); } } } } class _05_ { public static void main(String[] args) { Building b = new Building("MG Road", 5); Building.Floor f = b.new Floor(2, 4); Building.Floor.Apartment a = f.new Apartment(201, 850.5, true); System.out.println("Building Address: " + b.address); System.out.println("Floor Number: " + f.floorNumber); a.displayApartment(); } }

Sample Output

Building Address: MG Road Floor Number: 2 Apartment No: 201 Size: 850.5 Occupied: true
Key Concepts:
  • Nested Inner Classes: Apartment is inside Floor, which is inside Building
  • Multi-Level Nesting: Three levels of class hierarchy
  • Object Creation Chain: Building → Floor → Apartment
  • Hierarchical Relationship: Models real-world containment
Real-World Use: Complex hierarchical structures like organizations, file systems
