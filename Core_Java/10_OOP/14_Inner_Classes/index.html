Inner Classes

Inner Classes in Java

Classes defined within another class for better encapsulation and code organization.

What are Inner Classes?

Inner classes (also called nested classes) are classes defined within another class. They provide a way to logically group classes that are only used in one place, increase encapsulation, and make code more readable and maintainable.

Types of Inner Classes:
Type Description Access to Outer Class
Member Inner Class Non-static class inside another class Can access all members (including private)
Static Nested Class Static class inside another class Can only access static members
Local Inner Class Class defined inside a method Can access final/effectively final local variables
Anonymous Inner Class Class without a name, defined inline Used for one-time implementations
Advantages of Inner Classes:
  • Encapsulation: Hide implementation details from other classes
  • Readability: Logical grouping of classes that belong together
  • Access: Can access private members of outer class
  • Callbacks: Useful for event handling and callbacks

1. Member Inner Class

A non-static class defined at the member level of another class. It has access to all members of the outer class, including private members.

Syntax:
class OuterClass {
    // Outer class members
    
    class InnerClass {
        // Inner class members
    }
}
Example 1: Basic Member Inner Class
class Outer {
    private String message = "Hello from Outer Class";
    
    // Member Inner Class
    class Inner {
        void display() {
            System.out.println(message);  // Can access private member
        }
    }
    
    void createInner() {
        Inner inner = new Inner();
        inner.display();
    }
}

public class MemberInnerDemo {
    public static void main(String[] args) {
        Outer outer = new Outer();
        outer.createInner();
        
        // Alternative way to create inner class object
        Outer.Inner inner = outer.new Inner();
        inner.display();
    }
}

Hello from Outer Class
Example 2: Inner Class Accessing Outer Members
class Car {
    private String brand = "Toyota";
    private int speed = 0;
    
    class Engine {
        private int horsepower = 150;
        
        void start() {
            speed = 60;  // Modify outer class variable
            System.out.println(brand + " engine started!");
            System.out.println("Speed: " + speed + " km/h");
            System.out.println("Horsepower: " + horsepower);
        }
    }
    
    void startCar() {
        Engine engine = new Engine();
        engine.start();
    }
}

public class CarDemo {
    public static void main(String[] args) {
        Car car = new Car();
        car.startCar();
    }
}
Toyota engine started!
Speed: 60 km/h
Horsepower: 150
💡 Key Points:
  • Inner class object requires an outer class object
  • Can access all members of outer class (even private)
  • Cannot have static members (except static final constants)
  • Syntax to create: Outer.Inner inner = outer.new Inner();

2. Static Nested Class

A static class defined inside another class. It doesn't require an instance of the outer class and can only access static members of the outer class.

Syntax:
class OuterClass {
    static class StaticNestedClass {
        // Static nested class members
    }
}
Example 1: Basic Static Nested Class
class Outer {
    private static String staticMessage = "Static Message";
    private String instanceMessage = "Instance Message";
    
    static class StaticNested {
        void display() {
            System.out.println(staticMessage);  // Can access static members
            // System.out.println(instanceMessage);  // ERROR: Cannot access instance members
        }
    }
}

public class StaticNestedDemo {
    public static void main(String[] args) {
        // No need for outer class instance
        Outer.StaticNested nested = new Outer.StaticNested();
        nested.display();
    }
}
Static Message
Example 2: Builder Pattern Using Static Nested Class
class Student {
    private String name;
    private int age;
    private String course;
    
    private Student(Builder builder) {
        this.name = builder.name;
        this.age = builder.age;
        this.course = builder.course;
    }
    
    // Static Nested Builder Class
    static class Builder {
        private String name;
        private int age;
        private String course;
        
        Builder setName(String name) {
            this.name = name;
            return this;
        }
        
        Builder setAge(int age) {
            this.age = age;
            return this;
        }
        
        Builder setCourse(String course) {
            this.course = course;
            return this;
        }
        
        Student build() {
            return new Student(this);
        }
    }
    
    void display() {
        System.out.println("Name: " + name);
        System.out.println("Age: " + age);
        System.out.println("Course: " + course);
    }
}

public class BuilderDemo {
    public static void main(String[] args) {
        Student student = new Student.Builder()
            .setName("John")
            .setAge(20)
            .setCourse("Computer Science")
            .build();
        
        student.display();
    }
}
Name: John
Age: 20
Course: Computer Science
💡 Key Points:
  • Doesn't need outer class instance to be created
  • Can only access static members of outer class
  • Can have static and non-static members
  • Commonly used for Builder pattern

3. Local Inner Class

A class defined inside a method or block. It can access the members of the outer class and final or effectively final local variables of the enclosing method.

Example 1: Basic Local Inner Class
class Outer {
    private String outerField = "Outer Field";
    
    void outerMethod() {
        final String localVar = "Local Variable";
        
        // Local Inner Class
        class LocalInner {
            void display() {
                System.out.println(outerField);
                System.out.println(localVar);  // Can access final local variables
            }
        }
        
        LocalInner inner = new LocalInner();
        inner.display();
    }
}

public class LocalInnerDemo {
    public static void main(String[] args) {
        Outer outer = new Outer();
        outer.outerMethod();
    }
}
Outer Field
Local Variable
Example 2: Local Class for Validation
class Validator {
    void validateEmail(String email) {
        final String emailPattern = "^[A-Za-z0-9+_.-]+@(.+)$";
        
        // Local Inner Class
        class EmailValidator {
            boolean isValid() {
                return email.matches(emailPattern);
            }
            
            void printResult() {
                if (isValid()) {
                    System.out.println(email + " is valid");
                } else {
                    System.out.println(email + " is invalid");
                }
            }
        }
        
        EmailValidator validator = new EmailValidator();
        validator.printResult();
    }
}

public class ValidationDemo {
    public static void main(String[] args) {
        Validator validator = new Validator();
        validator.validateEmail("test@example.com");
        validator.validateEmail("invalid-email");
    }
}
test@example.com is valid
invalid-email is invalid
💡 Key Points:
  • Defined inside a method or block
  • Scope limited to the method/block where it's defined
  • Can access final or effectively final local variables
  • Cannot be accessed outside the method

4. Anonymous Inner Class

A class without a name that is defined and instantiated in a single expression. Commonly used for implementing interfaces or extending classes on-the-fly.

Example 1: Anonymous Class Implementing Interface
interface Greeting {
    void sayHello();
}

public class AnonymousDemo {
    public static void main(String[] args) {
        // Anonymous Inner Class
        Greeting greeting = new Greeting() {
            @Override
            public void sayHello() {
                System.out.println("Hello from Anonymous Class!");
            }
        };
        
        greeting.sayHello();
    }
}
Hello from Anonymous Class!
Example 2: Anonymous Class Extending Abstract Class
abstract class Animal {
    abstract void makeSound();
}

public class AnimalDemo {
    public static void main(String[] args) {
        // Anonymous class extending Animal
        Animal dog = new Animal() {
            @Override
            void makeSound() {
                System.out.println("Woof! Woof!");
            }
        };
        
        Animal cat = new Animal() {
            @Override
            void makeSound() {
                System.out.println("Meow! Meow!");
            }
        };
        
        dog.makeSound();
        cat.makeSound();
    }
}
Woof! Woof!
Meow! Meow!
Example 3: Anonymous Class with Thread
public class ThreadDemo {
    public static void main(String[] args) {
        // Anonymous class implementing Runnable
        Thread thread = new Thread(new Runnable() {
            @Override
            public void run() {
                for (int i = 1; i <= 5; i++) {
                    System.out.println("Count: " + i);
                }
            }
        });
        
        thread.start();
    }
}
Count: 1
Count: 2
Count: 3
Count: 4
Count: 5
💡 Key Points:
  • No class name, defined inline
  • Used for one-time implementations
  • Can implement one interface or extend one class
  • Cannot have constructors (no name!)
  • Modern alternative: Lambda expressions (Java 8+)

Practical Examples

Example 1: Data Structure with Inner Class
class LinkedList {
    private Node head;
    
    // Member Inner Class
    private class Node {
        int data;
        Node next;
        
        Node(int data) {
            this.data = data;
            this.next = null;
        }
    }
    
    void add(int data) {
        Node newNode = new Node(data);
        if (head == null) {
            head = newNode;
        } else {
            Node temp = head;
            while (temp.next != null) {
                temp = temp.next;
            }
            temp.next = newNode;
        }
    }
    
    void display() {
        Node temp = head;
        while (temp != null) {
            System.out.print(temp.data + " -> ");
            temp = temp.next;
        }
        System.out.println("null");
    }
}

public class LinkedListDemo {
    public static void main(String[] args) {
        LinkedList list = new LinkedList();
        list.add(10);
        list.add(20);
        list.add(30);
        list.display();
    }
}
10 -> 20 -> 30 -> null
Example 2: Event Handling with Anonymous Class
interface ClickListener {
    void onClick();
}

class Button {
    private String label;
    private ClickListener listener;
    
    Button(String label) {
        this.label = label;
    }
    
    void setOnClickListener(ClickListener listener) {
        this.listener = listener;
    }
    
    void click() {
        System.out.println(label + " button clicked!");
        if (listener != null) {
            listener.onClick();
        }
    }
}

public class EventDemo {
    public static void main(String[] args) {
        Button submitBtn = new Button("Submit");
        
        // Anonymous inner class for event handling
        submitBtn.setOnClickListener(new ClickListener() {
            @Override
            public void onClick() {
                System.out.println("Form submitted!");
            }
        });
        
        submitBtn.click();
    }
}
Submit button clicked!
Form submitted!
