Question 1: Person Class - Parameterized Constructor

Problem Statement

Create a Person class with the following attributes and methods: • Private attributes name and age • A parameterized constructor that initializes name and age • Public methods getName and getAge to return the name and age • A method displayInfo to print the name and age

Solution

import java.util.Scanner; class _01_ { private String name; private int age; _01_(String n, int a) { name = n; age = a; } public String getName() { return name; } public int getAge() { return age; } void displayInfo() { System.out.println("Name: " + name); System.out.println("Age: " + age); } public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter name: "); String name = sn.next(); System.out.print("Enter age: "); int age = sn.nextInt(); _01_ p = new _01_(name, age); p.displayInfo(); sn.close(); } }

Sample Output

Enter name: Alice Enter age: 25 Name: Alice Age: 25
Key Concepts:
  • Parameterized Constructor: Accepts arguments to initialize attributes
  • Object Initialization: Values set at the time of object creation
  • Constructor Name: Same as class name, no return type
  • Encapsulation: Private fields accessed via getters
Benefit: Ensures objects are created with valid initial state

Question 2: Car Class - Default & Parameterized Constructors

Problem Statement

Create a Car class with the following attributes and methods: • Private attributes make, model, and year • A default constructor that initializes the attributes to default values • A parameterized constructor that initializes make, model, and year • Public methods getMake, getModel, and getYear to return the make, model, and year • A method displayCarInfo to print the car details

Solution

import java.util.Scanner; class _02_ { private String make; private String model; private int year; _02_() { make = "Unknown"; model = "Unknown"; year = 0; } _02_(String m, String mo, int y) { make = m; model = mo; year = y; } public String getMake() { return make; } public String getModel() { return model; } public int getYear() { return year; } void displayCarInfo() { System.out.println("Make: " + make); System.out.println("Model: " + model); System.out.println("Year: " + year); } public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter make: "); String make = sn.next(); System.out.print("Enter model: "); String model = sn.next(); System.out.print("Enter year: "); int year = sn.nextInt(); _02_ car = new _02_(make, model, year); car.displayCarInfo(); sn.close(); } }

Sample Output

Enter make: Toyota Enter model: Camry Enter year: 2023 Make: Toyota Model: Camry Year: 2023
Key Concepts:
  • Default Constructor: No parameters, sets default values
  • Constructor Overloading: Multiple constructors with different parameters
  • Flexibility: Create objects with or without initial values
  • Default Values: "Unknown" for strings, 0 for numbers
Use Case: Provides flexibility in object creation

Question 3: BankAccount Class - Overloaded Constructors

Problem Statement

Create a BankAccount class with the following attributes and methods: • Private attributes accountNumber and balance • A parameterized constructor that initializes accountNumber and balance • An overloaded constructor that initializes only accountNumber with a default balance of 0.0 • Public methods getAccountNumber and getBalance to return the accountNumber and balance • A method displayAccountInfo to print the account details

Solution

import java.util.Scanner; class _03_ { private String accountNumber; private double balance; _03_(String acc, double bal) { accountNumber = acc; balance = bal; } _03_(String acc) { accountNumber = acc; balance = 0.0; } public String getAccountNumber() { return accountNumber; } public double getBalance() { return balance; } void displayAccountInfo() { System.out.println("Account Number: " + accountNumber); System.out.println("Balance: " + balance); } public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter account number: "); String acc = sn.next(); System.out.print("Enter balance: "); double bal = sn.nextDouble(); _03_ bank = new _03_(acc, bal); bank.displayAccountInfo(); sn.close(); } }

Sample Output

Enter account number: ACC1001 Enter balance: 5000 Account Number: ACC1001 Balance: 5000.0
Key Concepts:
  • Constructor Overloading: Two constructors with different parameter counts
  • Default Balance: Second constructor sets balance to 0.0
  • Convenience: Create account with or without initial deposit
  • Parameter Differentiation: Constructors differ by parameter count
Real-World: New accounts can start with zero or initial balance

Question 4: Rectangle Class - Overloaded Constructors

Problem Statement

Create a Rectangle class with the following attributes and methods: • Private attributes length and width • A parameterized constructor that initializes length and width • An overloaded constructor that initializes length and width to 1.0 • Public methods getLength, getWidth, getArea, and getPerimeter to return the length, width, area, and perimeter • A method displayRectangleInfo to print the rectangle details

Solution

import java.util.Scanner; class _04_ { private double length; private double width; _04_(double l, double w) { length = l; width = w; } _04_() { length = 1.0; width = 1.0; } public double getLength() { return length; } public double getWidth() { return width; } public double getArea() { return length * width; } public double getPerimeter() { return 2 * (length + width); } void displayRectangleInfo() { System.out.println("Length: " + length); System.out.println("Width: " + width); System.out.println("Area: " + getArea()); System.out.println("Perimeter: " + getPerimeter()); } public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter length: "); double l = sn.nextDouble(); System.out.print("Enter width: "); double w = sn.nextDouble(); _04_ r = new _04_(l, w); r.displayRectangleInfo(); sn.close(); } }

Sample Output

Enter length: 5 Enter width: 3 Length: 5.0 Width: 3.0 Area: 15.0 Perimeter: 16.0
Key Concepts:
  • Default Constructor: Creates unit rectangle (1.0 x 1.0)
  • Parameterized Constructor: Creates rectangle with custom dimensions
  • Calculated Properties: Area and perimeter computed from length/width
  • Method Reuse: displayRectangleInfo() calls getArea() and getPerimeter()
Pattern: Default constructor provides sensible default values

Question 5: Book Class - Multiple Overloaded Constructors

Problem Statement

Create a Book class with the following attributes and methods: • Private attributes title, author, and price • A parameterized constructor that initializes title, author, and price • An overloaded constructor that initializes title and author with a default price of 0.0 • An overloaded constructor that initializes title with default values for author ("Unknown") and price (0.0) • Public methods getTitle, getAuthor, and getPrice to return the title, author, and price • A method displayBookInfo to print the book details

Solution

import java.util.Scanner; class _05_ { private String title; private String author; private double price; _05_(String t, String a, double p) { title = t; author = a; price = p; } _05_(String t, String a) { title = t; author = a; price = 0.0; } _05_(String t) { title = t; author = "Unknown"; price = 0.0; } public String getTitle() { return title; } public String getAuthor() { return author; } public double getPrice() { return price; } void displayBookInfo() { System.out.println("Title: " + title); System.out.println("Author: " + author); System.out.println("Price: " + price); } public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter title: "); String title = sn.next(); System.out.print("Enter author: "); String author = sn.next(); System.out.print("Enter price: "); double price = sn.nextDouble(); _05_ book = new _05_(title, author, price); book.displayBookInfo(); sn.close(); } }

Sample Output

Enter title: Java Enter author: Gosling Enter price: 500 Title: Java Author: Gosling Price: 500.0
Key Concepts:
  • Three Overloaded Constructors: Different parameter combinations
  • Progressive Defaults: Each constructor provides more default values
  • Maximum Flexibility: Create books with full, partial, or minimal info
  • Constructor Chaining: Could use this() to call other constructors
Design: Supports various levels of information availability
