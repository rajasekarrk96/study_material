Question 1: Menu System

Problem Statement

Create a menu that displays at least once and keeps showing until user enters 0 to exit.

Sample Input and Output

Sample Input 1:
1
2
3
0

Sample Output 1:
Menu: 1-Option1 2-Option2 3-Option3 0-Exit
You selected 1
Menu: 1-Option1 2-Option2 3-Option3 0-Exit
You selected 2
...
Goodbye!
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class MenuSystem { public static void main(String[] args) { Scanner sn = new Scanner(System.in); int choice; do { System.out.println("Menu: 1-Option1 2-Option2 3-Option3 0-Exit"); System.out.print("Enter choice: "); choice = sn.nextInt(); if (choice != 0) { System.out.println("You selected " + choice); } } while (choice != 0); System.out.println("Goodbye!"); sn.close(); } }

Step-by-Step Explanation

  1. Do-while guarantees menu shows at least once: Unlike while loops, do-while checks condition after execution. Perfect for menus that must display before getting user input.
  2. Exit-controlled loop pattern: User must see options before deciding to exit. This is why do-while is the natural choice for interactive menus and input validation.
  3. Condition checked at the end: After displaying menu and getting choice, we check if it's 0. If not, loop repeats. This "do first, check later" is do-while's defining characteristic.
  4. Real-world use case: ATMs, game menus, admin panels all use this pattern. Master it for user-interactive programs that need at least one iteration guaranteed.

Question 2: Password Validation

Problem Statement

Keep asking for password until user enters correct password "admin123".

Sample Input and Output

Sample Input 1:
wrong
alsowrong
admin123

Sample Output 1:
Enter password:
Wrong! Try again
Enter password:
Wrong! Try again
Enter password:
Access granted!
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class PasswordCheck { public static void main(String[] args) { Scanner sn = new Scanner(System.in); String password; do { System.out.print("Enter password: "); password = sn.nextLine(); if (!password.equals("admin123")) { System.out.println("Wrong! Try again"); } } while (!password.equals("admin123")); System.out.println("Access granted!"); sn.close(); } }

Step-by-Step Explanation

  1. Input validation needs do-while: You must get input first before validating it. Do-while naturally fits this pattern - get input, then check if valid.
  2. Negation in condition: Loop continues while password is NOT correct (!password.equals()). When password is finally correct, negation makes condition false and loop exits.
  3. User feedback in loop: Printing "Wrong! Try again" inside the loop (but not on success) guides users. Good UX requires feedback at each step.
  4. String comparison caution: Always use .equals() for strings, never ==. This is a common beginner mistake that causes bugs even when strings look identical.

Question 3: Countdown Timer

Problem Statement

Create a countdown from N to 1, then print "Blast off!" using do-while.

Sample Input and Output

Sample Input 1:
5

Sample Output 1:
5
4
3
2
1
Blast off!
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class Countdown { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter countdown start: "); int n = sn.nextInt(); do { System.out.println(n); n--; } while (n > 0); System.out.println("Blast off!"); sn.close(); } }

Step-by-Step Explanation

  1. Do-while ensures at least one number prints: Even if user enters 0 or negative (poor input), the first number still prints before condition check. While loop wouldn't print anything.
  2. Decrement inside loop: n-- happens after printing, so we print current value then decrease. Order matters - print-then-decrement vs decrement-then-print gives different results.
  3. Condition becomes false at 0: When n reaches 0, condition (n > 0) is false, loop exits. Understanding when exactly loops stop prevents off-by-one errors.
  4. Countdown pattern common in games: Timer countdowns, loading screens, game loops often use this structure. It's a fundamental pattern in interactive applications.

Question 4: Sum Until Negative

Problem Statement

Keep taking numbers from user and add to sum. Stop when user enters a negative number. Print final sum.

Sample Input and Output

Sample Input 1:
10
20
30
-1

Sample Output 1:
Total sum: 60
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class SumUntilNegative { public static void main(String[] args) { Scanner sn = new Scanner(System.in); int sum = 0; int num; do { System.out.print("Enter number (-ve to stop): "); num = sn.nextInt(); if (num >= 0) { sum += num; } } while (num >= 0); System.out.println("Total sum: " + sum); sn.close(); } }

Step-by-Step Explanation

  1. Sentinel value technique: Using -1 or any negative as a "stop signal" is called sentinel value. Common pattern when you don't know how many inputs you'll get beforehand.
  2. Check before adding: The if (num >= 0) prevents adding the sentinel value to sum. Always validate data before using it in calculations.
  3. Why do-while here?: We must get at least one input to check if it's the sentinel. Do-while's "input first, check later" pattern fits perfectly.
  4. Accumulator pattern: sum starts at 0 and grows with each valid input. This incremental building of results is fundamental to many algorithms.

Question 5: Guessing Game

Problem Statement

Create a number guessing game. Secret number is 7. Keep asking until user guesses correctly.

Sample Input and Output

Sample Input 1:
5
10
7

Sample Output 1:
Guess the number: Too low!
Guess the number: Too high!
Guess the number: Correct! You win!
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class GuessingGame { public static void main(String[] args) { Scanner sn = new Scanner(System.in); int secret = 7; int guess; do { System.out.print("Guess the number: "); guess = sn.nextInt(); if (guess < secret) { System.out.println("Too low!"); } else if (guess > secret) { System.out.println("Too high!"); } } while (guess != secret); System.out.println("Correct! You win!"); sn.close(); } }

Step-by-Step Explanation

  1. Game loop pattern: Do-while is perfect for games - player must make at least one move, then we check win condition. This loop-until-win pattern is core to game development.
  2. Feedback drives gameplay: "Too low" and "Too high" hints make the game playable. Without feedback, it would be frustrating. This shows programming isn't just logic - UX matters.
  3. Exit condition is success state: Loop runs while guess != secret. When player guesses correctly, condition fails and we congratulate them. Designing when loops exit is as important as what they do.
  4. Extension possibilities: Could add attempt counter, random secret number, difficulty levels. Understanding basic structure lets you enhance features systematically.
