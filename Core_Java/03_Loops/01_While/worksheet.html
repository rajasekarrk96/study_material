Question 1: Print Numbers from Start to End

Problem Statement

Write a Java program that takes two numbers (start and end) and prints all numbers from start to end using a while loop.

Sample Input and Output

Sample Input 1:
Enter start number: 5
Enter end number: 10

Sample Output 1:
5
6
7
8
9
10
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class PrintNumbers { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter start number: "); int start = sn.nextInt(); System.out.print("Enter end number: "); int end = sn.nextInt(); while (start <= end) { System.out.println(start); start++; } sn.close(); } }

Step-by-Step Explanation

  1. While loop checks condition before each iteration: The condition (start <= end) is evaluated before entering the loop body. If false initially, the loop never executes.
  2. Loop variable must be updated: The start++ increments the counter each iteration. Without this, you'd have an infinite loop since start would never exceed end.
  3. Simple counting pattern: This is the most basic while loop use case - counting from one number to another. Master this pattern as it forms the foundation for more complex loops.
  4. Comparison with for loop: This same task could be done with a for loop, but while loops are better when the number of iterations isn't known in advance or when the update logic is more complex.

Question 2: Sum of Digits

Problem Statement

Write a program that calculates the sum of all digits in a number using a while loop.

Sample Input and Output

Sample Input 1:
1234

Sample Output 1:
Sum of digits: 10

Sample Input 2:
567

Sample Output 2:
Sum of digits: 18
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class SumDigits { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a number: "); int num = sn.nextInt(); int sum = 0; while (num > 0) { sum += num % 10; // Add last digit num /= 10; // Remove last digit } System.out.println("Sum of digits: " + sum); sn.close(); } }

Step-by-Step Explanation

  1. Extracting digits with modulo: num % 10 gets the rightmost digit. For 1234, it gives 4. This is a crucial technique for digit manipulation.
  2. Removing digits with division: num /= 10 removes the last digit. 1234 becomes 123. Integer division automatically truncates the decimal.
  3. Loop continues until num becomes 0: Each iteration processes one digit. When all digits are processed, num becomes 0 and the loop stops.
  4. Pattern for digit processing: This extract-and-remove pattern (% and /) is fundamental for many number manipulation problems like reversing digits, counting digits, or checking palindromes.

Question 3: Factorial Calculator

Problem Statement

Calculate the factorial of a number using a while loop. Factorial of n (n!) = n × (n-1) × (n-2) × ... × 1.

Sample Input and Output

Sample Input 1:
5

Sample Output 1:
Factorial of 5 = 120

Sample Input 2:
4

Sample Output 2:
Factorial of 4 = 24
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class Factorial { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a number: "); int n = sn.nextInt(); int factorial = 1; int i = n; while (i > 0) { factorial *= i; i--; } System.out.println("Factorial of " + n + " = " + factorial); sn.close(); } }

Step-by-Step Explanation

  1. Initialize result to 1, not 0: Factorial uses multiplication, so starting with 0 would make everything 0. Always initialize multiplicative accumulators to 1.
  2. Countdown approach: Starting from n and counting down to 1 is natural for factorial. Could also count up from 1 to n - both work equally well.
  3. Handle edge case of 0: This code works for 0! which equals 1 (the loop never runs, factorial stays 1). Understanding edge cases prevents bugs.
  4. Overflow consideration: Factorials grow extremely fast. 13! exceeds int range. For large numbers, use long or BigInteger. This demonstrates why understanding data types matters.

Question 4: Reverse a Number

Problem Statement

Write a program to reverse the digits of a number using a while loop.

Sample Input and Output

Sample Input 1:
1234

Sample Output 1:
Reversed: 4321

Sample Input 2:
5060

Sample Output 2:
Reversed: 605
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class ReverseNumber { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a number: "); int num = sn.nextInt(); int reversed = 0; while (num > 0) { reversed = reversed * 10 + num % 10; num /= 10; } System.out.println("Reversed: " + reversed); sn.close(); } }

Step-by-Step Explanation

  1. Build reversed number digit by digit: Each iteration extracts the last digit of num and appends it to reversed. This gradually builds the number in reverse order.
  2. The × 10 operation shifts digits left: Multiplying by 10 makes room for the new digit. 12 × 10 = 120, then adding 3 gives 123. This is how we construct multi-digit numbers.
  3. Trailing zeros are lost: Notice 5060 becomes 605, not 0605. When building integers, leading zeros don't exist. If you need to preserve them, use strings instead.
  4. Common algorithm pattern: This extract-transform-build pattern appears in many problems: reversing, digit manipulation, number base conversion. Master it for algorithmic thinking.

Question 5: Fibonacci Sequence

Problem Statement

Print the first N numbers of the Fibonacci sequence using a while loop. Fibonacci: 0, 1, 1, 2, 3, 5, 8...

Sample Input and Output

Sample Input 1:
7

Sample Output 1:
0 1 1 2 3 5 8

Sample Input 2:
5

Sample Output 2:
0 1 1 2 3
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class Fibonacci { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter N: "); int n = sn.nextInt(); int first = 0, second = 1, count = 0; while (count < n) { System.out.print(first + " "); int next = first + second; first = second; second = next; count++; } sn.close(); } }

Step-by-Step Explanation

  1. Track two previous numbers: Fibonacci requires the two preceding numbers to calculate the next. Variables first and second maintain this sliding window of values.
  2. Update pattern crucial: Store next in temp variable, then shift: first becomes second, second becomes next. Direct assignment like "first = second; second = first + second" would use new first value incorrectly.
  3. Counter-based termination: Using count < n is clearer than trying to make first or second the loop condition. Sometimes an explicit counter is the simplest solution.
  4. Famous sequence in programming: Fibonacci appears everywhere - recursion examples, dynamic programming, algorithm complexity analysis. Understanding it builds foundational CS knowledge.
