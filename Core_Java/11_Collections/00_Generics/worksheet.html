Question 1: Generic Method to Sort a Collection

Problem Statement

Write a Java program that demonstrates the use of generics to implement a generic method that sorts a collection of elements.

Solution

import java.util.*; class _01_ { static <T extends Comparable<T>> void sortList(List<T> list) { Collections.sort(list); } public static void main(String[] args) { List<Integer> list = new ArrayList<>(); list.add(5); list.add(2); list.add(8); list.add(1); sortList(list); System.out.println(list); } }

Sample Output

[1, 2, 5, 8]
Key Concepts:
  • Bounded Type Parameter: <T extends Comparable<T>> ensures T can be compared
  • Generic Method: Works with any type that implements Comparable
  • Collections.sort(): Sorts list in natural order
  • Type Safety: Compile-time checking prevents non-comparable types
Why Comparable? To sort elements, they must be comparable to each other

Question 2: Generic Linked List Class

Problem Statement

Write a Java program that demonstrates the use of generics to implement a generic class that represents a linked list.

Solution

class _02_<T> { class Node { T data; Node next; Node(T d) { data = d; } } Node head; void add(T data) { Node n = new Node(data); if (head == null) head = n; else { Node temp = head; while (temp.next != null) temp = temp.next; temp.next = n; } } void display() { Node temp = head; while (temp != null) { System.out.print(temp.data + " "); temp = temp.next; } } public static void main(String[] args) { _02_<String> list = new _02_<>(); list.add("A"); list.add("B"); list.add("C"); list.display(); } }

Sample Output

A B C
Key Concepts:
  • Generic Class: _02_<T> can store any type
  • Inner Node Class: Uses outer class's type parameter T
  • Type Reusability: Same code works for String, Integer, etc.
  • Linked List: Dynamic data structure with nodes
Advantages: No casting needed, type-safe operations, code reuse

Question 3: Generic Method to Compute Sum of Numbers

Problem Statement

Write a Java program that demonstrates the use of generics to implement a generic method that computes the sum of a collection of numbers.

Solution

import java.util.*; class _03_ { static <T extends Number> double sum(Collection<T> c) { double s = 0; for (T n : c) s += n.doubleValue(); return s; } public static void main(String[] args) { List<Integer> list = Arrays.asList(1, 2, 3, 4); System.out.println("Sum = " + sum(list)); } }

Sample Output

Sum = 10.0
Key Concepts:
  • Bounded Type: <T extends Number> restricts to numeric types
  • Number Class: Parent of Integer, Double, Float, etc.
  • doubleValue(): Converts any Number to double
  • Collection Interface: Works with List, Set, etc.
Works with: Integer, Double, Float, Long, Short, Byte

Question 4: Generic Method to Merge Two Collections

Problem Statement

Write a Java program that demonstrates the use of generics to implement a generic method that merges two collections of elements.

Solution

import java.util.*; class _04_ { static <T> List<T> merge(Collection<T> c1, Collection<T> c2) { List<T> result = new ArrayList<>(); result.addAll(c1); result.addAll(c2); return result; } public static void main(String[] args) { List<String> a = Arrays.asList("A", "B"); List<String> b = Arrays.asList("C", "D"); System.out.println(merge(a, b)); } }

Sample Output

[A, B, C, D]
Key Concepts:
  • Unbounded Type Parameter: <T> accepts any type
  • addAll(): Adds all elements from one collection to another
  • Type Consistency: Both collections must have same type
  • Return Type: Returns List<T> containing merged elements
Note: Creates new list, original collections unchanged

Question 5: Generic Method to Find Common Elements

Problem Statement

Write a Java program that demonstrates the use of generics to implement a generic method that finds the common elements between two collections.

Solution

import java.util.*; class _05_ { static <T> Set<T> common(Collection<T> c1, Collection<T> c2) { Set<T> set = new HashSet<>(c1); set.retainAll(c2); return set; } public static void main(String[] args) { List<Integer> a = Arrays.asList(1, 2, 3, 4); List<Integer> b = Arrays.asList(3, 4, 5); System.out.println(common(a, b)); } }

Sample Output

[3, 4]
Key Concepts:
  • Set Interface: Automatically removes duplicates
  • retainAll(): Keeps only elements present in both collections
  • HashSet Constructor: Creates set from collection
  • Intersection Operation: Mathematical set intersection
Use Cases: Finding common items, set operations, data comparison
