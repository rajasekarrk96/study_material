Question 1: Banking Application with User Authentication

Problem Statement

Develop a banking application with multi-user support using JDBC. This example demonstrates user authentication by verifying username and password against a MySQL database. The application includes features such as:
  • User authentication (login system)
  • Database connectivity using JDBC
  • PreparedStatement for SQL injection prevention
  • Proper exception handling
Database Setup: Create a database named bankdb with a users table containing columns: username and password.

Solution

import java.sql.*; class _01_ { public static void main(String[] args) { try { Class.forName("com.mysql.cj.jdbc.Driver"); Connection con = DriverManager.getConnection( "jdbc:mysql://localhost:3306/bankdb", "root", "password"); PreparedStatement ps = con.prepareStatement("SELECT * FROM users WHERE username=? AND password=?"); ps.setString(1, "admin"); ps.setString(2, "admin123"); ResultSet rs = ps.executeQuery(); if (rs.next()) System.out.println("Login Successful"); else System.out.println("Invalid Login"); con.close(); } catch (Exception e) { e.printStackTrace(); } } }

Sample Output

Login Successful -- OR -- Invalid Login

Database Schema

CREATE DATABASE bankdb; USE bankdb; CREATE TABLE users ( username VARCHAR(50) PRIMARY KEY, password VARCHAR(50) NOT NULL ); INSERT INTO users VALUES ('admin', 'admin123'); INSERT INTO users VALUES ('user1', 'pass123');
Key Concepts:
  • JDBC Driver Loading: Class.forName() loads the MySQL JDBC driver
  • Connection: DriverManager.getConnection() establishes database connection
  • PreparedStatement: Prevents SQL injection by using parameterized queries
  • ResultSet: rs.next() checks if any matching record exists
  • Resource Management: Always close connections to prevent memory leaks
Security Note: In production, passwords should be hashed (e.g., using BCrypt) before storing in the database.

Question 2: Hospital Management System - Patient Registration

Problem Statement

Create a hospital management system using JDBC with patient registration functionality. This application demonstrates:
  • Inserting patient records into a database
  • Using PreparedStatement for data insertion
  • Managing patient information (ID, name, disease)
  • Proper database connection handling
Database Setup: Create a database named hospitaldb with a patient table containing columns: id, name, and disease.

Solution

import java.sql.*; class _02_ { public static void main(String[] args) { try { Connection con = DriverManager.getConnection( "jdbc:mysql://localhost:3306/hospitaldb", "root", "password"); PreparedStatement ps = con.prepareStatement("INSERT INTO patient VALUES (?, ?, ?)"); ps.setInt(1, 101); ps.setString(2, "Ravi"); ps.setString(3, "Fever"); ps.executeUpdate(); System.out.println("Patient Registered"); con.close(); } catch (Exception e) { e.printStackTrace(); } } }

Sample Output

Patient Registered

Database Schema

CREATE DATABASE hospitaldb; USE hospitaldb; CREATE TABLE patient ( id INT PRIMARY KEY, name VARCHAR(100) NOT NULL, disease VARCHAR(100) ); -- After running the program, verify: SELECT * FROM patient; -- Output: 101 | Ravi | Fever
Key Concepts:
  • INSERT Operation: INSERT INTO patient VALUES (?, ?, ?) adds new records
  • PreparedStatement Methods:
    • setInt() - Sets integer parameter
    • setString() - Sets string parameter
  • executeUpdate(): Used for INSERT, UPDATE, DELETE operations (returns affected row count)
  • No Driver Loading: Modern JDBC drivers auto-register (JDBC 4.0+)
Extension Ideas: Add features like doctor scheduling, appointment booking, medical record management, and billing.
