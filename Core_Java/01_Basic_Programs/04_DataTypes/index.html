Primitive Data Types

Primitive data types are the fundamental data types built into Java. They represent single values and are stored directly in memory. Java has 8 primitive types divided into four categories.

Integer Types:

Type Size Range Example
byte 1 byte (8 bits) -128 to 127 byte age = 25;
short 2 bytes (16 bits) -32,768 to 32,767 short year = 2025;
int 4 bytes (32 bits) -2,147,483,648 to 2,147,483,647 int count = 1000;
long 8 bytes (64 bits) -9.2 quintillion to 9.2 quintillion long distance = 1000000L;

Floating-point Types:

Type Size Precision Example
float 4 bytes ~6-7 decimal digits float price = 19.99f;
double 8 bytes ~15 decimal digits double pi = 3.14159;

Character and Boolean Types:

Type Size Range Example
char 2 bytes (16 bits) 0 to 65,535 (Unicode) char grade = 'A';
boolean 1 bit (1 or 0) true or false boolean isActive = true;

Complete Example:

public class PrimitiveDataTypes {
    public static void main(String[] args) {
        // Integer types
        byte b = 100;
        short s = 5000;
        int i = 100000;
        long l = 15000000000L;
        
        // Floating-point types
        float f = 3.14f;
        double d = 3.14159265;
        
        // Character and Boolean
        char c = 'J';
        boolean flag = true;
        
        System.out.println("Byte: " + b);
        System.out.println("Short: " + s);
        System.out.println("Int: " + i);
        System.out.println("Long: " + l);
        System.out.println("Float: " + f);
        System.out.println("Double: " + d);
        System.out.println("Char: " + c);
        System.out.println("Boolean: " + flag);
    }
}
Byte: 100
Short: 5000
Int: 100000
Long: 15000000000
Float: 3.14
Double: 3.14159265
Char: J
Boolean: true
💡 Key Points: Use int by default for integers, double for decimals. Add 'L' suffix for long values and 'f' suffix for float values. Boolean values are not integers in Java (unlike C).

Reference Data Types

Reference types store references (memory addresses) to objects, not the actual values. They include classes, interfaces, arrays, and enums. The most common reference type is String.

Reference Types vs Primitive Types:

  • Primitive types store actual values; reference types store memory addresses
  • Primitive types have fixed sizes; reference types size depends on object
  • Reference types can be null; primitives cannot (without wrapper classes)
  • Reference types require instantiation with new keyword

String Example:

public class ReferenceTypes {
    public static void main(String[] args) {
        // Reference type - String
        String name = "Java";
        String message = new String("Hello, World!");
        
        // Array (reference type)
        int[] numbers = new int[5];
        String[] fruits = {"Apple", "Banana", "Orange"};
        
        // Custom class reference
        Student student = new Student("John", 101);
        
        System.out.println("Name: " + name);
        System.out.println("Message: " + message);
        System.out.println("Numbers array length: " + numbers.length);
        System.out.println("Fruits: " + java.util.Arrays.toString(fruits));
    }
}

class Student {
    String name;
    int id;
    
    Student(String n, int i) {
        name = n;
        id = i;
    }
}
Name: Java
Message: Hello, World!
Numbers array length: 5
Fruits: [Apple, Banana, Orange]

Type Conversion and Casting

Type conversion is the process of converting one data type to another. Java supports both implicit (automatic) and explicit (manual) conversions.

Widening Conversion (Automatic):

Converting a smaller type to a larger type automatically. No data loss occurs.

public class WideningConversion {
    public static void main(String[] args) {
        byte b = 10;
        short s = b;        // byte to short
        int i = s;          // short to int
        long l = i;         // int to long
        float f = l;        // long to float
        double d = f;       // float to double
        
        System.out.println("Byte: " + b);
        System.out.println("Short: " + s);
        System.out.println("Int: " + i);
        System.out.println("Long: " + l);
        System.out.println("Float: " + f);
        System.out.println("Double: " + d);
    }
}
Byte: 10
Short: 10
Int: 10
Long: 10
Float: 10.0
Double: 10.0

Narrowing Conversion (Manual/Casting):

Converting a larger type to a smaller type requires explicit casting. Data loss may occur.

public class NarrowingConversion {
    public static void main(String[] args) {
        double d = 9.78;
        float f = (float) d;        // double to float
        long l = (long) f;          // float to long
        int i = (int) l;            // long to int
        short s = (short) i;        // int to short
        byte b = (byte) s;          // short to byte
        
        System.out.println("Original double: " + d);
        System.out.println("After casting to byte: " + b);
    }
}
Original double: 9.78
After casting to byte: 9

String to Number Conversion:

String str = "123";
int num = Integer.parseInt(str);        // String to int
double d = Double.parseDouble("3.14");  // String to double

System.out.println("Number: " + (num + 10));
System.out.println("Double: " + (d + 0.86));
Number: 133
Double: 4.0

Memory Allocation

Understanding how Java allocates memory for different data types helps write efficient code.

  • Stack: Stores primitive values and references. Memory is automatically freed when variables go out of scope.
  • Heap: Stores objects and arrays. Memory is managed by garbage collector.
public class MemoryAllocation {
    public static void main(String[] args) {
        // Primitive type - value stored on stack
        int age = 25;
        
        // Reference type - reference on stack, object on heap
        String name = "John";
        
        // Array - reference on stack, array object on heap
        int[] numbers = new int[]{1, 2, 3, 4, 5};
        
        // Multiple references to same object
        String str1 = new String("Java");
        String str2 = str1;  // Both reference same object
        
        System.out.println("age: " + age);
        System.out.println("name: " + name);
        System.out.println("numbers[0]: " + numbers[0]);
        System.out.println("str1.equals(str2): " + str1.equals(str2));
    }
}
age: 25
name: John
numbers[0]: 1
str1.equals(str2): true
💡 Memory Best Practice: Use primitive types when possible (better memory efficiency). Be aware of null pointer exceptions with reference types. Understand when objects are eligible for garbage collection.

ASCII Notes

ASCII is directly related to the char data type. Each char value is actually an ASCII code stored as a numeric value.

ASCII as a Data Type Value:

char dataType stores ASCII codes (0-127 and beyond)
┌─────────────────────────────────────────────────────────────┐ │ Variable: ch (char) │ │ Memory: 2 bytes (16 bits) in Java │ │ Value: 'A' (displays as letter A) │ │ Actual: 65 (ASCII code in memory) │ │ Binary: 0000000001000001 │ └─────────────────────────────────────────────────────────────┘

ASCII and Char Data Type Relationship:

public class ASCIIDataType {
    public static void main(String[] args) {
        // char stores ASCII values
        char a = 65;              // Displays as 'A'
        char b = 'B';             // ASCII 66
        
        // char and int relationship
        char digit = '5';         // ASCII 53
        int digitValue = digit;   // Auto-widening to int
        
        System.out.println("Character 'A' stores: " + (int)a);
        System.out.println("Character 'B' stores: " + (int)b);
        System.out.println("Character '5' stores: " + digitValue);
        
        // Arithmetic with ASCII values
        char start = 'A';
        System.out.println("\nNext character after 'A': " + (char)(start + 1));
        System.out.println("Previous character before 'A': " + (char)(start - 1));
    }
}
Character 'A' stores: 65
Character 'B' stores: 66
Character '5' stores: 53

Next character after 'A': B
Previous character before 'A': @
✓ Data Type Insight: The char data type is essentially storing numeric ASCII values. When you use 'A', Java internally stores 65. The character is just a visual representation.
