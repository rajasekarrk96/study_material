Question 1: Display All Arguments

Problem Statement

Write a Java program that displays all command line arguments passed to it, along with the total count.

Sample Input and Output

Sample Input 1:
java DisplayArgs Hello World Java

Sample Output 1:
Total arguments: 3
Argument 0: Hello
Argument 1: World
Argument 2: Java

Sample Input 2:
java DisplayArgs Programming

Sample Output 2:
Total arguments: 1
Argument 0: Programming
Show Solution

Solution (Java Program)

public class DisplayArgs { public static void main(String[] args) { System.out.println("Total arguments: " + args.length); for (int i = 0; i < args.length; i++) { System.out.println("Argument " + i + ": " + args[i]); } } }

Step-by-Step Explanation

  1. args[] is an array of strings from the command line: When you run "java DisplayArgs Hello World", Java automatically creates an array where args[0]="Hello", args[1]="World". This allows programs to accept input without user interaction.
  2. Array indexing starts at 0, not 1: The first argument is args[0], not args[1]. This is a fundamental concept in Java - all arrays use zero-based indexing. The program name itself is not included in args.
  3. args.length tells us how many arguments were provided: This is crucial for validation - we can check if the user provided the expected number of arguments before processing them, preventing array index errors.
  4. Command line args enable automation: Unlike Scanner input which requires user interaction, command line arguments allow scripts and other programs to run your Java program automatically with different inputs.

Question 2: Sum of Numbers from Arguments

Problem Statement

Write a Java program that takes numbers as command line arguments and calculates their sum.

Sample Input and Output

Sample Input 1:
java SumArgs 10 20 30

Sample Output 1:
Sum: 60

Sample Input 2:
java SumArgs 5 15 25 35

Sample Output 2:
Sum: 80
Show Solution

Solution (Java Program)

public class SumArgs { public static void main(String[] args) { int sum = 0; for (String arg : args) { sum += Integer.parseInt(arg); } System.out.println("Sum: " + sum); } }

Step-by-Step Explanation

  1. Command line arguments are always strings: Even though we type "10 20 30", Java receives them as the strings "10", "20", "30". We must explicitly convert them to numbers to perform arithmetic.
  2. Integer.parseInt() converts string to int: This method parses the string representation of a number and returns the actual integer value. Without this conversion, "10" + "20" would give "1020" (string concatenation), not 30.
  3. Enhanced for loop simplifies iteration: The syntax for (String arg : args) automatically loops through each element without needing an index variable. This is cleaner and less error-prone than traditional for loops.
  4. Error handling is important in real applications: This code assumes all arguments are valid numbers. In production code, we'd wrap parseInt() in try-catch to handle invalid input gracefully.

Question 3: Argument Validation

Problem Statement

Write a Java program that validates the number of command line arguments and displays an error message if incorrect.

Sample Input and Output

Sample Input 1:
java ValidateArgs John 25

Sample Output 1:
Name: John
Age: 25

Sample Input 2:
java ValidateArgs John

Sample Output 2:
Error: Please provide exactly 2 arguments (name and age)
Show Solution

Solution (Java Program)

public class ValidateArgs { public static void main(String[] args) { if (args.length != 2) { System.out.println("Error: Please provide exactly 2 arguments (name and age)"); return; } String name = args[0]; int age = Integer.parseInt(args[1]); System.out.println("Name: " + name); System.out.println("Age: " + age); } }

Step-by-Step Explanation

  1. Validation prevents crashes and confusion: Checking args.length before accessing elements prevents ArrayIndexOutOfBoundsException. If we tried args[1] when only one argument was provided, the program would crash.
  2. Early return stops execution gracefully: The return statement exits the main method immediately after showing the error. This prevents the rest of the code from running with invalid data.
  3. Clear error messages guide users: Instead of a cryptic exception, we tell users exactly what went wrong and how to fix it. Good error messages are crucial for usability.
  4. This pattern is standard in CLI tools: Professional command-line programs always validate input first. They often show usage instructions when arguments are missing or incorrect, helping users understand how to use the program correctly.

Question 4: Simple Calculator with Arguments

Problem Statement

Write a Java program that performs arithmetic operations based on command line arguments: number1 operator number2.

Sample Input and Output

Sample Input 1:
java Calculator 15 + 5

Sample Output 1:
Result: 20

Sample Input 2:
java Calculator 20 * 3

Sample Output 2:
Result: 60
Show Solution

Solution (Java Program)

public class Calculator { public static void main(String[] args) { if (args.length != 3) { System.out.println("Usage: java Calculator "); return; } double num1 = Double.parseDouble(args[0]); String operator = args[1]; double num2 = Double.parseDouble(args[2]); double result = 0; switch (operator) { case "+": result = num1 + num2; break; case "-": result = num1 - num2; break; case "*": result = num1 * num2; break; case "/": result = num1 / num2; break; default: System.out.println("Invalid operator. Use +, -, *, or /"); return; } System.out.println("Result: " + result); } }

Step-by-Step Explanation

  1. Command line calculators demonstrate practical CLI usage: This pattern is common in Unix/Linux tools - accept input as arguments, process it, output results. It's perfect for scripting and automation where interactive input isn't feasible.
  2. Switch statement handles multiple operation choices: Rather than nested if-else statements, switch provides a clean way to handle different operators. Each case represents one operation, making the code organized and readable.
  3. Using doubles instead of ints prevents precision loss: Division like 15/4 would give 3 with integers, but 3.75 with doubles. By using Double.parseDouble(), we maintain decimal precision for all operations.
  4. Default case catches invalid operators: If someone types "java Calculator 5 % 2", the default case catches this invalid operator and provides feedback. This defensive programming prevents unexpected behavior.

Question 5: Processing Variable Number of Arguments

Problem Statement

Write a Java program that finds the maximum value among any number of command line arguments.

Sample Input and Output

Sample Input 1:
java FindMax 45 23 67 12 89 34

Sample Output 1:
Maximum value: 89

Sample Input 2:
java FindMax 100 50 75

Sample Output 2:
Maximum value: 100
Show Solution

Solution (Java Program)

public class FindMax { public static void main(String[] args) { if (args.length == 0) { System.out.println("Error: Please provide at least one number"); return; } int max = Integer.parseInt(args[0]); for (int i = 1; i < args.length; i++) { int current = Integer.parseInt(args[i]); if (current > max) { max = current; } } System.out.println("Maximum value: " + max); } }

Step-by-Step Explanation

  1. Initialize max with the first element, not zero: Starting with max=0 would fail if all numbers are negative. By using args[0] as the initial value, we guarantee max starts with an actual value from the input.
  2. Loop starts at index 1, not 0: Since we already used args[0] to initialize max, we begin comparing from args[1]. This avoids redundantly comparing the first element with itself.
  3. The algorithm is a linear search for maximum: We examine each number once, updating max whenever we find a larger value. This is efficient (O(n) time complexity) and simple to understand.
  4. Flexibility is the key advantage: Unlike a fixed-parameter function, this program works with any number of arguments - 3, 10, or 100 numbers. This demonstrates the power of arrays and loops for handling variable-length input.
