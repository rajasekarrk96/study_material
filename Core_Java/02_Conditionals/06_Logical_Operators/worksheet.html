Question 1: Login Validation (AND)

Problem Statement

Check login credentials. Both username must be "admin" AND password must be "1234".

Sample Input and Output

Sample Input 1:
admin
1234

Sample Output 1:
Login successful

Sample Input 2:
admin
9999

Sample Output 2:
Login failed
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class LoginValidation { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter username: "); String user = sn.nextLine(); System.out.print("Enter password: "); String pass = sn.nextLine(); if (user.equals("admin") && pass.equals("1234")) { System.out.println("Login successful"); } else { System.out.println("Login failed"); } sn.close(); } }

Step-by-Step Explanation

  1. AND requires both conditions true: If either username OR password is wrong, login fails. The && operator models this "both must be true" logic perfectly, which is fundamental to security and access control.
  2. Short-circuit evaluation optimizes performance: If first condition (username) is false, Java doesn't even check the password. This saves computation and is crucial when conditions involve expensive operations like database queries.
  3. String comparison uses .equals(): For strings, == checks if they're the same object in memory, not if they have the same content. Always use .equals() to compare string values correctly.
  4. Security note: Real applications hash passwords and never store them as plain text. They also limit login attempts and use secure authentication protocols. This demonstrates the logic, not production security.

Question 2: Weekend Check (OR)

Problem Statement

Input a day number (1-7, where 1=Monday...7=Sunday). Print "Weekend!" if it's Saturday (6) OR Sunday (7), otherwise print "Weekday".

Sample Input and Output

Sample Input 1:
6

Sample Output 1:
Weekend!

Sample Input 2:
3

Sample Output 2:
Weekday
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class WeekendCheck { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter day (1-7): "); int day = sn.nextInt(); if (day == 6 || day == 7) { System.out.println("Weekend!"); } else { System.out.println("Weekday"); } sn.close(); } }

Step-by-Step Explanation

  1. OR requires only one condition true: If day is 6 OR 7, it's weekend. Only one needs to match. This "at least one must be true" logic is perfect for checking multiple acceptable values.
  2. Short-circuit with OR too: If first condition (day == 6) is true, Java doesn't check second condition. It already knows the result is true, saving unnecessary evaluation.
  3. Alternative: use || for same-category checks: When checking if a value matches any of several options (like weekend days, vowels, prime numbers under 10), || is cleaner than nested if statements.
  4. Readability consideration: For many values (like checking all weekdays), a switch statement or arrays might be clearer. || works well for 2-3 options; beyond that, consider alternatives.

Question 3: Access Denied (NOT)

Problem Statement

Check if access is denied. If user is NOT an admin (isAdmin = false), print "Access Denied", otherwise print "Access Granted".

Sample Input and Output

Sample Input 1:
false

Sample Output 1:
Access Denied

Sample Input 2:
true

Sample Output 2:
Access Granted
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class AccessControl { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Is admin (true/false): "); boolean isAdmin = sn.nextBoolean(); if (!isAdmin) { System.out.println("Access Denied"); } else { System.out.println("Access Granted"); } sn.close(); } }

Step-by-Step Explanation

  1. NOT operator reverses boolean value: The ! operator flips true to false and false to true. If isAdmin is false, !isAdmin becomes true, executing the denial message.
  2. Improves code readability: "if not admin then deny" reads more naturally than "if admin is false". The ! operator makes conditions match natural language logic.
  3. Avoid double negatives: Writing if (!isNotAdmin) is confusing. Use positive variable names (isAdmin, hasPermission) and apply ! when needed for clarity.
  4. Combining operators: Can combine ! with && and ||. For example: if (!isAdmin && isSensitiveData) means "if not admin AND trying to access sensitive data". Parentheses help clarity in complex expressions.
