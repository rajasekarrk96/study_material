Question 1: Check Positive Number

Problem Statement

Write a Java program that checks if a number is positive. If it is positive, print "Positive number", otherwise print nothing.

Sample Input and Output

Sample Input 1:
15

Sample Output 1:
Positive number

Sample Input 2:
-5

Sample Output 2:
(no output)
Show Solution

Solution (Java Program)

import java.util.Scanner; public class CheckPositive { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a number: "); int num = sn.nextInt(); if (num > 0) { System.out.println("Positive number"); } sn.close(); } }

Step-by-Step Explanation

  1. The if statement creates conditional execution: Code inside the if block only runs when the condition is true. This is fundamental to making programs that respond differently to different inputs.
  2. The condition (num > 0) evaluates to boolean: The comparison operator > returns true or false. If true, the code block executes. If false, execution skips the block entirely and continues after it.
  3. Single if statements are useful for optional actions: When you only want something to happen under certain conditions (like displaying a message only for positive numbers), a standalone if statement is perfect.
  4. Understanding control flow is key: The program reads input, evaluates the condition, decides whether to execute the print statement, then closes the scanner. This decision-making is what makes programs intelligent.

Question 2: Check Voting Eligibility

Problem Statement

Write a Java program that checks if a person is eligible to vote. A person can vote if their age is 18 or above.

Sample Input and Output

Sample Input 1:
20

Sample Output 1:
You are eligible to vote

Sample Input 2:
16

Sample Output 2:
(no output)
Show Solution

Solution (Java Program)

import java.util.Scanner; public class VotingEligibility { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter your age: "); int age = sn.nextInt(); if (age >= 18) { System.out.println("You are eligible to vote"); } sn.close(); } }

Step-by-Step Explanation

  1. The >= operator includes the boundary value: Using >= instead of > is crucial here because 18 is the minimum voting age, and someone who is exactly 18 should be eligible. This attention to boundary conditions prevents bugs.
  2. Real-world rules translate to conditions: The legal requirement "must be 18 or older" directly becomes the condition age >= 18. Programming often involves converting real-world rules into logical expressions.
  3. Silent failure for invalid cases: When age < 18, the program produces no output. In real applications, you'd typically want to inform the user why they don't see a message, which is where else clauses become useful.
  4. Input validation is implicit: This program assumes valid integer input. In production code, you'd add error handling for non-numeric input or negative ages to make the program robust.

Question 3: Check Even Number

Problem Statement

Write a Java program that checks if a number is even. If it is even, print "Even number".

Sample Input and Output

Sample Input 1:
24

Sample Output 1:
Even number

Sample Input 2:
17

Sample Output 2:
(no output)
Show Solution

Solution (Java Program)

import java.util.Scanner; public class CheckEven { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a number: "); int num = sn.nextInt(); if (num % 2 == 0) { System.out.println("Even number"); } sn.close(); } }

Step-by-Step Explanation

  1. Modulo operator reveals divisibility: The expression num % 2 gives the remainder when dividing by 2. Even numbers divide evenly by 2, leaving remainder 0. Odd numbers leave remainder 1.
  2. Mathematical property becomes a condition: The definition of even numbers (divisible by 2) translates perfectly to the condition num % 2 == 0. This shows how mathematical concepts map to programming logic.
  3. Modulo is powerful for patterns: Beyond even/odd checking, the modulo operator is used for cycling through ranges (like array indices), detecting multiples, and extracting digits from numbers.
  4. Zero is considered even: If someone enters 0, the condition 0 % 2 == 0 evaluates to true, correctly identifying 0 as an even number. Understanding edge cases like this prevents bugs.

Question 4: Check Passing Grade

Problem Statement

Write a Java program that checks if a student has passed an exam. The passing grade is 40 or above. If passed, print "Congratulations! You passed".

Sample Input and Output

Sample Input 1:
75

Sample Output 1:
Congratulations! You passed

Sample Input 2:
35

Sample Output 2:
(no output)
Show Solution

Solution (Java Program)

import java.util.Scanner; public class PassingGrade { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter your grade: "); int grade = sn.nextInt(); if (grade >= 40) { System.out.println("Congratulations! You passed"); } sn.close(); } }

Step-by-Step Explanation

  1. Threshold-based decisions are common in real applications: Many systems need to check if values meet minimum requirements - credit scores, temperature limits, inventory levels, etc. The if statement is the fundamental tool for implementing these checks.
  2. The condition defines the success criteria: By setting grade >= 40, we've encoded the passing requirement into the program. Changing this threshold (say, to 50) requires modifying only one number, showing good program design.
  3. Positive feedback motivates users: The congratulatory message makes the program feel more human and engaging. This demonstrates that programming isn't just about logic - user experience matters too.
  4. Consider what happens at the boundary: A grade of exactly 40 triggers the message (because of >=), which is correct. Testing boundary values (39, 40, 41) is crucial for verifying your logic works as intended.

Question 5: Check Divisibility by 5

Problem Statement

Write a Java program that checks if a number is divisible by 5. If it is, print "Divisible by 5".

Sample Input and Output

Sample Input 1:
25

Sample Output 1:
Divisible by 5

Sample Input 2:
23

Sample Output 2:
(no output)
Show Solution

Solution (Java Program)

import java.util.Scanner; public class DivisibleBy5 { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a number: "); int num = sn.nextInt(); if (num % 5 == 0) { System.out.println("Divisible by 5"); } sn.close(); } }

Step-by-Step Explanation

  1. Divisibility testing generalizes beyond 2: While we used % 2 for even numbers, the same pattern works for any divisor. num % 5 == 0 checks divisibility by 5, num % 3 == 0 would check divisibility by 3, and so on.
  2. This pattern appears in many algorithms: Divisibility checks are used in prime number detection, finding factors, determining leap years, and countless other algorithms. Mastering this simple pattern opens doors to solving complex problems.
  3. Understanding modulo prevents common mistakes: Beginners sometimes write num / 5 == 0, which only works for numbers less than 5. The modulo operator % is specifically designed for divisibility testing.
  4. Combining conditions creates powerful logic: You could check if a number is divisible by both 5 AND 3 using (num % 5 == 0 && num % 3 == 0), or by either 5 OR 3 using (num % 5 == 0 || num % 3 == 0). This composability makes if statements incredibly versatile.
