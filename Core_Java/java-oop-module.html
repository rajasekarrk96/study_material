Java Object-Oriented Programming (OOP)

Master the fundamental concepts of Object-Oriented Programming in Java. Learn about classes, objects, inheritance, polymorphism, encapsulation, and abstraction to write scalable, maintainable Java applications.

What is Object-Oriented Programming?

Object-Oriented Programming (OOP) is a programming paradigm that organizes software design around objects and data rather than functions and logic. In Java, everything revolves around objects, which are instances of classes. OOP helps you write code that is modular, reusable, and easier to maintain.

Key Principles of OOP:

  • Encapsulation: Bundling data and methods together, hiding internal details
  • Abstraction: Hiding complex implementation details, showing only essential features
  • Inheritance: Creating new classes based on existing ones to promote code reuse
  • Polymorphism: Objects can take multiple forms or methods can have multiple behaviors

Benefits of OOP in Java:

  • Improved code organization and structure
  • Better code reusability through inheritance
  • Enhanced security through encapsulation
  • Flexibility through polymorphism
  • Easier maintenance and debugging
💡 Key Insight: OOP is the foundation of Java. Understanding these four pillars will make you a better Java developer and help you design robust applications.

Classes and Objects: The Building Blocks

A class is a blueprint or template for creating objects. An object is an instance of a class, representing a real-world entity with specific attributes and behaviors. Every class in Java has a constructor, which is a special method used to initialize objects.

Components of a Class:

Component Description Example
Attributes (Fields) Variables that store data for the object private String name;
Methods Functions that define the behavior of the object public void getName() { ... }
Constructor Special method that initializes the object public Person(String name) { ... }
Modifiers Keywords that control access to members public, private, protected

Basic Class Definition Syntax:

public class Student {
    // Attributes
    private String name;
    private int studentId;
    private double gpa;

    // Constructor
    public Student(String name, int studentId, double gpa) {
        this.name = name;
        this.studentId = studentId;
        this.gpa = gpa;
    }

    // Methods
    public void displayInfo() {
        System.out.println("Name: " + name);
        System.out.println("ID: " + studentId);
        System.out.println("GPA: " + gpa);
    }

    public double getGpa() {
        return gpa;
    }
}

Creating and Using Objects:

// Creating an object (instantiation)
Student student1 = new Student("Alice Johnson", 101, 3.8);
Student student2 = new Student("Bob Smith", 102, 3.5);

// Calling methods
student1.displayInfo();
System.out.println("GPA: " + student1.getGpa());

✓ Real-World Example: Think of a class as a recipe for cookies. The recipe (class) contains ingredients (attributes) and instructions (methods). Each batch of cookies you make (object) follows the same recipe but can have different flavors (different attribute values).
💡 Key Concept: Use the new keyword to create an object. The constructor is automatically called when you instantiate a class. Always use meaningful names for classes (capitalized) and follow Java naming conventions.

Inheritance: Extending Classes and Promoting Reuse

Inheritance allows you to create a new class based on an existing class. The new class (child/subclass) inherits all the attributes and methods of the existing class (parent/superclass). This promotes code reuse and establishes a logical hierarchy.

Types of Inheritance in Java:

  • Single Inheritance: A child class inherits from one parent class
  • Multilevel Inheritance: A class inherits from a class that inherits from another class
  • Hierarchical Inheritance: Multiple child classes inherit from one parent class

Inheritance Syntax:

// Parent class
public class Animal {
    protected String name;

    public Animal(String name) {
        this.name = name;
    }

    public void eat() {
        System.out.println(name + " is eating.");
    }

    public void sleep() {
        System.out.println(name + " is sleeping.");
    }
}

// Child class inherits from Animal
public class Dog extends Animal {
    private String breed;

    public Dog(String name, String breed) {
        super(name);  // Call parent constructor
        this.breed = breed;
    }

    // Override parent method
    @Override
    public void eat() {
        System.out.println(name + " is eating dog food.");
    }

    // New method specific to Dog
    public void bark() {
        System.out.println(name + " says: Woof! Woof!");
    }
}

Using Inherited Classes:

Dog myDog = new Dog("Max", "Golden Retriever");
myDog.eat();      // Output: Max is eating dog food.
myDog.sleep();    // Output: Max is sleeping.
myDog.bark();     // Output: Max says: Woof! Woof!
Max is eating dog food.
Max is sleeping.
Max says: Woof! Woof!
✓ Real-World Example: A Vehicle is the parent class with properties like speed and color. Car and Motorcycle are child classes that inherit from Vehicle but add their own specific features like numberOfDoors and hasCarrier.
💡 Key Concept: Use the extends keyword to create an inheritance relationship. Use super() to call the parent constructor. Override methods using the @Override annotation to provide specialized behavior in child classes.

Polymorphism: One Interface, Multiple Forms

Polymorphism means "many forms." It allows objects to take multiple forms and enables you to use a single reference variable to point to objects of different types. There are two main types of polymorphism in Java: compile-time (method overloading) and runtime (method overriding).

Method Overloading (Compile-time Polymorphism):

Multiple methods with the same name but different parameters in the same class.

public class Calculator {
    // Overloaded methods with different parameter types
    public int add(int a, int b) {
        return a + b;
    }

    public double add(double a, double b) {
        return a + b;
    }

    public int add(int a, int b, int c) {
        return a + b + c;
    }
}

Calculator calc = new Calculator();
System.out.println(calc.add(5, 10));              // Calls first method
System.out.println(calc.add(5.5, 10.5));         // Calls second method
System.out.println(calc.add(5, 10, 15));         // Calls third method
15
16.0
30

Method Overriding (Runtime Polymorphism):

A child class provides a specific implementation of a method already present in the parent class.

public class Shape {
    public void draw() {
        System.out.println("Drawing a generic shape");
    }
}

public class Circle extends Shape {
    @Override
    public void draw() {
        System.out.println("Drawing a circle");
    }
}

public class Rectangle extends Shape {
    @Override
    public void draw() {
        System.out.println("Drawing a rectangle");
    }
}

// Polymorphic behavior
Shape shape1 = new Circle();
Shape shape2 = new Rectangle();

shape1.draw();    // Output: Drawing a circle
shape2.draw();    // Output: Drawing a rectangle
Drawing a circle
Drawing a rectangle
✓ Real-World Example: A payment system can accept multiple payment methods (CreditCard, PayPal, Bitcoin). Each has its own implementation of the process() method. The system doesn't need to know which specific payment method is being used; it just calls process() on any Payment object.
💡 Key Concept: Polymorphism makes your code flexible and extensible. Always use parent class references to point to child class objects. This allows you to add new classes without modifying existing code, following the Open-Closed Principle.

Encapsulation: Data Hiding and Protection

Encapsulation is the practice of bundling data (attributes) and methods together, while hiding the internal details from the outside world. This is achieved through access modifiers like private, public, and protected. Encapsulation protects data integrity and provides controlled access through getters and setters.

Access Modifiers in Java:

Modifier Same Class Same Package Subclass Outside
private ✓ ✗ ✗ ✗
protected ✓ ✓ ✓ ✗
public ✓ ✓ ✓ ✓
default (no modifier) ✓ ✓ ✗ ✗

Encapsulation with Getters and Setters:

public class BankAccount {
    private String accountNumber;
    private double balance;
    private String accountHolder;

    // Constructor
    public BankAccount(String accountNumber, double balance, String accountHolder) {
        this.accountNumber = accountNumber;
        this.balance = balance;
        this.accountHolder = accountHolder;
    }

    // Getter for balance
    public double getBalance() {
        return balance;
    }

    // Setter for balance with validation
    public void setBalance(double amount) {
        if (amount >= 0) {
            this.balance = amount;
        } else {
            System.out.println("Balance cannot be negative!");
        }
    }

    // Method to deposit money
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            System.out.println("Deposited: $" + amount);
        }
    }

    // Method to withdraw money
    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            System.out.println("Withdrawn: $" + amount);
        } else {
            System.out.println("Invalid withdrawal amount!");
        }
    }
}

Using Encapsulated Class:

BankAccount account = new BankAccount("12345", 1000, "John Doe");
account.deposit(500);           // Output: Deposited: $500
System.out.println("Balance: $" + account.getBalance()); // Output: Balance: $1500
account.withdraw(200);          // Output: Withdrawn: $200
account.setBalance(-100);       // Output: Balance cannot be negative!
Deposited: $500
Balance: $1500
Withdrawn: $200
Balance cannot be negative!
✓ Real-World Example: An ATM machine encapsulates the banking system. Users can only interact through public methods like withdraw() and deposit(). They cannot directly access the internal balance or account data, ensuring security and data integrity.
💡 Key Concept: Always make data fields private and provide public getters and setters. This allows you to add validation logic, control access, and change internal implementation without affecting external code. This is crucial for maintaining data integrity.

Abstraction: Hiding Complexity

Abstraction is the process of hiding complex implementation details and showing only the essential features of an object. In Java, this is achieved using abstract classes and interfaces. Abstraction helps you focus on what an object does rather than how it does it.

Abstract Classes:

An abstract class cannot be instantiated and may contain abstract methods (without implementation) and concrete methods (with implementation).

// Abstract class
public abstract class Vehicle {
    private String color;

    public Vehicle(String color) {
        this.color = color;
    }

    // Abstract method (must be implemented by subclasses)
    public abstract void start();
    public abstract void stop();

    // Concrete method
    public void displayColor() {
        System.out.println("Vehicle color: " + color);
    }
}

// Concrete class implementing abstract methods
public class Car extends Vehicle {
    public Car(String color) {
        super(color);
    }

    @Override
    public void start() {
        System.out.println("Car engine started with a key.");
    }

    @Override
    public void stop() {
        System.out.println("Car engine stopped.");
    }
}

// Using abstract class
Car myCar = new Car("Red");
myCar.start();           // Output: Car engine started with a key.
myCar.displayColor();    // Output: Vehicle color: Red
myCar.stop();            // Output: Car engine stopped.
Car engine started with a key.
Vehicle color: Red
Car engine stopped.
💡 Key Concept: Use abstract classes when you have a common base with shared code. Abstract methods define a contract that all subclasses must follow. This ensures consistency across different implementations while allowing flexibility.

Interfaces: Defining Contracts

An interface is a contract that specifies what methods a class must implement. Unlike abstract classes, interfaces contain only abstract methods (and now static/default methods in Java 8+) and no instance variables. Interfaces enable multiple inheritance of type in Java and are useful for defining standard behaviors that unrelated classes can implement.

Interface Definition and Implementation:

// Interface definition
public interface Drawable {
    void draw();
    void erase();
}

public interface Resizable {
    void resize(int width, int height);
}

// Class implementing multiple interfaces
public class Shape implements Drawable, Resizable {
    private String name;

    public Shape(String name) {
        this.name = name;
    }

    @Override
    public void draw() {
        System.out.println("Drawing " + name);
    }

    @Override
    public void erase() {
        System.out.println("Erasing " + name);
    }

    @Override
    public void resize(int width, int height) {
        System.out.println("Resizing " + name + " to " + width + "x" + height);
    }
}

// Using interfaces
Shape shape = new Shape("Triangle");
shape.draw();                    // Output: Drawing Triangle
shape.resize(100, 150);         // Output: Resizing Triangle to 100x150
shape.erase();                   // Output: Erasing Triangle
Drawing Triangle
Resizing Triangle to 100x150
Erasing Triangle

Interface vs Abstract Class Comparison:

Feature Interface Abstract Class
Methods All abstract (Java 7) / Can have default methods (Java 8+) Mix of abstract and concrete methods
Variables Static and final only Any type of variables
Inheritance Multiple inheritance possible Single inheritance only
Instantiation Cannot be instantiated Cannot be instantiated
Use Case Define contracts for unrelated classes Share code among closely related classes
✓ Real-World Example: In a music player application, implement interfaces like Playable (play, pause, stop) and Recordable (record, stopRecording) for different media types like Audio and Video. Each class implements only the interfaces relevant to it.
💡 Key Concept: Use interfaces to define what a class can do (contracts), and use abstract classes to define what a class is (hierarchy). Interfaces promote loose coupling and make your code more flexible and testable. Consider using interfaces as your primary design tool.
