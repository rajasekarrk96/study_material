Question 1: try-catch-finally Blocks

Problem Statement

Write a Java program that demonstrates the use of try-catch-finally blocks to handle exceptions. Show how the finally block executes regardless of whether an exception occurs.

Solution

class _01_ { public static void main(String[] args) { try { int a = 10 / 0; } catch (ArithmeticException e) { System.out.println("Exception caught"); } finally { System.out.println("Finally block executed"); } } }

Sample Output

Exception caught Finally block executed
Key Concepts:
  • try Block: Contains code that might throw an exception
  • catch Block: Handles the exception if thrown
  • finally Block: Always executes (cleanup code)
  • Execution Order: try → catch (if exception) → finally

Question 2: Multiple catch Blocks

Problem Statement

Write a Java program that demonstrates the use of multiple catch blocks to handle different types of exceptions. Show how specific exceptions are caught before general exceptions.

Solution

class _02_ { public static void main(String[] args) { try { int[] arr = new int[5]; arr[10] = 50; } catch (ArithmeticException e) { System.out.println("Arithmetic Exception"); } catch (ArrayIndexOutOfBoundsException e) { System.out.println("Array Index Out Of Bounds Exception"); } catch (Exception e) { System.out.println("General Exception"); } } }

Sample Output

Array Index Out Of Bounds Exception
Key Concepts:
  • Multiple catch: Handle different exception types
  • Order Matters: Specific exceptions before general ones
  • First Match: Only one catch block executes

Question 3: throw and throws Keywords

Problem Statement

Write a Java program that demonstrates the use of throw and throws keywords to handle exceptions. Show how to explicitly throw an exception and declare it in the method signature.

Solution

class _03_ { static void checkAge(int age) throws ArithmeticException { if (age < 18) throw new ArithmeticException("Not eligible"); else System.out.println("Eligible"); } public static void main(String[] args) { checkAge(16); } }

Sample Output

Exception in thread "main" java.lang.ArithmeticException: Not eligible at _03_.checkAge(_03_.java:5) at _03_.main(_03_.java:11)
Key Concepts:
  • throw: Explicitly throws an exception
  • throws: Declares exceptions in method signature
  • Propagation: Exception propagates to caller

Question 4: Checked and Unchecked Exceptions

Problem Statement

Write a Java program that demonstrates the use of checked and unchecked exceptions. Show the difference between exceptions that must be handled and those that don't require explicit handling.

Solution

import java.io.*; class _07_ { public static void main(String[] args) { try { FileReader fr = new FileReader("file.txt"); // Checked } catch (Exception e) { System.out.println("Checked exception handled"); } int a = 10 / 0; // Unchecked exception } }

Sample Output

Checked exception handled Exception in thread "main" java.lang.ArithmeticException: / by zero
Key Concepts:
  • Checked: Must be caught or declared (IOException, SQLException)
  • Unchecked: Runtime exceptions (ArithmeticException, NullPointerException)
  • Compile-time: Checked exceptions verified at compile time

Question 5: StackTraceElement Class

Problem Statement

Write a Java program that demonstrates the use of the StackTraceElement class to print the stack trace of an exception. Show how to access detailed information about where the exception occurred.

Solution

class _08_ { public static void main(String[] args) { try { int a = 10 / 0; } catch (Exception e) { StackTraceElement[] st = e.getStackTrace(); for (StackTraceElement s : st) { System.out.println(s); } } } }

Sample Output

_08_.main(_08_.java:5)
Key Concepts:
  • getStackTrace(): Returns array of StackTraceElement
  • Stack Trace: Shows method call hierarchy
  • Debugging: Helps locate exception source

Question 6: finally for Cleanup Operations

Problem Statement

Write a Java program that demonstrates the use of the finally block to perform cleanup operations after handling an exception. Show that finally executes even when an exception is caught.

Solution

class _09_ { public static void main(String[] args) { try { int a = 5 / 0; } catch (Exception e) { System.out.println("Exception handled"); } finally { System.out.println("Cleanup done"); } } }

Sample Output

Exception handled Cleanup done
Key Concepts:
  • finally Always Runs: Executes regardless of exception
  • Cleanup: Close resources, release locks
  • Use Cases: File closing, database connections, network sockets
