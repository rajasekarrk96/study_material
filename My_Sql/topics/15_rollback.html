Rollback

ROLLBACK is a TCL command used to undo changes made in a transaction that is not yet committed.

What does ROLLBACK do?

🔹 Key Features
🔹 Reverts changes made by INSERT, UPDATE, DELETE
🔹 Works only before COMMIT
🔹 Used to recover from mistakes

Basic ROLLBACK

🔹 Example
Undoing an update.

🔹 Example
START TRANSACTION;\n\nUPDATE students\nSET age = 25\nWHERE id = 1;\n\nROLLBACK;
✔ Result: Age remains unchanged

ROLLBACK with DELETE

🔹 Example
Restoring a deleted row.

🔹 Example
START TRANSACTION;\n\nDELETE FROM students\nWHERE id = 3;\n\nROLLBACK;
✔ Result: Row with ID 3 is restored

ROLLBACK with SAVEPOINT

🔹 Example
Partial rollback to a specific point.

🔹 Syntax
SAVEPOINT sp1;\nROLLBACK TO sp1;
🔹 Example
START TRANSACTION;\n\nINSERT INTO students VALUES (5, 'Kiran', 22);\nSAVEPOINT s1;\n\nUPDATE students SET age = 23 WHERE id = 5;\nROLLBACK TO s1;\n\nCOMMIT;
✔ Changes after SAVEPOINT s1 are undone.

Query:

WHERE id = 1;

Query:
ROLLBACK;

Query OK
