Normalization

Normalization is the process of organizing a database to reduce redundancy and improve data integrity by splitting large tables into smaller, related ones.

Purpose & Core Concepts

🔹 Why Normalize?

  • Remove duplicate data.
  • Avoid insertion, update, and deletion anomalies.
  • Ensure data consistency and optimize storage.

Normal Forms (NF) Summary

A quick reference for the transformation stages.

Normal FormRequirement
1NFEach column contains atomic values, no repeating groups.
2NFIn 1NF, and all non-key columns fully depend on the primary key.
3NFIn 2NF, and no transitive dependency exists.
BCNFStronger 3NF; every determinant is a candidate key.
4NFHandles multi-valued dependencies.
5NFHandles join dependencies.

First Normal Form (1NF)

🔹 Rule: Make columns atomic (no multiple values in a single cell).

🔹 Example
CREATE TABLE Students_1NF (\n    StudentID INT,\n    StudentName VARCHAR(50),\n    Course VARCHAR(50),\n    Instructor VARCHAR(50)\n);

StudentIDStudentNameCourseInstructor

1JohnPhysicsMr.B
2AliceMathMr.A

Second Normal Form (2NF)

🔹 Rule: Remove partial dependencies. Split data into logical entities.

🔹 Example
-- Students Table\nCREATE TABLE Students (\n    StudentID INT PRIMARY KEY,\n    StudentName VARCHAR(50)\n);\n\n-- Courses Table\nCREATE TABLE Courses (\n    Course VARCHAR(50) PRIMARY KEY,\n    Instructor VARCHAR(50)\n);\n\n-- Enrollment Join Table\nCREATE TABLE Enrollment (\n    StudentID INT,\n    Course VARCHAR(50),\n    PRIMARY KEY (StudentID, Course),\n    FOREIGN KEY (StudentID) REFERENCES Students(StudentID),\n    FOREIGN KEY (Course) REFERENCES Courses(Course)\n);
Students:

2Alice
Courses:

Third Normal Form (3NF)

🔹 Rule: Remove transitive dependencies (non-key columns shouldn't depend on other non-key columns).

✅ Satisfied if StudentName depends only on StudentID, and Instructor depends only on Course.

Boyce-Codd Normal Form (BCNF)

🔹 Rule: For every functional dependency (X → Y), X must be a superkey.

🔹 Example
-- Decomposing to satisfy BCNF\nCREATE TABLE Course_Room (CourseID VARCHAR(10) PRIMARY KEY, Room VARCHAR(10));\nCREATE TABLE Room_Instructor (Room VARCHAR(10) PRIMARY KEY, Instructor VARCHAR(50));
✔ Resolves anomalies where a non-prime attribute determines another non-prime attribute.

Fourth Normal Form (4NF)

🔹 Rule: No multi-valued dependencies. Independent facts go to separate tables.

🔹 Example
-- Student hobbies and phones split into different tables\nCREATE TABLE Student_Hobby (StudentID INT, Hobby VARCHAR(50));\nCREATE TABLE Student_Phone (StudentID INT, Phone VARCHAR(15));
✔ Prevents redundancy when one key has multiple independent attributes.

Fifth Normal Form (5NF)

🔹 Rule: Handles join dependencies; ensures data can be reconstructed exactly.

🔹 Example
-- Splitting Employee, Project, and Skill into 3 binary relationships\nCREATE TABLE Employee_Project (...);\nCREATE TABLE Employee_Skill (...);\nCREATE TABLE Project_Skill (...);
✔ Minimal redundancy for complex N-way relationships.

Practice Worksheet

Q1. What is normalization?
Answer:
Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity. It divides large tables into smaller, related tables and defines relationships between them.

Example: A table storing student and department data:
StudentID  Name   DeptID  DeptName
1          Arun   101     IT
2          Priya  102     HR
3          Kiran  101     IT

Redundant DeptName column exists.
Q2. First Normal Form (1NF)
Rule:
- Remove repeating groups
- Each column must hold atomic values (no multiple values in a column)

Example Table:
StudentID   Name    Subjects
1           Arun    Math, Science

Normalized 1NF Table:
StudentID   Name    Subject
1           Arun    Math
1           Arun    Science
Q3. Second Normal Form (2NF)
Rule:
- Must be in 1NF
- Remove partial dependency (columns should depend on the whole primary key)

Normalized 2NF Tables:
Students Table:
StudentID  Name
1          Arun

Courses Table:
CourseID   CourseName   Instructor

Enrollment Table:
StudentID  CourseID
1          101
Q4. Third Normal Form (3NF)
Rule:
- Must be in 2NF
- Remove transitive dependency (non-key columns should not depend on other non-key columns)

Normalized 3NF Tables:
Students Table:
StudentID  Name  DeptID

Departments Table:
DeptID   DeptName
Q5. Benefits of Normalization
- Reduces data redundancy
- Improves data integrity
- Easier to maintain and update
- Makes queries more efficient

Example: Instead of repeating department names in multiple student rows, you store department once in a separate table and link with DeptID.
