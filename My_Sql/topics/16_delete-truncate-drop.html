DELETE / TRUNCATE / DROP

Commands to remove data or entire table structures.

DELETE

🔹 Definition
Removes selected rows (or all rows) from a table.

🔹 Syntax
DELETE FROM table_name\nWHERE condition;
🔹 Example
DELETE FROM students\nWHERE age < 20;
🔹 Key Points
✔ DML command
✔ Can use WHERE
✔ Can be rolled back
✔ Triggers are fired
✔ Slower for large data

TRUNCATE

🔹 Definition
Removes all rows from a table at once.

🔹 Syntax
TRUNCATE TABLE table_name;
🔹 Example
TRUNCATE TABLE students;
🔹 Key Points
✔ DDL command
✔ No WHERE clause allowed
✔ Cannot be rolled back
✔ Faster than DELETE
✔ Resets AUTO_INCREMENT

DROP

🔹 Definition
Deletes the entire table structure along with all its data.

🔹 Syntax
DROP TABLE table_name;
🔹 Example
DROP TABLE students;
🔹 Key Points
✔ DDL command
✔ Table no longer exists
✔ Cannot be rolled back
✔ Frees storage space

Practice Worksheet

Q1. Delete a specific record (student with id = 2).
Query:
DELETE FROM students
WHERE id = 2;

Output:
Query OK, 1 row affected
Q2. Delete all records from students table.
Query:
DELETE FROM students;

Output:
Query OK, 4 rows affected
Q3. Rollback the DELETE operation.
Query:
ROLLBACK;

Output:
Query OK
Q4. Remove all records using TRUNCATE.
Query:
TRUNCATE TABLE students;

Output:
Query OK, 0 rows affected
Q5. Delete the entire table using DROP.
Query:
DROP TABLE students;

Output:
Query OK, 0 rows affected
