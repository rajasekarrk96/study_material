Views

A VIEW is a virtual table created from a SQL query. It does not store data itself; it shows data from one or more tables.

Basic Concepts

🔹 Definition
A virtual table that represents the result of a stored query.

🔹 Simplifies complex queries
🔹 Improves security
🔹 Reusable query
🔹 Always shows latest data

CREATE VIEW

🔹 Definition
Creates a view that filters or combines data from existing tables.

🔹 Syntax
CREATE VIEW view_name AS\nSELECT column1, column2\nFROM table_name\nWHERE condition;
🔹 Example
CREATE VIEW high_salary_employees AS\nSELECT emp_id, emp_name, salary\nFROM employees\nWHERE salary > 50000;

Update Data Using VIEW

🔹 Definition
Updating records through a view also updates the underlying base table.

🔹 Syntax
UPDATE view_name\nSET column = value\nWHERE condition;
🔹 Example
UPDATE high_salary_employees\nSET salary = 65000\nWHERE emp_id = 101;
✔ Original table employees is updated.

DROP VIEW

🔹 Definition
Deletes the virtual table definition from the database.

🔹 Syntax
DROP VIEW view_name;
🔹 Example
DROP VIEW high_salary_employees;
Query OK, 0 rows affected

Q1. Create a VIEW to display only id and name from students.
Query:
CREATE VIEW student_view AS

Q3. Create a VIEW with a condition (age > 20).
Query:
CREATE VIEW adult_students AS
SELECT * FROM students
WHERE age > 20;

Query:

SET name = 'Arun Kumar'
WHERE id = 1;

Query:

Output:
