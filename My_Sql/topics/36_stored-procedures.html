Stored Procedures

A Stored Procedure is a predefined set of SQL statements stored in the database that can be executed whenever needed, supporting parameters and reusable logic.

Basic Concept & Syntax

🔹 Logic
Procedures can take input parameters, return output through OUT variables, and are used for bulk logic, security, and performance.

🔹 Syntax
-- Change delimiter to allow semicolons inside procedure\nDELIMITER //\n\nCREATE PROCEDURE procedure_name (IN input_param INT, OUT output_param VARCHAR(50))\nBEGIN\n    -- SQL statements\n    SELECT column_name INTO output_param\n    FROM table_name\n    WHERE id = input_param;\nEND //\n\nDELIMITER ;

Parameter Types

    \n
  • IN: Input parameter (passed from caller).
    • \n
  • OUT: Output parameter (returned to caller).
    • \n
  • INOUT: Acts as both input and output.
    • \n
  • SELECT ... INTO: Stores a single result into an OUT variable.
    • \n

Example: Create Stored Procedure

🔹 Goal
Create a procedure that retrieves a student name by their ID.

🔹 Example
DELIMITER //\n\nCREATE PROCEDURE GetStudentName(IN sid INT, OUT sname VARCHAR(50))\nBEGIN\n    SELECT student_name INTO sname\n    FROM students\n    WHERE student_id = sid;\nEND //\n\nDELIMITER ;

Example: Calling the Procedure

🔹 Logic
Use a user-defined variable (starting with @) to capture the OUT value.

🔹 Example
CALL GetStudentName(2, @name);\nSELECT @name;
Result:
@name
Alice

Practice Worksheet

Q1. Simple procedure to select all students.
Query:
DELIMITER $$

CREATE PROCEDURE get_all_students()
BEGIN
    SELECT * FROM students;
END $$

DELIMITER ;

Usage Example:
CALL get_all_students();

Output:
+----+--------+-----+
| id | name   | age |
+----+--------+-----+
| 1  | Arun   | 22  |
| 2  | Priya  | 23  |
| 3  | Kiran  | 23  |
| 4  | Meena  | 24  |
+----+--------+-----+
Q2. Procedure with input parameter – Get student by ID.
Query:
DELIMITER $$

CREATE PROCEDURE get_student_by_id(IN stu_id INT)
BEGIN
    SELECT * FROM students WHERE id = stu_id;
END $$

DELIMITER ;

Usage Example:
CALL get_student_by_id(2);

Output:
+----+-------+-----+
| id | name  | age |
+----+-------+-----+
| 2  | Priya | 23  |
+----+-------+-----+
Q3. Procedure with output parameter – Get student age.
Query:
DELIMITER $$

CREATE PROCEDURE get_student_age(IN stu_id INT, OUT stu_age INT)
BEGIN
    SELECT age INTO stu_age FROM students WHERE id = stu_id;
END $$

DELIMITER ;

Usage Example:
CALL get_student_age(3, @age);
SELECT @age;

Output:
+------+
| @age |
+------+
| 23   |
+------+
Q4. Procedure to insert a new student.
Query:
DELIMITER $$

CREATE PROCEDURE add_student(IN stu_name VARCHAR(50), IN stu_age INT)
BEGIN
    INSERT INTO students(name, age) VALUES(stu_name, stu_age);
END $$

DELIMITER ;

Usage Example:
CALL add_student('Ravi', 22);
SELECT * FROM students;

Output:
+----+--------+-----+
| id | name   | age |
+----+--------+-----+
| 1  | Arun   | 22  |
| 2  | Priya  | 23  |
| 3  | Kiran  | 23  |
| 4  | Meena  | 24  |
| 5  | Ravi   | 22  |
+----+--------+-----+
Q5. Procedure to update student age.
Query:
DELIMITER $$

CREATE PROCEDURE update_student_age(IN stu_id INT, IN new_age INT)
BEGIN
    UPDATE students SET age = new_age WHERE id = stu_id;
END $$

DELIMITER ;

Usage Example:
CALL update_student_age(5, 23);
SELECT * FROM students WHERE id = 5;

Output:
+----+-------+-----+
| id | name  | age |
+----+-------+-----+
| 5  | Ravi  | 23  |
+----+-------+-----+
