Constraints

Constraints

Constraints are rules applied to table columns to ensure data accuracy, integrity, and reliability.

NOT NULL

🔹 Definition
Prevents a column from having NULL values.

🔹 Syntax
column_name datatype NOT NULL
🔹 Example
CREATE TABLE students (\n    student_id INT NOT NULL,\n    student_name VARCHAR(50) NOT NULL\n);
❌ NULL values not allowed in student_id and student_name

UNIQUE

🔹 Definition
Ensures all values in a column are different.

🔹 Syntax
column_name datatype UNIQUE
🔹 Example
CREATE TABLE users (\n    email VARCHAR(100) UNIQUE\n);
❌ Duplicate email values not allowed

PRIMARY KEY

🔹 Definition
Uniquely identifies each record in a table (NOT NULL + UNIQUE).

🔹 Syntax
PRIMARY KEY (column_name)
🔹 Example
CREATE TABLE employees (\n    employee_id INT PRIMARY KEY,\n    employee_name VARCHAR(50)\n);
✔ Each employee_id is unique and not null

FOREIGN KEY

🔹 Definition
Creates a relationship between two tables.

🔹 Syntax
FOREIGN KEY (column_name)\nREFERENCES parent_table(column_name)
🔹 Example
CREATE TABLE orders (\n    order_id INT PRIMARY KEY,\n    customer_id INT,\n    FOREIGN KEY (customer_id)\n    REFERENCES customers(customer_id)\n);
❌ Cannot insert a customer_id that does not exist in customers

CHECK

🔹 Definition
Restricts values based on a condition.

🔹 Syntax
CHECK (condition)
🔹 Example
CREATE TABLE products (\n    price DECIMAL(10,2),\n    CHECK (price > 0)\n);
❌ Price must be greater than 0

DEFAULT

🔹 Definition
Assigns a default value if no value is provided.

🔹 Syntax
column_name datatype DEFAULT value
🔹 Example
CREATE TABLE accounts (\n    status VARCHAR(20) DEFAULT 'Active'\n);
✔ If status not given → 'Active' is inserted

AUTO_INCREMENT (MySQL)

🔹 Definition
Automatically generates incrementing numbers.

🔹 Syntax
column_name INT AUTO_INCREMENT
🔹 Example
CREATE TABLE classes (\n    class_id INT AUTO_INCREMENT PRIMARY KEY,\n    class_name VARCHAR(50)\n);
✔ class_id values → 1, 2, 3, 4, ...
