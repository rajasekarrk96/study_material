Indexes

An INDEX is a database object used to speed up data retrieval from a table. It works like an index in a book, helping the database find rows faster.

Basic Concepts

🔹 Definition
Used to speed up search queries. Invisible to end users.

🔹 Index improves SELECT performance
🔹 Uses extra storage
🔹 Slows down INSERT / UPDATE / DELETE slightly

CREATE INDEX

🔹 Definition
Creates a standard index on one or more columns.

🔹 Syntax
CREATE INDEX index_name\nON table_name (column_name);
🔹 Example
CREATE INDEX idx_student_name\nON students (student_name);
✔ Faster execution of queries like:
SELECT * FROM students WHERE student_name = 'Arun';

✔ No visible change in data, but performance improves.

UNIQUE INDEX

🔹 Definition
Creates an index where all values in the index must be unique (similar to UNIQUE constraint).

🔹 Syntax
CREATE UNIQUE INDEX index_name\nON table_name (column_name);
🔹 Example
CREATE UNIQUE INDEX idx_email\nON students (email);
❌ Duplicate email values not allowed

DROP INDEX (MySQL)

🔹 Definition
Permanently removes an existing index from a table.

🔹 Syntax
DROP INDEX index_name ON table_name;
🔹 Example
DROP INDEX idx_student_name ON students;
Query OK, 0 rows affected
