ACID Properties in SQL

ACID Properties in SQL

ACID properties ensure that database transactions are reliable and safe.
ACID = Atomicity, Consistency, Isolation, Durability.

Atomicity

🔹 Definition
A transaction is all or nothing. If one operation fails, the entire transaction is rolled back.

🔹 Example
START TRANSACTION;

INSERT INTO accounts VALUES (1, 'Vetri', 5000);
INSERT INTO accounts VALUES (2, 'Ravi', 3000);

ROLLBACK;
0 rows inserted
(accounts table remains unchanged)

Consistency

🔹 Definition
Ensures the database always follows rules, constraints, and integrity.

🔹 Example
UPDATE accounts
SET balance = -1000
WHERE id = 1;
ERROR: CHECK constraint failed
(No change in table)

Isolation

🔹 Definition
Transactions execute independently without affecting each other.

🔹 Example
-- Transaction 1
START TRANSACTION;
UPDATE accounts SET balance = balance - 500 WHERE id = 1;

-- Transaction 2
SELECT balance FROM accounts WHERE id = 1;
Transaction 2 shows old balance (uncommitted data not visible)

Durability

🔹 Definition
Once committed, data is permanently saved, even after failure.

🔹 Example
COMMIT;
Transaction committed successfully
(Data remains saved permanently)

Practice Worksheet

Q1. Atomicity
A money transfer deducts 500 from Account 1 and adds 500 to Account 2.
Second query fails. What will be the final output?
Query:
START TRANSACTION;

UPDATE accounts SET balance = balance - 500 WHERE acc_id = 1;
UPDATE accounts SET balance = balance + 500 WHERE acc_id = 3;

COMMIT;

Output:
❌ Error (Account 3 does not exist)
Transaction ROLLBACK

acc_id balance
1 5000
2 3000

➡ Atomicity ensured (all or nothing)
Q2. Consistency
Salary must be ≥ 10000. What happens if we insert salary = 8000?
Table:
CREATE TABLE employee (
  emp_id INT PRIMARY KEY,
  salary INT CHECK (salary >= 10000)
);

Query:
INSERT INTO employee VALUES (101, 8000);

Output:
❌ Error: CHECK constraint failed
❌ Row NOT inserted

➡ Consistency maintained
Q3. Isolation
Transaction T1 updates price but does not commit. What value will T2 see?
Transaction T1:
START TRANSACTION;
UPDATE product SET price = 1200 WHERE id = 1;

Transaction T2:
SELECT price FROM product WHERE id = 1;

Output (T2):
price
1000

➡ Uncommitted data not visible → Isolation
Q4. Durability
After commit, system crashes. Will data remain?
Query:
START TRANSACTION;
INSERT INTO orders VALUES (1, 'Laptop', 50000);
COMMIT;

After Restart:
SELECT * FROM orders;

Output:
order_id product price
1 Laptop 50000

➡ Committed data is permanent → Durability
Q5. Identify ACID using Query Output
Query:
START TRANSACTION;
DELETE FROM students WHERE id = 5;
ROLLBACK;

Output:
id name
5 Arun

ACID Property:
✅ Atomicity (rollback restores data)
