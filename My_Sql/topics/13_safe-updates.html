Safe Updates

Safe Update Mode is a MySQL feature that prevents accidental UPDATE or DELETE operations on an entire table.

Error due to Safe Update

🔹 Symptom
Occurs when trying to update/delete without a WHERE clause using a primary key.

🔹 Example
-- Error Code: 1175\n-- You are using safe update mode and you tried to update a table without a WHERE that uses a KEY column.
❌ Operation blocked by MySQL Workbench

UPDATE not allowed (Safe Mode ON)

🔹 Example
A generic update without a key in the WHERE clause.

🔹 Example
-- ❌ Wrong (will cause Error 1175)\nUPDATE students\nSET age = 22;
❌ Error: Safe update mode enabled

Correct UPDATE (Safe Mode ON)

🔹 Solution
Always use the Primary Key in the WHERE clause.

🔹 Example
UPDATE students\nSET age = 22\nWHERE id = 1;
✔ Correct: Primary key used. Update successful.

Disable Safe Update (Temporary)

🔹 Command
Turns off safe mode for the current session.

🔹 Example
SET SQL_SAFE_UPDATES = 0;\n\n-- ✔️ Now you can run:\nUPDATE students\nSET age = 22;
✔ Safe mode disabled for this session.

Enable Safe Update Again

🔹 Best Practice
Always turn safe mode back on after performing bulk updates.

🔹 Example
SET SQL_SAFE_UPDATES = 1;
✔ Safe mode re-enabled.
