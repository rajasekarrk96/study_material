Bootstrap Interview Challenges

Practice these 5 Bootstrap challenges ranging from Easy to Hard.

1. Mobile-First Grid Translation (Easy)

Problem: Write the HTML markup for a div.row containing three div children. Requirements:
1. On mobile devices (extra small), the children should stack vertically (1 per row).
2. On medium devices (tablets), the children should be 50% width each (2 per row, 3rd wraps to next line).
3. On large devices (desktops), the children should be 33.33% width each (3 per row).

Goal: Test fundamental understanding of Bootstrap breakpoints (sm/md/lg) and 12-column math.

View Solution Sketch 
<div class="row">
  <div class="col-12 col-md-6 col-lg-4">Column 1</div>
  <div class="col-12 col-md-6 col-lg-4">Column 2</div>
  <div class="col-12 col-md-6 col-lg-4">Column 3</div>
</div>

2. Margin and Padding Aliases (Easy)

Problem: Without writing custom CSS, use Bootstrap utility classes to create a red button that has text reading "Delete". Ensure the button has 3rem of padding on the left and right, 0.5rem of padding on top and bottom, and 2rem of margin added exclusively to its bottom.

Goal: Verify fluency with Bootstrap's spacing utility conventions (pt, pb, px, py, mt, mb...).

3. Visibility Management (Medium)

Problem: Provide the Bootstrap utility classes necessary to apply to an image so that it is completely hidden on mobile screens, visible on Medium screens, hidden again on Large screens, and visible again on Extra Large (xl) screens.

Goal: Validate deeper understanding of `display` utilities overriding each other up through responsive tiers (e.g., `d-none d-md-block d-lg-none d-xl-block`).

4. Complex Nested Layouts (Medium)

Problem: Implement a hero section. It requires a container. Inside the container, a row spanning 8 columns out of 12 by default, horizontally centered (using grid techniques, not auto margins). Within that centered column, establish another nested grid system that holds an image on the left (4 cols) and text on the right (8 cols).

Goal: Ensure knowledge of nested grids and `.justify-content-center` or `.offset-*` classes.

5. Customizing Bootstrap Variables via SASS (Hard)

Problem: You are tasked with integrating Bootstrap via Node/SASS (not CDN). The design team requires the primary color to be changed from default blue to deep purple (#6f42c1). What is the exact sequence of SASS imports and variable declarations required in your main.scss file to achieve this safely without directly modifying Bootstrap's source files?

Goal: Test advanced framework customization knowledge. Validating that variables must be overridden *before* importing Bootstrap.

View Solution Sketch 
// 1. Define custom variable overrides
$primary: #6f42c1;

// 2. ONLY THEN import bootstrap source
@import "node_modules/bootstrap/scss/bootstrap";
