CSS Interview Challenges

Practice these 5 CSS challenges ranging from Easy to Hard.

1. The Holy Grail Centering (Easy)

Problem: Create a container div taking up the full screen height and width. Inside it, place a smaller child box (e.g., 200x200px). Write the CSS to perfectly center the child box horizontally and vertically on the screen. Provide two different solutions (Flexbox and Absolute Positioning).

Goal: Test absolute fundamentals of modern alignment.

View Solution Sketch 
/* Solution 1: Flexbox */
.parent {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 100vh;
}

/* Solution 2: Absolute Positioning */
.parent {
  position: relative;
  height: 100vh;
}
.child {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}

2. Nav Bar using Flexbox (Easy)

Problem: Build a navigation bar. The logo should sit on the far left. A group of list items (Home, About, Contact) should be grouped together on the far right. Use Flexbox to separate them without explicitly assigning widths or hardcoded margins.

Goal: Test understanding of `justify-content` and typical flexbox patterns.

3. Specificity Calculator (Medium)

Problem: Given the following HTML: <div id="main" class="container"><p class="text">Hello</p></div>.
And the following CSS rules, what color will the text be, and why?

  • div p { color: blue; }
  • #main p { color: green; }
  • .container .text { color: red; }
  • div p.text { color: purple; }

Goal: Validate profound understanding of the cascade and specificity rules.

View Answer

Green. Specificity values:
1. div p = (0, 0, 0, 2)
2. #main p = (0, 1, 0, 1) -> WINS (1 ID)
3. .container .text = (0, 0, 2, 0)
4. div p.text = (0, 0, 1, 2)

4. The Responsive Auto Grid (Medium)

Problem: Write CSS for a product card container. It should display cards in columns. The cards should never be smaller than 250px wide, but should expand to take up remaining space. As the browser resizes, the columns should automatically wrap. Constraint: Do not use any @media queries.

Goal: Ensure knowledge of CSS Grid's powerful native responsive features (repeat, auto-fit, minmax).

5. CSS Triangle / Shapes (Hard)

Problem: Using only a single empty <div> element, write the CSS to render an upward-pointing triangle. Do not use border-radius, SVG, or background images. Rely entirely on the border property behavior.

Goal: Test knowledge of how CSS draws thick borders where colors meet at an angle. Often asked to see if a candidate builds tooltips/carets cleanly.

View Solution Sketch 
.triangle-up {
  width: 0;
  height: 0;
  border-left: 50px solid transparent;
  border-right: 50px solid transparent;
  border-bottom: 100px solid red; 
  /* Top border is 0, so bottom border expands upwards into the transparent space */
}
