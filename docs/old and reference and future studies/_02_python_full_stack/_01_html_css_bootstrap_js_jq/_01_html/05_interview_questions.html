HTML Interview Challenges

Practice these 5 HTML challenges ranging from Easy to Hard.

1. Semantic Layout Structure (Easy)

Problem: Create a standard web page layout using only semantic HTML5 elements (no div or span). The layout should include a header, navigation, main content area with two articles, a sidebar, and a footer.

Goal: Test understanding of document structure and semantic tags.

</header>

  <article>...</article>
  <aside>...</aside>
</main>
<footer>...</footer>

2. Accessible Form Data Collection (Easy)

Problem: Build an accessible login form collecting an email and password. Ensure screen readers can interpret the inputs, and implement basic front-end validation (required fields, email format) using pure HTML attributes.

Goal: Test knowledge of form attributes like for, id, required, type="email".

3. Complex Data Table Construction (Medium)

Problem: Implement an HTML table to display financial data. Requirements: 1. A main header spanning multiple columns. 2. A footer row indicating totals. 3. Use proper accessibility tags (caption, thead, tbody, tfoot, th scope="col", th scope="row").

Goal: Validate deeper understanding of table structures beyond basic tr/td combinations.

4. Media Player with Fallbacks & Subtitles (Medium)

Problem: Code a native HTML5 video player that plays "demo.mp4". It must provide a fallback format ("demo.ogg"), show a placeholder image before playing, and include an English closed-caption track.

Goal: Test knowledge of multimedia elements (video, source, track) and their attributes (poster, kind, srclang).

5. Custom Data Attributes & ARIA States (Hard)

Problem: Create the markup for a custom toggle switch (a button acting as a checkbox). You cannot use an actual <input type="checkbox">. You must use a button element. Implement the necessary aria-* attributes and data-* attributes to ensure it is fully accessible and maintains state that JS can later read.

Goal: Test advanced accessibility concepts (ARIA roles/states) and HTML5 custom data attributes.

View Solution Sketch 
<button 
  role="switch" 
  aria-checked="false" 
  data-state="off"
  id="theme-toggler">
  Toggle Dark Mode
</button>
