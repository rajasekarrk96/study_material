jQuery Interview Challenges

Practice these 5 jQuery challenges ranging from Easy to Hard.

1. Selecting and Modifying Multiple Elements (Easy)

Problem: Given a list <ul id="tasks"> containing several <li class="todo"> elements, write a single line of jQuery to select all elements with the class "todo", add a class "completed" to them, and change their text color to gray.

Goal: Test knowledge of basic selector syntax and method chaining.

View Solution Sketch 
// jQuery implicitly loops over all matched elements, 
// so no explicit `each()` loop is needed for this.
$('.todo').addClass('completed').css('color', 'gray');

2. Event Delegation in jQuery (Medium)

Problem: You have a button that dynamically adds new <li class="dynamic-item"> elements to an existing <ul id="list">. Explain why $('.dynamic-item').click(function() {...}) will not work for these newly added elements. Then, write the correct jQuery implementation using Event Delegation so that clicks on both existing and new list items are detected.

Goal: Testing modern jQuery event attachment using the .on() method to solve bubbling issues with dynamically injected nodes.

View Solution Sketch

Direct binding binds handlers only to elements that currently exist in the DOM. To handle dynamically added elements, you must bind to a static parent and use the selector parameter in .on():

$('#list').on('click', '.dynamic-item', function() {
    console.log("Item clicked:", $(this).text());
});

3. document.ready() vs window.onload() (Medium)

Problem: Explain the key difference between jQuery's $(document).ready() and native JavaScript's window.onload event. If a page has many large images, which one fires first and why does it matter?

Goal: Validate deeper understanding of browser rendering life cycle and resource loading.

View Answer

$(document).ready() fires as soon as the DOM hierarchy has been fully constructed (HTML parsed). It does NOT wait for images, iframes, or async scripts to finish downloading.

window.onload waits until the DOM structure AND all external resources (images, stylesheets, frames) are completely loaded.

Conclusion: ready() almost always fires first. It matters because waiting for `onload` on a media-heavy page will make your interactive scripts unresponsive for several seconds while giant images download.

4. Basic AJAX Data Fetching (Medium)

Problem: Write a jQuery AJAX request that performs a GET operation to the hypothetical endpoint /api/metrics. If the request succeeds, it should inject the returned JSON's "total" property into a div with ID "score". If it fails, it should alert "Error".

Goal: Test memory of the $.ajax (or $.get) configuration object and syntax.

View Solution Sketch 
$.ajax({
    url: '/api/metrics',
    type: 'GET',
    dataType: 'json',
    success: function(response) {
        $('#score').text(response.total);
    },
    error: function(xhr, status, error) {
        alert("Error: " + error);
    }
});

5. jQuery `this` Context and $.each() (Hard)

Problem: You have an array of strings: var fruits = ['apple', 'banana', 'cherry'];. Write a jQuery $.each() loop that iterates over this array. For each fruit, it should create an <li> element, append it to an <ul>, and immediately attach a click listener to that specific newly created <li> that alerts its text when clicked, referencing $(this).

Goal: Tests combining array iteration, DOM element creation on the fly, and understanding the context of `this` within a newly created jQuery node.

View Solution Sketch 
var fruits = ['apple', 'banana', 'cherry'];
var $ul = $('<ul></ul>');

$.each(fruits, function(index, fruitName) {
    // Create the node in memory
    var $li = $('<li></li>').text(fruitName);
    
    // Attach listener to the node directly before appending it
    $li.click(function() {
        alert("You clicked: " + $(this).text());
    });
    
    $ul.append($li);
});

$('body').append($ul);
