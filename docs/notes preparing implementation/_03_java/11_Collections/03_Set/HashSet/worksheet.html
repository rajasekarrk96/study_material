Question 1: Get the Number of Elements in a HashSet

Problem Statement

Write a Java program to get the number of elements in a hash set.

Solution

import java.util.HashSet; class _01_ { public static void main(String[] args) { HashSet<String> set = new HashSet<>(); set.add("Red"); set.add("Green"); set.add("Blue"); System.out.println("Number of elements: " + set.size()); } }

Sample Output

Number of elements: 3
Key Concepts:
  • size(): Returns the number of elements in the set
  • add(): Adds an element to the HashSet
  • Unique Elements: HashSet automatically prevents duplicates
  • No Order: Elements are not stored in any particular order
Note: Adding duplicate elements won't increase the size

Question 2: Empty a HashSet

Problem Statement

Write a Java program to empty a hash set.

Solution

import java.util.HashSet; class _02_ { public static void main(String[] args) { HashSet<String> set = new HashSet<>(); set.add("Apple"); set.add("Banana"); set.clear(); System.out.println("HashSet after clearing: " + set); } }

Sample Output

HashSet after clearing: []
Key Concepts:
  • clear(): Removes all elements from the set
  • Empty Set: Size becomes 0 after clearing
  • Reusable: Can add new elements after clearing
  • Efficient: O(n) operation to remove all elements
Use Cases: Resetting data, freeing memory, reusing set objects

Question 3: Check if HashSet is Empty

Problem Statement

Write a Java program to test if a hash set is empty or not.

Solution

import java.util.HashSet; class _03_ { public static void main(String[] args) { HashSet<String> set = new HashSet<>(); System.out.println("Is HashSet empty? " + set.isEmpty()); set.add("Java"); System.out.println("Is HashSet empty? " + set.isEmpty()); } }

Sample Output

Is HashSet empty? true Is HashSet empty? false
Key Concepts:
  • isEmpty(): Returns true if set has no elements
  • Validation: Useful for checking before processing
  • Alternative: Can also use size() == 0
  • Performance: O(1) constant time operation
Best Practice: Use isEmpty() instead of size() == 0 for clarity

Question 4: Convert HashSet to Array

Problem Statement

Write a Java program to convert a hash set to an array.

Solution

import java.util.HashSet; class _04_ { public static void main(String[] args) { HashSet<String> set = new HashSet<>(); set.add("One"); set.add("Two"); set.add("Three"); String[] arr = new String[set.size()]; set.toArray(arr); for (String s : arr) { System.out.println(s); } } }

Sample Output

One Two Three
Key Concepts:
  • toArray(): Converts set to array
  • Array Size: Must match or exceed set size
  • Type Safety: Generic array type matches set type
  • Order: Array order may differ from insertion order
Alternative: Use toArray() without parameter to get Object[]

Question 5: Compare Two HashSets

Problem Statement

Write a Java program to compare two hash sets.

Solution

import java.util.HashSet; class _05_ { public static void main(String[] args) { HashSet<String> set1 = new HashSet<>(); HashSet<String> set2 = new HashSet<>(); set1.add("A"); set1.add("B"); set1.add("C"); set2.add("A"); set2.add("B"); set2.add("D"); System.out.println("Common elements:"); for (String s : set1) { if (set2.contains(s)) System.out.println(s); } } }

Sample Output

Common elements: A B
Key Concepts:
  • contains(): Checks if element exists in set
  • Set Intersection: Finding common elements between sets
  • Fast Lookup: O(1) average time for contains()
  • Set Operations: Can also use retainAll() for intersection
Other Operations: Union (addAll), Difference (removeAll), Subset (containsAll)
