Question 1: String Literal vs new String()

Problem Statement

Write a Java program that demonstrates the difference between string literals and strings created with new keyword. Use == and equals() to compare references and values.

Solution

import java.util.Scanner; class _01_ { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a string: "); String input = sn.nextLine(); // String literals - stored in String Pool String s1 = input; String s2 = input; // new keyword - creates object in heap String s3 = new String(input); String s4 = new String(input); System.out.println("\nString Literals (Pool):"); System.out.println("s1: " + s1); System.out.println("s2: " + s2); System.out.println("s1 == s2: " + (s1 == s2)); // Same reference System.out.println("s1.equals(s2): " + s1.equals(s2)); // Same value System.out.println("\nnew String() (Heap):"); System.out.println("s3: " + s3); System.out.println("s4: " + s4); System.out.println("s3 == s4: " + (s3 == s4)); // Different references System.out.println("s3.equals(s4): " + s3.equals(s4)); // Same value System.out.println("\nLiteral vs new:"); System.out.println("s1 == s3: " + (s1 == s3)); // Different locations System.out.println("s1.equals(s3): " + s1.equals(s3)); // Same value System.out.println("\n✓ Literals share same object in pool"); System.out.println("✓ new String() creates separate objects in heap"); sn.close(); } }

Sample Output

Enter a string: Java String Literals (Pool): s1: Java s2: Java s1 == s2: true s1.equals(s2): true new String() (Heap): s3: Java s4: Java s3 == s4: false s3.equals(s4): true Literal vs new: s1 == s3: false s1.equals(s3): true ✓ Literals share same object in pool ✓ new String() creates separate objects in heap
Key Concepts:
  • String Pool: Literals stored in special memory area
  • Heap Memory: new String() creates objects outside pool
  • == Operator: Compares memory references (addresses)
  • equals() Method: Compares actual string content
Best Practice: Use string literals for better memory efficiency

Question 2: intern() Method Demonstration

Problem Statement

Write a Java program that demonstrates the intern() method. Show how it moves a heap string to the pool and allows reference comparison to work.

Solution

import java.util.Scanner; class _02_ { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a string: "); String input = sn.nextLine(); String s1 = new String(input); // Heap String s2 = input; // Pool String s3 = s1.intern(); // Move to pool System.out.println("\nBefore intern():"); System.out.println("s1 (heap): " + s1); System.out.println("s2 (pool): " + s2); System.out.println("s1 == s2: " + (s1 == s2)); // false System.out.println("\nAfter intern():"); System.out.println("s3 = s1.intern()"); System.out.println("s3 (pool): " + s3); System.out.println("s3 == s2: " + (s3 == s2)); // true System.out.println("s1 == s3: " + (s1 == s3)); // false System.out.println("\n✓ intern() returns reference from pool"); System.out.println("✓ s3 and s2 point to same pool object"); System.out.println("✓ s1 still points to heap object"); sn.close(); } }

Sample Output

Enter a string: Hello Before intern(): s1 (heap): Hello s2 (pool): Hello s1 == s2: false After intern(): s3 = s1.intern() s3 (pool): Hello s3 == s2: true s1 == s3: false ✓ intern() returns reference from pool ✓ s3 and s2 point to same pool object ✓ s1 still points to heap object
Key Concepts:
  • intern() Method: Returns pool reference if exists, adds if not
  • Memory Optimization: Reduces duplicate strings in memory
  • Reference Equality: Interned strings can use == for comparison
  • Original Unchanged: intern() doesn't modify original object
Use Case: Useful when dealing with many duplicate strings

Question 3: String Pool with Concatenation

Problem Statement

Write a Java program that shows how string concatenation affects pool storage. Compare compile-time concatenation (literals) vs runtime concatenation (variables).

Solution

import java.util.Scanner; class _03_ { public static void main(String[] args) { Scanner sn = new Scanner(System.in); // Compile-time concatenation - goes to pool String s1 = "Hello" + "World"; // Compiler combines to "HelloWorld" String s2 = "HelloWorld"; System.out.println("Compile-time concatenation:"); System.out.println("s1 = \"Hello\" + \"World\""); System.out.println("s2 = \"HelloWorld\""); System.out.println("s1 == s2: " + (s1 == s2)); // true // Runtime concatenation - creates new object System.out.print("\nEnter first part: "); String part1 = sn.next(); System.out.print("Enter second part: "); String part2 = sn.next(); String s3 = part1 + part2; // Runtime concatenation String s4 = "HelloWorld"; System.out.println("\nRuntime concatenation:"); System.out.println("s3 = part1 + part2 = " + s3); System.out.println("s4 = \"HelloWorld\""); System.out.println("s3 == s4: " + (s3 == s4)); // false System.out.println("s3.equals(s4): " + s3.equals(s4)); // true if same value // Using intern() to add to pool String s5 = s3.intern(); System.out.println("\nAfter intern():"); System.out.println("s5 = s3.intern()"); System.out.println("s5 == s4: " + (s5 == s4)); // true if same value sn.close(); } }

Sample Output

Compile-time concatenation: s1 = "Hello" + "World" s2 = "HelloWorld" s1 == s2: true Enter first part: Hello Enter second part: World Runtime concatenation: s3 = part1 + part2 = HelloWorld s4 = "HelloWorld" s3 == s4: false s3.equals(s4): true After intern(): s5 = s3.intern() s5 == s4: true
Key Concepts:
  • Compile-Time: Literal concatenation optimized by compiler
  • Runtime: Variable concatenation creates new object
  • Compiler Optimization: "Hello" + "World" becomes "HelloWorld" at compile time
  • Pool Addition: intern() can add runtime strings to pool
Rule: Only literal concatenation goes directly to pool

Question 4: Multiple References to Pool

Problem Statement

Write a Java program that creates multiple string variables with the same literal value. Demonstrate that they all share the same object in the pool, saving memory.

Solution

import java.util.Scanner; class _04_ { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a string: "); String value = sn.nextLine(); // Create multiple variables with same literal String s1 = value; String s2 = value; String s3 = value; String s4 = value; String s5 = value; System.out.println("\nAll variables:"); System.out.println("s1: " + s1); System.out.println("s2: " + s2); System.out.println("s3: " + s3); System.out.println("s4: " + s4); System.out.println("s5: " + s5); System.out.println("\nReference comparisons:"); System.out.println("s1 == s2: " + (s1 == s2)); System.out.println("s2 == s3: " + (s2 == s3)); System.out.println("s3 == s4: " + (s3 == s4)); System.out.println("s4 == s5: " + (s4 == s5)); System.out.println("s1 == s5: " + (s1 == s5)); System.out.println("\n✓ All 5 variables point to SAME object in pool"); System.out.println("✓ Memory saved: 4 duplicate objects avoided"); System.out.println("✓ Only 1 String object created for \"" + value + "\""); sn.close(); } }

Sample Output

Enter a string: Java All variables: s1: Java s2: Java s3: Java s4: Java s5: Java Reference comparisons: s1 == s2: true s2 == s3: true s3 == s4: true s4 == s5: true s1 == s5: true ✓ All 5 variables point to SAME object in pool ✓ Memory saved: 4 duplicate objects avoided ✓ Only 1 String object created for "Java"
Key Concepts:
  • Memory Efficiency: Pool prevents duplicate string objects
  • Reference Sharing: Multiple variables can reference same object
  • Automatic Optimization: JVM handles pool management
  • Space Savings: Significant for applications with many strings
Benefit: Reduces memory footprint in string-heavy applications

Question 5: Pool vs Heap Memory Comparison

Problem Statement

Write a Java program that creates the same string value using both pool (literal) and heap (new) methods. Count and compare the number of objects created in each approach.

Solution

import java.util.Scanner; class _05_ { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a string: "); String value = sn.nextLine(); System.out.print("How many times to create? "); int count = sn.nextInt(); // Method 1: Using literals (Pool) System.out.println("\n=== Method 1: String Literals (Pool) ==="); String[] poolStrings = new String[count]; for (int i = 0; i < count; i++) { poolStrings[i] = value; } System.out.println("Created " + count + " variables"); System.out.println("Checking references:"); boolean allSame = true; for (int i = 1; i < count; i++) { if (poolStrings[0] != poolStrings[i]) { allSame = false; break; } } System.out.println("All point to same object: " + allSame); System.out.println("Objects created: 1"); // Method 2: Using new (Heap) System.out.println("\n=== Method 2: new String() (Heap) ==="); String[] heapStrings = new String[count]; for (int i = 0; i < count; i++) { heapStrings[i] = new String(value); } System.out.println("Created " + count + " variables"); System.out.println("Checking references:"); int uniqueObjects = 0; for (int i = 0; i < count; i++) { boolean unique = true; for (int j = 0; j < i; j++) { if (heapStrings[i] == heapStrings[j]) { unique = false; break; } } if (unique) uniqueObjects++; } System.out.println("All point to same object: false"); System.out.println("Objects created: " + uniqueObjects); System.out.println("\n=== Comparison ==="); System.out.println("Pool approach: 1 object for " + count + " variables"); System.out.println("Heap approach: " + count + " objects for " + count + " variables"); System.out.println("Memory saved: " + (count - 1) + " objects"); sn.close(); } }

Sample Output

Enter a string: Test How many times to create? 5 === Method 1: String Literals (Pool) === Created 5 variables Checking references: All point to same object: true Objects created: 1 === Method 2: new String() (Heap) === Created 5 variables Checking references: All point to same object: false Objects created: 5 === Comparison === Pool approach: 1 object for 5 variables Heap approach: 5 objects for 5 variables Memory saved: 4 objects
Key Concepts:
  • Pool Efficiency: One object serves multiple references
  • Heap Overhead: Each new creates separate object
  • Scalability: Pool benefits increase with more duplicates
  • Performance: Pool reduces garbage collection pressure
Recommendation: Always prefer string literals unless heap object specifically needed
