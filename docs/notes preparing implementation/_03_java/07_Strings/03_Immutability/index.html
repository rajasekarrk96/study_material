String Immutability

String Immutability in Java

Understanding why strings cannot be modified after creation.

What is Immutability?

In Java, strings are immutable - once created, their value cannot be changed. Any operation that appears to modify a string actually creates a new string object.

💡 Key Point: String objects cannot be modified. Methods like toUpperCase(), replace() create NEW strings.

Immutability Demonstration

public class ImmutabilityDemo {
    public static void main(String[] args) {
        String str = "Hello";
        System.out.println("Original: " + str);
        
        // These create NEW strings, don't modify original
        String upper = str.toUpperCase();
        String replaced = str.replace('H', 'J');
        
        System.out.println("After toUpperCase: " + str);  // Still "Hello"
        System.out.println("Upper: " + upper);  // "HELLO"
        System.out.println("Replaced: " + replaced);  // "Jello"
    }
}
Original: Hello
After toUpperCase: Hello
Upper: HELLO
Replaced: Jello

Why Immutability?

✓ Benefits:
• Thread-safe - multiple threads can share strings safely
• Security - strings used in passwords, URLs can't be modified
• String Pool optimization - same strings can be reused
• Hashcode caching - improves performance in HashMaps

⚠️ Drawback:
• Creates many objects when doing frequent modifications
• Use StringBuilder/StringBuffer for heavy string manipulation
