Question 1: Print Array Elements

Problem Statement

Print all elements of an integer array using for-each loop.

Sample Input and Output

Sample Input 1:
Array: {10, 20, 30, 40, 50}

Sample Output 1:
10
20
30
40
50
▼ Show Solution

Solution (Java Program)

public class PrintArray { public static void main(String[] args) { int[] numbers = {10, 20, 30, 40, 50}; for (int num : numbers) { System.out.println(num); } } }

Step-by-Step Explanation

  1. For-each simplifies array traversal: No index variable needed. "for each num in numbers" reads naturally. Compare to for(int i=0; i
  2. Variable holds current element: Each iteration, num automatically gets the next array element. First iteration num=10, second num=20, etc. No manual indexing like numbers[i].
  3. Cannot modify array through for-each: num is a copy of the array element. Changing num doesn't change the array. For-each is read-only iteration.
  4. Perfect for simple traversal: When you just need to process each element sequentially without modifying or needing the index, for-each is the cleanest choice.

Question 2: Sum of Array Elements

Problem Statement

Calculate sum of all elements in an array using for-each loop.

Sample Input and Output

Sample Input 1:
Array: {5, 10, 15, 20}

Sample Output 1:
Sum: 50
▼ Show Solution

Solution (Java Program)

public class ArraySum { public static void main(String[] args) { int[] numbers = {5, 10, 15, 20}; int sum = 0; for (int num : numbers) { sum += num; } System.out.println("Sum: " + sum); } }

Step-by-Step Explanation

  1. Accumulator pattern with for-each: sum starts at 0, then each element is added to it. For-each makes this pattern simpler since we don't care about position, just values.
  2. No risk of index errors: Traditional loops might have i < numbers.length-1 (off-by-one error). For-each automatically handles all elements correctly - no boundary mistakes possible.
  3. Works with any array size: Whether array has 4 elements or 4000, code stays the same. For-each adapts automatically to collection size.
  4. When NOT to use for-each: If you needed index for something (like "sum only even positions"), you'd need traditional for loop. For-each trades flexibility for simplicity.

Question 3: Find Maximum Element

Problem Statement

Find the largest number in an array using for-each loop.

Sample Input and Output

Sample Input 1:
Array: {23, 67, 12, 89, 45}

Sample Output 1:
Maximum: 89
▼ Show Solution

Solution (Java Program)

public class FindMax { public static void main(String[] args) { int[] numbers = {23, 67, 12, 89, 45}; int max = numbers[0]; // Initialize with first element for (int num : numbers) { if (num > max) { max = num; } } System.out.println("Maximum: " + max); } }

Step-by-Step Explanation

  1. Initialize max with first element: Starting with numbers[0] guarantees max is a valid array value. Starting with 0 or Integer.MIN_VALUE works but is less elegant.
  2. Comparison pattern: Each element is compared to current max. If larger, it becomes new max. This max-tracking pattern is fundamental to many algorithms.
  3. Why for-each works here: We only need values, not positions. Don't care WHERE the max is, just WHAT it is. Perfect for-each use case.
  4. Extension to min or other extremes: Same logic works for minimum (if num < min), second largest (track two variables), etc. Understanding the pattern lets you adapt it.

Question 4: Count Specific Elements

Problem Statement

Count how many times a specific number appears in an array using for-each.

Sample Input and Output

Sample Input 1:
Array: {5, 3, 5, 7, 5, 9}
Search for: 5

Sample Output 1:
5 appears 3 times
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class CountOccurrences { public static void main(String[] args) { int[] numbers = {5, 3, 5, 7, 5, 9}; Scanner sn = new Scanner(System.in); System.out.print("Search for: "); int target = sn.nextInt(); int count = 0; for (int num : numbers) { if (num == target) { count++; } } System.out.println(target + " appears " + count + " times"); sn.close(); } }

Step-by-Step Explanation

  1. Counter pattern with conditionals: count increments only when condition matches. Combining for-each with if creates a filtering+counting pattern used everywhere.
  2. For-each + search is common: Looking for elements matching criteria doesn't need index. For-each handles iteration, if handles filtering. Clean separation of concerns.
  3. Works for any type: Same logic works for String arrays (use .equals()), char arrays, object arrays. For-each is type-flexible.
  4. Multiple conditions possible: Could count numbers >10, or even numbers, or numbers divisible by 3. The for-each structure stays same, only the condition changes.

Question 5: Average of Array

Problem Statement

Calculate average of numbers in an array using for-each loop.

Sample Input and Output

Sample Input 1:
Array: {10, 20, 30, 40, 50}

Sample Output 1:
Average: 30.0
▼ Show Solution

Solution (Java Program)

public class ArrayAverage { public static void main(String[] args) { int[] numbers = {10, 20, 30, 40, 50}; int sum = 0; for (int num : numbers) { sum += num; } double average = (double) sum / numbers.length; System.out.println("Average: " + average); } }

Step-by-Step Explanation

  1. Still need .length for count: For-each doesn't give us the count directly. We use numbers.length to get array size for average calculation. This is the trade-off - simpler iteration, but length is separate.
  2. Cast to double for precision: (double) sum prevents integer division. Without it, 50/5 would give 10, not 10.0. For accurate averages, ensure floating-point division.
  3. Two-step calculation pattern: First accumulate sum, then divide by count. Many statistical operations follow this pattern - sum first, then apply final operation.
  4. When for-each shines: This problem demonstrates for-each perfectly - simple element access, no modification, no index needed. When these conditions apply, for-each is always the best choice.
