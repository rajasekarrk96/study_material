Question 1: Social Networking Platform - User Profile Creation

Problem Statement

Build a social networking platform using JDBC. This example demonstrates user profile creation functionality. A complete social networking platform would include:
  • User profile management (create, update, delete)
  • Friend connections and social graph
  • Messaging system
  • Post creation and commenting
  • Notification management
This example focuses on the CREATE operation for user profiles.

Database Setup: Create a database named socialdb with a users table containing columns: id and username.

Solution

import java.sql.*; class _03_ { public static void main(String[] args) { try { Connection con = DriverManager.getConnection( "jdbc:mysql://localhost:3306/socialdb", "root", "password"); PreparedStatement ps = con.prepareStatement("INSERT INTO users VALUES (?, ?)"); ps.setInt(1, 1); ps.setString(2, "john_doe"); ps.executeUpdate(); System.out.println("User Profile Created"); con.close(); } catch (Exception e) { e.printStackTrace(); } } }

Sample Output

User Profile Created

Database Schema

CREATE DATABASE socialdb; USE socialdb; CREATE TABLE users ( id INT PRIMARY KEY, username VARCHAR(50) UNIQUE NOT NULL ); -- After running the program, verify: SELECT * FROM users; -- Output: 1 | john_doe -- Extended schema for a full social network: CREATE TABLE friends ( user_id INT, friend_id INT, status VARCHAR(20), FOREIGN KEY (user_id) REFERENCES users(id), FOREIGN KEY (friend_id) REFERENCES users(id) ); CREATE TABLE posts ( post_id INT PRIMARY KEY AUTO_INCREMENT, user_id INT, content TEXT, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (user_id) REFERENCES users(id) ); CREATE TABLE comments ( comment_id INT PRIMARY KEY AUTO_INCREMENT, post_id INT, user_id INT, comment_text TEXT, FOREIGN KEY (post_id) REFERENCES posts(post_id), FOREIGN KEY (user_id) REFERENCES users(id) );
Key Concepts:
  • CREATE Operation: INSERT statement adds new user profiles to the database
  • Primary Key: id uniquely identifies each user
  • UNIQUE Constraint: Ensures no duplicate usernames
  • PreparedStatement: Safely handles user input to prevent SQL injection
Extension Ideas: Implement friend requests, messaging, posts with likes/comments, and real-time notifications.

Question 2: Online Auction System - Item Listing

Problem Statement

Develop an online auction system using JDBC. This example demonstrates item listing functionality. A complete auction system would include:
  • Item listing and management
  • Bidding system with real-time updates
  • Auction management (start, end, extend)
  • User authentication and authorization
  • Payment processing integration
  • Email notifications for bids and auction results
This example focuses on the CREATE operation for listing items.

Database Setup: Create a database named auctiondb with an items table containing columns: id, name, and price.

Solution

import java.sql.*; class _04_ { public static void main(String[] args) { try { Connection con = DriverManager.getConnection( "jdbc:mysql://localhost:3306/auctiondb", "root", "password"); PreparedStatement ps = con.prepareStatement("INSERT INTO items VALUES (?, ?, ?)"); ps.setInt(1, 201); ps.setString(2, "Laptop"); ps.setDouble(3, 30000); ps.executeUpdate(); System.out.println("Item Listed Successfully"); con.close(); } catch (Exception e) { e.printStackTrace(); } } }

Sample Output

Item Listed Successfully

Database Schema

CREATE DATABASE auctiondb; USE auctiondb; CREATE TABLE items ( id INT PRIMARY KEY, name VARCHAR(100) NOT NULL, price DOUBLE NOT NULL ); -- After running the program, verify: SELECT * FROM items; -- Output: 201 | Laptop | 30000.0 -- Extended schema for a full auction system: CREATE TABLE auctions ( auction_id INT PRIMARY KEY AUTO_INCREMENT, item_id INT, start_time DATETIME, end_time DATETIME, starting_price DOUBLE, current_price DOUBLE, status VARCHAR(20), FOREIGN KEY (item_id) REFERENCES items(id) ); CREATE TABLE bids ( bid_id INT PRIMARY KEY AUTO_INCREMENT, auction_id INT, user_id INT, bid_amount DOUBLE, bid_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (auction_id) REFERENCES auctions(auction_id) ); CREATE TABLE payments ( payment_id INT PRIMARY KEY AUTO_INCREMENT, auction_id INT, user_id INT, amount DOUBLE, payment_status VARCHAR(20), payment_date TIMESTAMP, FOREIGN KEY (auction_id) REFERENCES auctions(auction_id) );
Key Concepts:
  • Data Types:
    • setInt() - For integer item IDs
    • setString() - For item names
    • setDouble() - For prices (supports decimal values)
  • CRUD - Create: INSERT operation adds new auction items
  • Price Handling: DOUBLE data type for accurate price representation
  • Transaction Safety: PreparedStatement prevents injection attacks
Extension Ideas: Add bidding logic, auction timers, winner notification, payment gateway integration, and bid history tracking.
