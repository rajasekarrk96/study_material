Variable Arguments (Varargs)

Varargs in Java

Pass variable number of arguments to methods.

What are Varargs?

Varargs (Variable Arguments) allow you to pass zero or more arguments of the same type to a method without creating an array explicitly.

  • Denoted by three dots (...) after the type
  • Treated as an array inside the method
  • Must be the last parameter
  • Only one varargs parameter per method
💡 Benefit: Makes method calls cleaner when the number of arguments varies.

Syntax

public returnType methodName(dataType... paramName) {
    // paramName is treated as an array
}

Example 1: Sum of Numbers

public class VarargsDemo {
    public int sum(int... numbers) {
        int total = 0;
        for (int num : numbers) {
            total += num;
        }
        return total;
    }
    
    public static void main(String[] args) {
        VarargsDemo demo = new VarargsDemo();
        System.out.println("Sum: " + demo.sum(1, 2, 3));
        System.out.println("Sum: " + demo.sum(10, 20, 30, 40));
        System.out.println("Sum: " + demo.sum());  // Zero arguments
    }
}
Sum: 6
Sum: 100
Sum: 0

Example 2: Print Messages

public class MessagePrinter {
    public void printAll(String... messages) {
        for (String msg : messages) {
            System.out.println(msg);
        }
    }
    
    public static void main(String[] args) {
        MessagePrinter printer = new MessagePrinter();
        printer.printAll("Hello", "World");
        printer.printAll("Java", "is", "awesome");
    }
}

World
Java
is
awesome

Example 3: Mixed Parameters

public class MixedParams {
    public void display(String prefix, int... numbers) {
        System.out.print(prefix + ": ");
        for (int num : numbers) {
            System.out.print(num + " ");
        }
        System.out.println();
    }
    
    public static void main(String[] args) {
        MixedParams mp = new MixedParams();
        mp.display("Numbers", 1, 2, 3, 4, 5);
        mp.display("Values", 10, 20);
    }
}
Numbers: 1 2 3 4 5
Values: 10 20

Varargs Rules

✓ Valid:
• void method(int... nums)
• void method(String str, int... nums)

✗ Invalid:
• void method(int... nums, String str) - Varargs must be last
• void method(int... nums, double... values) - Only one varargs allowed
