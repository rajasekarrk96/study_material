Java Introduction

Introduction to Java Programming

Learn the fundamentals of Java, its history, features, and set up your development environment to write your first Java program.

What is Java?

Java is a high-level, object-oriented, general-purpose programming language created by James Gosling at Sun Microsystems in 1995. It is one of the most popular programming languages used for building scalable, robust applications across various domains.

Key Characteristics:

  • Simple and easy to learn syntax similar to C++
  • Compiled and interpreted language (bytecode execution)
  • Strongly typed and supports object-oriented programming
  • Automatic memory management through garbage collection
  • Rich standard library and framework ecosystem
💡 Know: Java's motto is "Write Once, Run Anywhere" (WORA), which means Java code compiled on one platform can run on any other platform with Java Runtime Environment (JRE).

Why Learn Java?

Java is used in millions of devices and applications worldwide. Here are compelling reasons to learn Java:

  • High Demand: One of the most sought-after programming skills in the job market
  • Versatile: Used for web development, mobile apps (Android), enterprise applications, and more
  • Stable & Mature: 25+ years of development with proven reliability
  • Strong Community: Massive community support, libraries, and frameworks (Spring, Hibernate, etc.)
  • Cross-Platform: Write code once, run anywhere
  • Secure: Built-in security features protect against various attacks
  • Performance: Just-in-Time (JIT) compilation provides excellent runtime performance
✓ Real-World Applications: LinkedIn, Twitter, Uber, Netflix, eBay, and Google all use Java in their backend systems.

Key Features of Java

Java provides several powerful features that make it ideal for building robust applications:

  • Object-Oriented: Everything is an object, promoting modularity and reusability
  • Platform Independent: Bytecode runs on JVM across all platforms
  • Secure: Built-in security features, classloader, bytecode verifier
  • Multi-threaded: Built-in support for concurrent programming
  • Robust: Strong memory management, exception handling, type checking
  • Distributed: Designed for building networked applications
  • Dynamic: Runtime flexibility and reflection capabilities
💡 Key Concept: The JVM (Java Virtual Machine) is the heart of Java's platform independence. It acts as an intermediary between your Java code and the operating system.

Java Development Environment Setup

Required Components:

  • JDK (Java Development Kit): Contains tools for developing Java applications (compiler, debugger, etc.)
  • JRE (Java Runtime Environment): Minimum environment to run Java applications (included in JDK)
  • IDE (Integrated Development Environment): Tools like Eclipse, IntelliJ IDEA, or VS Code

Installation Steps:

1. Download JDK from oracle.com (Java 17 or later recommended)
2. Run the installer and follow the installation wizard
3. Set JAVA_HOME environment variable
4. Add JDK's bin directory to PATH
5. Verify installation:
   - Open Command Prompt/Terminal
   - Type: java -version
   - Type: javac -version

Your First Java Program

Let's write the classic "Hello, World!" program to understand Java's basic structure.

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Program Breakdown:

  • public class HelloWorld - Declares a public class named HelloWorld (filename must match)
  • public static void main(String[] args) - Entry point of the program
  • System.out.println() - Prints text to console followed by a newline

Compilation and Execution:

// Save file as: HelloWorld.java

// Compilation (creates HelloWorld.class)
javac HelloWorld.java

// Execution
java HelloWorld
Hello, World!
✓ Program Structure: Every Java application must have at least one class with a main() method. The main method is where execution begins. The String[] args parameter allows you to pass command-line arguments to your program.
💡 Key Points: Java is case-sensitive. The filename must match the public class name. Method names are case-sensitive too. Always use meaningful class names following naming conventions (PascalCase for classes, camelCase for methods).

Using ASCII in Java:

public class ASCIIExample {
    public static void main(String[] args) {
        // Cast char to int to get ASCII value
        char letter = 'A';
        int ascii = (int) letter;
        System.out.println("ASCII value of 'A': " + ascii);
        
        // Cast int to char to get character from ASCII
        int code = 65;
        char character = (char) code;
        System.out.println("Character from ASCII 65: " + character);
        
        // Print ASCII values of uppercase letters
        System.out.println("\nASCII Table:");
        for (int i = 65; i <= 90; i++) {
            System.out.print((char)i + " ");
        }
    }
}
