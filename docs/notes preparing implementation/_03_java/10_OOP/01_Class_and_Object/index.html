Classes and Objects

Section 1: OOP Fundamentals

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of objects. OOP provides a structured approach to solving problems by modeling real-world entities as objects that interact with each other. Java is designed with OOP as its core principle.

Four Pillars of OOP

  • Encapsulation: Bundling data and methods together, hiding internal details
  • Inheritance: Creating new classes based on existing ones
  • Polymorphism: Objects can take multiple forms
  • Abstraction: Showing only necessary details, hiding complexity

💡 Key Benefit

OOP makes code more modular, reusable, maintainable, and easier to understand. Real-world problems map naturally to objects.

🎯 Real-World Analogy

Think of a class as a blueprint for a car. An object is an actual car built from that blueprint. Each car object has the same structure but can have different properties (color, model, etc.).

Section 2: Class Definition and Structure

A class is a template (blueprint) for creating objects. It defines the structure and behavior:

Class Syntax

public class ClassName { // Member variables (attributes) public String name; public int age; // Methods (behavior) public void display() { System.out.println("Name: " + name); } }

Class Components

Component Description Example
Class Declaration Starts the class definition public class Student
Member Variables Data attributes of the class String name; int rollNumber;
Constructor Initializes objects public Student() { }
Methods Define behavior public void study() { }

Complete Class Example

public class Student { // Member Variables (Attributes) public String name; public int rollNumber; public double gpa; // Constructor (initializes object) public Student(String n, int roll, double g) { name = n; rollNumber = roll; gpa = g; } // Method (Behavior) public void displayInfo() { System.out.println("Name: " + name); System.out.println("Roll: " + rollNumber); System.out.println("GPA: " + gpa); } }

📝 Naming Convention

Class names should be nouns and start with an uppercase letter (PascalCase): Student, Car, Employee, BankAccount

Section 3: Objects and Instantiation

An object is an instance of a class. Creating objects is called instantiation:

Creating Objects

// Syntax: ClassName objectName = new ClassName(); // Example: Student student1 = new Student("John", 101, 3.8); Student student2 = new Student("Alice", 102, 3.9);

Example: Object Creation

public class ObjectCreation { public static void main(String[] args) { // Create first object Student student1 = new Student("John", 101, 3.8); student1.displayInfo(); System.out.println(); // Create second object Student student2 = new Student("Alice", 102, 3.9); student2.displayInfo(); // Accessing object attributes System.out.println("\nStudent 1 Name: " + student1.name); System.out.println("Student 2 GPA: " + student2.gpa); } }

Output

Name: John
Roll: 101
GPA: 3.8

Name: Alice
Roll: 102
GPA: 3.9

Student 1 Name: John
Student 2 GPA: 3.9

Understanding Object Creation Process

// Step 1: Class is loaded in memory // Step 2: new keyword allocates memory for object // Step 3: Constructor is called to initialize the object // Step 4: Reference is assigned to variable Student s = new Student("Bob", 103, 3.7); // s holds a reference to the object in memory

⚠️ Important Concept

Each object created is independent with its own data. Changing one object doesn't affect another object of the same class.

Section 4: Attributes and Methods

Attributes (variables) hold data, while methods perform actions on that data:

Attributes (Member Variables)

public class Car { // Attributes public String color; public String model; public int year; public double speed; // Current speed }

Methods (Behavior)

public class Car { public String color; public String model; // Method 1: Display car info public void display() { System.out.println("Model: " + model); System.out.println("Color: " + color); } // Method 2: Accelerate public void accelerate() { System.out.println("Car is accelerating..."); } // Method 3: Stop public void stop() { System.out.println("Car stopped!"); } }

Complete Example

public class Car { public String brand; public String color; public int year; public Car(String b, String c, int y) { brand = b; color = c; year = y; } public void displayDetails() { System.out.println("Brand: " + brand); System.out.println("Color: " + color); System.out.println("Year: " + year); } public void drive() { System.out.println(brand + " is driving..."); } } public class Main { public static void main(String[] args) { Car car = new Car("Toyota", "Blue", 2020); car.displayDetails(); car.drive(); } }

Output

Brand: Toyota
Color: Blue
Year: 2020
Toyota is driving...

💡 Method Purpose

Methods encapsulate behavior. Instead of directly manipulating data, you call methods to perform actions, which improves code safety and reusability.

Section 5: Real-World Class Examples

Practical examples of designing and using classes for real-world scenarios:

Example 1: Bank Account Class

public class BankAccount { public String accountNumber; public String holder; public double balance; public BankAccount(String num, String h, double b) { accountNumber = num; holder = h; balance = b; } public void deposit(double amount) { balance += amount; System.out.println("Deposited: $" + amount); } public void withdraw(double amount) { if (amount <= balance) { balance -= amount; System.out.println("Withdrawn: $" + amount); } else { System.out.println("Insufficient balance!"); } } public void displayBalance() { System.out.println("Account: " + accountNumber); System.out.println("Balance: $" + balance); } } public class TestBank { public static void main(String[] args) { BankAccount acc = new BankAccount("123456", "John", 1000); acc.displayBalance(); acc.deposit(500); acc.withdraw(200); acc.displayBalance(); } }

Example 2: Book Class

public class Book { public String title; public String author; public int year; public double price; public Book(String t, String a, int y, double p) { title = t; author = a; year = y; price = p; } public void applyDiscount(double discountPercent) { price = price - (price * discountPercent / 100); System.out.println("New price after " + discountPercent + "% discount: $" + price); } public void display() { System.out.println("Title: " + title); System.out.println("Author: " + author); System.out.println("Year: " + year); System.out.println("Price: $" + price); } }

Example 3: Employee Class

public class Employee { public String name; public int empID; public String department; public double salary; public Employee(String n, int id, String dept, double sal) { name = n; empID = id; department = dept; salary = sal; } public void giveSalaryIncrease(double percentIncrease) { salary = salary + (salary * percentIncrease / 100); } public void displayEmployeeInfo() { System.out.println("Name: " + name); System.out.println("ID: " + empID); System.out.println("Department: " + department); System.out.println("Salary: $" + salary); } } public class TestEmployee { public static void main(String[] args) { Employee emp1 = new Employee("John", 101, "IT", 50000); emp1.displayEmployeeInfo(); emp1.giveSalaryIncrease(10); System.out.println("After 10% increase:"); emp1.displayEmployeeInfo(); } }

💡 Design Principles

  • One class should represent one entity
  • Group related attributes and methods
  • Methods should modify or use the class attributes
  • Use meaningful names for clarity
