Question 1: Student Class - Basic Encapsulation

Problem Statement

Create a Student class with the following attributes and methods: • Private attributes name and grade • Public methods setName, getName, setGrade, and getGrade to set and get the values of the attributes • A method displayStudentInfo to print the student's information

Solution

import java.util.Scanner; class _01_ { private String name; private char grade; public void setName(String n) { name = n; } public String getName() { return name; } public void setGrade(char g) { grade = g; } public char getGrade() { return grade; } void displayStudentInfo() { System.out.println("Name: " + name); System.out.println("Grade: " + grade); } public static void main(String[] args) { Scanner sn = new Scanner(System.in); _01_ s = new _01_(); System.out.print("Enter name: "); s.setName(sn.next()); System.out.print("Enter grade: "); s.setGrade(sn.next().charAt(0)); s.displayStudentInfo(); sn.close(); } }

Sample Output

Enter name: Alice Enter grade: A Name: Alice Grade: A
Key Concepts:
  • Private Attributes: name and grade cannot be accessed directly
  • Public Getters/Setters: Controlled access to private data
  • Data Hiding: Internal state is protected from external modification
  • Encapsulation: Data and methods bundled together in a class
Benefits: Validation can be added in setters, data integrity maintained

Question 2: Product Class - Price Management

Problem Statement

Create a Product class with the following attributes and methods: • Private attributes productName and price • Public methods setProductName, getProductName, setPrice, and getPrice to set and get the values of the attributes • A method displayProductInfo to print the product's information

Solution

import java.util.Scanner; class _02_ { private String productName; private double price; public void setProductName(String p) { productName = p; } public String getProductName() { return productName; } public void setPrice(double pr) { price = pr; } public double getPrice() { return price; } void displayProductInfo() { System.out.println("Product Name: " + productName); System.out.println("Price: " + price); } public static void main(String[] args) { Scanner sn = new Scanner(System.in); _02_ p = new _02_(); System.out.print("Enter product name: "); p.setProductName(sn.next()); System.out.print("Enter price: "); p.setPrice(sn.nextDouble()); p.displayProductInfo(); sn.close(); } }

Sample Output

Enter product name: Laptop Enter price: 45000 Product Name: Laptop Price: 45000.0
Key Concepts:
  • Private Fields: productName and price are hidden
  • Controlled Access: Only through getter/setter methods
  • Data Type Safety: price is double, ensuring decimal values
  • Information Hiding: Implementation details are concealed
Enhancement: Can add price validation (e.g., price > 0) in setter

Question 3: Employee Class - Multiple Attributes

Problem Statement

Create an Employee class with the following attributes and methods: • Private attributes employeeId, name, and salary • Public methods setEmployeeId, getEmployeeId, setName, getName, setSalary, and getSalary to set and get the values of the attributes • A method displayEmployeeInfo to print the employee's information

Solution

import java.util.Scanner; class _03_ { private int employeeId; private String name; private double salary; public void setEmployeeId(int id) { employeeId = id; } public int getEmployeeId() { return employeeId; } public void setName(String n) { name = n; } public String getName() { return name; } public void setSalary(double s) { salary = s; } public double getSalary() { return salary; } void displayEmployeeInfo() { System.out.println("Employee ID: " + employeeId); System.out.println("Name: " + name); System.out.println("Salary: " + salary); } public static void main(String[] args) { Scanner sn = new Scanner(System.in); _03_ e = new _03_(); System.out.print("Enter employee ID: "); e.setEmployeeId(sn.nextInt()); System.out.print("Enter name: "); e.setName(sn.next()); System.out.print("Enter salary: "); e.setSalary(sn.nextDouble()); e.displayEmployeeInfo(); sn.close(); } }

Sample Output

Enter employee ID: 101 Enter name: Ravi Enter salary: 50000 Employee ID: 101 Name: Ravi Salary: 50000.0
Key Concepts:
  • Multiple Private Fields: Three attributes encapsulated
  • Getter/Setter Pairs: One pair for each private attribute
  • Data Protection: Salary information is sensitive and protected
  • Consistent Interface: Uniform access pattern for all attributes
Real-World: Employee data requires strict access control

Question 4: BankAccount Class - Transaction Methods

Problem Statement

Create a BankAccount class with the following attributes and methods: • Private attributes accountNumber and balance • Public methods setAccountNumber, getAccountNumber, setBalance, and getBalance to set and get the values of the attributes • A method deposit to add money to the balance • A method withdraw to subtract money from the balance • A method displayAccountInfo to print the account details

Solution

import java.util.Scanner; class _04_ { private String accountNumber; private double balance; public void setAccountNumber(String acc) { accountNumber = acc; } public String getAccountNumber() { return accountNumber; } public void setBalance(double b) { balance = b; } public double getBalance() { return balance; } void deposit(double amt) { balance += amt; } void withdraw(double amt) { balance -= amt; } void displayAccountInfo() { System.out.println("Account Number: " + accountNumber); System.out.println("Balance: " + balance); } public static void main(String[] args) { Scanner sn = new Scanner(System.in); _04_ acc = new _04_(); System.out.print("Enter account number: "); acc.setAccountNumber(sn.next()); System.out.print("Enter initial balance: "); acc.setBalance(sn.nextDouble()); System.out.print("Enter deposit amount: "); acc.deposit(sn.nextDouble()); System.out.print("Enter withdraw amount: "); acc.withdraw(sn.nextDouble()); acc.displayAccountInfo(); sn.close(); } }

Sample Output

Enter account number: ACC1001 Enter initial balance: 10000 Enter deposit amount: 5000 Enter withdraw amount: 2000 Account Number: ACC1001 Balance: 13000.0
Key Concepts:
  • Encapsulated Balance: Balance cannot be directly modified
  • Transaction Methods: deposit() and withdraw() control balance changes
  • Business Logic: Methods can include validation (e.g., sufficient funds)
  • Security: Critical financial data is protected
Enhancement: Add balance validation in withdraw() to prevent negative balance

Question 5: LibraryBook Class - Book Management

Problem Statement

Create a LibraryBook class with the following attributes and methods: • Private attributes bookId, title, and author • Public methods setBookId, getBookId, setTitle, getTitle, setAuthor, and getAuthor to set and get the values of the attributes • A method displayBookInfo to print the book details

Solution

import java.util.Scanner; class _05_ { private int bookId; private String title; private String author; public void setBookId(int id) { bookId = id; } public int getBookId() { return bookId; } public void setTitle(String t) { title = t; } public String getTitle() { return title; } public void setAuthor(String a) { author = a; } public String getAuthor() { return author; } void displayBookInfo() { System.out.println("Book ID: " + bookId); System.out.println("Title: " + title); System.out.println("Author: " + author); } public static void main(String[] args) { Scanner sn = new Scanner(System.in); _05_ b = new _05_(); System.out.print("Enter book ID: "); b.setBookId(sn.nextInt()); System.out.print("Enter title: "); b.setTitle(sn.next()); System.out.print("Enter author: "); b.setAuthor(sn.next()); b.displayBookInfo(); sn.close(); } }

Sample Output

Enter book ID: 501 Enter title: Java Enter author: Gosling Book ID: 501 Title: Java Author: Gosling
Key Concepts:
  • Complete Encapsulation: All three attributes are private
  • Read/Write Access: Both getters and setters provided
  • Data Integrity: Book information protected from unauthorized changes
  • Library System: Models real-world book management
Pattern: Standard encapsulation pattern with getter/setter pairs
