Question 1: Using super to Call Parent Method

Problem Statement

Create a Person class with a method introduce() that prints "I am a person". Then create a Student class that extends Person and overrides the introduce() method to print "I am a student". In the Student class, use the super keyword to call the introduce() method of the Person class. Demonstrate this in the main method.

Solution

class Person { void introduce() { System.out.println("I am a person"); } } class Student extends Person { void introduce() { super.introduce(); System.out.println("I am a student"); } } class _01_ { public static void main(String[] args) { Student s = new Student(); s.introduce(); } }

Sample Output

I am a person I am a student
Key Concepts:
  • Method Overriding: Student overrides Person's introduce() method
  • super.method(): Calls parent class method from child class
  • Extending Functionality: Child adds behavior while keeping parent's
  • Execution Order: Parent method executes first, then child's code
Use Case: Extending parent functionality without losing original behavior

Question 2: Using super to Call Parent Constructor

Problem Statement

Create a Vehicle class with a constructor that initializes the model attribute. Create a Car class that extends Vehicle and has its own constructor which calls the Vehicle constructor using the super keyword. Demonstrate this in the main method.

Solution

class Vehicle { String model; Vehicle(String m) { model = m; } } class Car extends Vehicle { Car(String m) { super(m); } void display() { System.out.println("Model: " + model); } } class _02_ { public static void main(String[] args) { Car c = new Car("Tesla"); c.display(); } }

Sample Output

Model: Tesla
Key Concepts:
  • super(): Calls parent class constructor
  • Constructor Chaining: Child constructor invokes parent constructor
  • First Statement: super() must be first statement in constructor
  • Parameter Passing: Arguments passed from child to parent constructor
Important: Parent constructor initializes inherited members

Question 3: Using super to Access Protected Method

Problem Statement

Create a Guy class with a private attribute name and a protected method getName(). Create an Employee class that extends Guy and adds a method to print the name using the getName() method. Use the super keyword to access the getName() method. Demonstrate this in the main method.

Solution

class Guy { private String name; Guy(String n) { name = n; } protected String getName() { return name; } } class Employee extends Guy { Employee(String n) { super(n); } void printName() { System.out.println("Name: " + super.getName()); } } class _03_ { public static void main(String[] args) { Employee e = new Employee("Ravi"); e.printName(); } }

Sample Output

Name: Ravi
Key Concepts:
  • Protected Access: Child can access protected members of parent
  • super.method(): Explicitly calls parent's protected method
  • Encapsulation: Private field accessed through protected getter
  • Constructor Chain: Employee passes name to Guy constructor
Access Modifiers: Protected allows access in subclasses

Question 4: Using super to Call Overridden Method

Problem Statement

Create a Shape class with a method area() that returns a default area of 0. Create a Rectangle class that extends Shape and overrides the area() method to calculate the area of a rectangle. In the Rectangle class, use super.area() to call the method in the Shape class. Demonstrate this in the main method.

Solution

class Shape { double area() { return 0; } } class Rectangle extends Shape { double length = 5; double width = 4; double area() { System.out.println("Default Area: " + super.area()); return length * width; } } class _04_ { public static void main(String[] args) { Rectangle r = new Rectangle(); System.out.println("Rectangle Area: " + r.area()); } }

Sample Output

Default Area: 0.0 Rectangle Area: 20.0
Key Concepts:
  • Method Overriding: Rectangle overrides Shape's area() method
  • super.area(): Calls parent's version of overridden method
  • Return Values: Both methods return double values
  • Polymorphism: Child provides specific implementation
Pattern: Calling parent method before adding child-specific logic

Question 5: Using super to Access Parent Class Data

Problem Statement

Create a BankAccount class with private attributes accountNumber and balance. Create a SavingsAccount class that extends BankAccount and adds an attribute interestRate. In the SavingsAccount class, use super to access the accountNumber and balance from the BankAccount class. Demonstrate this in the main method.

Solution

class BankAccount { private String accountNumber; private double balance; BankAccount(String acc, double bal) { accountNumber = acc; balance = bal; } protected String getAccountNumber() { return accountNumber; } protected double getBalance() { return balance; } } class SavingsAccount extends BankAccount { double interestRate; SavingsAccount(String acc, double bal, double rate) { super(acc, bal); interestRate = rate; } void displayAccount() { System.out.println("Account Number: " + super.getAccountNumber()); System.out.println("Balance: " + super.getBalance()); System.out.println("Interest Rate: " + interestRate); } } class _05_ { public static void main(String[] args) { SavingsAccount s = new SavingsAccount("SB1001", 5000, 4.5); s.displayAccount(); } }

Sample Output

Account Number: SB1001 Balance: 5000.0 Interest Rate: 4.5
Key Concepts:
  • Private Members: Parent's private fields accessed via protected getters
  • super() Constructor: Initializes parent class attributes
  • super.method(): Calls parent's getter methods
  • Inheritance: Child extends parent with additional attribute
Best Practice: Use getters to access private parent fields
