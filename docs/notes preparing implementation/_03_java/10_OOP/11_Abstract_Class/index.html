Abstract Classes & Interfaces

Abstract Classes & Interfaces

Create flexible, extensible designs with abstraction.

What is Abstraction?

Abstraction hides implementation details and shows only essential features. Achieved through abstract classes and interfaces.

Abstract Classes

abstract class Shape {
    String color;
    
    // Abstract method (no body)
    abstract double area();
    
    // Concrete method
    public void setColor(String c) {
        color = c;
    }
}

class Circle extends Shape {
    double radius;
    
    Circle(double r) {
        radius = r;
    }
    
    // Must implement abstract method
    @Override
    double area() {
        return Math.PI * radius * radius;
    }
}

public class AbstractDemo {
    public static void main(String[] args) {
        Circle c = new Circle(5);
        c.setColor("Red");
        System.out.println("Area: " + c.area());
        System.out.println("Color: " + c.color);
    }
}
Area: 78.53981633974483
Color: Red

Interfaces

interface Drawable {
    void draw();  // Abstract by default
}

interface Colorable {
    void setColor(String color);
}

class Rectangle implements Drawable, Colorable {
    String color;
    
    public void draw() {
        System.out.println("Drawing rectangle");
    }
    
    public void setColor(String c) {
        color = c;
        System.out.println("Color set to " + c);
    }
}

public class InterfaceDemo {
    public static void main(String[] args) {
        Rectangle rect = new Rectangle();
        rect.draw();
        rect.setColor("Blue");
    }
}
Drawing rectangle
Color set to Blue
💡 Difference:
• Abstract class: Can have both abstract and concrete methods
• Interface: All methods abstract (before Java 8)
