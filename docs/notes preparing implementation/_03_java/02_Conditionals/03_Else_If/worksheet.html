Question 1: Grade Calculator

Problem Statement

Write a Java program that assigns grades based on marks. 90+ = A, 80-89 = B, 70-79 = C, 60-69 = D, below 60 = F.

Sample Input and Output

Sample Input 1:
85

Sample Output 1:
Grade: B

Sample Input 2:
55

Sample Output 2:
Grade: F
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class GradeCalculator { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter marks: "); int marks = sn.nextInt(); if (marks >= 90) { System.out.println("Grade: A"); } else if (marks >= 80) { System.out.println("Grade: B"); } else if (marks >= 70) { System.out.println("Grade: C"); } else if (marks >= 60) { System.out.println("Grade: D"); } else { System.out.println("Grade: F"); } sn.close(); } }

Step-by-Step Explanation

  1. Order matters in else-if chains: We check highest values first (90+) because once a condition matches, remaining conditions are skipped. If we checked 60+ first, a score of 85 would incorrectly get grade D.
  2. Overlapping ranges work naturally: Though all conditions use >=, they don't conflict because of the chain structure. When marks=85, it fails >= 90, then succeeds >= 80 and stops.
  3. Only one block executes ever: Unlike separate if statements which would all execute, else-if ensures exactly one grade is assigned. This prevents multiple outputs and logical errors.
  4. Final else catches all remaining cases: Any marks below 60 automatically get F without needing an explicit condition. This ensures complete coverage of all possible inputs.

Question 2: Month to Days

Problem Statement

Write a program that takes a month number (1-12) and prints the number of days in that month (assume non-leap year).

Sample Input and Output

Sample Input 1:
2

Sample Output 1:
28 days

Sample Input 2:
7

Sample Output 2:
31 days
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class MonthDays { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter month (1-12): "); int month = sn.nextInt(); if (month == 2) { System.out.println("28 days"); } else if (month == 4 || month == 6 || month == 9 || month == 11) { System.out.println("30 days"); } else if (month >= 1 && month <= 12) { System.out.println("31 days"); } else { System.out.println("Invalid month"); } sn.close(); } }

Step-by-Step Explanation

  1. Group similar cases with OR operator: Months 4,6,9,11 all have 30 days, so using || (OR) combines them into one condition. This is more efficient than four separate else-if blocks.
  2. Validate input to prevent errors: The final else-if checks if month is between 1-12 before assigning 31 days. This catches invalid inputs like 0 or 15.
  3. Most specific conditions first: February is special (28 days), so we check it first. Then grouped 30-day months, then remaining valid months get 31 days by elimination.
  4. Error handling improves robustness: The final else provides user feedback for invalid input rather than silently failing or crashing. Real-world applications need this defensive programming.

Question 3: Temperature Category

Problem Statement

Categorize temperature: below 0 = "Freezing", 0-15 = "Cold", 16-25 = "Moderate", above 25 = "Hot".

Sample Input and Output

Sample Input 1:
-5

Sample Output 1:
Freezing

Sample Input 2:
30

Sample Output 2:
Hot
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class TemperatureCategory { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter temperature: "); int temp = sn.nextInt(); if (temp < 0) { System.out.println("Freezing"); } else if (temp <= 15) { System.out.println("Cold"); } else if (temp <= 25) { System.out.println("Moderate"); } else { System.out.println("Hot"); } sn.close(); } }

Step-by-Step Explanation

  1. Boundary values require careful operators: Using <= for "Cold" (temp <=15) means 15 is included in Cold category. Using < would put 15 in Moderate. These details matter for correctness.
  2. Sequential checks build implicit ranges: We don't write "temp >= 0 && temp <= 15" for Cold because the if (temp < 0) already failed, so temp must be>= 0. The chain builds ranges automatically.
  3. Code clarity vs explicit conditions: Could write all conditions explicitly with && for clarity, but the chain structure makes them implicit. Understanding this implicit logic is key to reading others' code.
  4. Real-world categorization pattern: Many applications categorize continuous values into discrete ranges - scores to grades, income to tax brackets, age to life stages. Else-if is the standard tool for this.
