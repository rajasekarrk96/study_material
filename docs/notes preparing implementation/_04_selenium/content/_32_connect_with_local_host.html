32. Connect Localhost

📖 Introduction

Sometimes you need to test an application that is running on your own machine (localhost) or a dev server behind a corporate firewall. Selenium handles "localhost" URLs just like any other URL, but there are some nuances.

💡 What You'll Learn

  • Testing apps running on http://localhost:8080
  • Mocking detailed networking conditions
  • Accessing local files directly

🔍 Detailed Explanation

Localhost URLs

If your developer runs the app on their laptop, you can access it via:

  • http://localhost:3000 (React default)
  • http://127.0.0.1:8080 (Java/Spring default)

💻 Code Examples

// 1. Open Local App
driver.get("http://localhost:8080/login");

// 2. Open Local File (HTML)
driver.get("file:///C:/Users/Sedhu/Desktop/test.html");

⚠️ Common Pitfalls

HTTPS Warnings: Localhost often uses self-signed certificates, causing "Not Private" warnings. Use options.setAcceptInsecureCerts(true) to bypass this.

🎯 Real-World Scenario

Practice Scenario

Goal: Test the Website files you are building right now!

  1. Open this very index.html in the browser.
  2. Copy the URL (file:///...).
  3. Write a Selenium script to open it.
  4. Click on the sidebar links and verify they work.

📝 Interview Questions

Q1: Can Selenium test APIs?

Answer: Strictly speaking, no. Selenium is for UI. For APIs, we use RestAssured. However, you can trigger API calls via JS in the browser if needed.
