17. Drag & Drop

📖 Introduction

Modern web apps often allow you to drag elements (like cards in Trello or files in Google Drive). Selenium handles these advanced interactions using the Actions class. This class simulates complex user gestures like click-and-hold, move, and release.

💡 What You'll Learn

  • The Actions class in Selenium
  • How to drag an element and drop it onto another
  • The difference between build() and perform()
  • Drag and Drop by offset (coordinates)

🔍 Detailed Explanation

The Actions Class Sequence

To drag an element, you need to chain multiple commands:

  1. Click and Hold: Grab the source element.
  2. Move to Element: Drag it to the target location.
  3. Release: Let go of the mouse button.
  4. Build: compile the sequence.
  5. Perform: execute it.
Actions act = new Actions(driver);
act.clickAndHold(source).moveToElement(target).release().build().perform();

The Shortcut Method

Selenium provides a convenience method for this common task:

act.dragAndDrop(source, target).perform();

💻 Code Examples

Standard Drag and Drop

WebElement source = driver.findElement(By.id("box-a"));
WebElement target = driver.findElement(By.id("box-b"));

Actions actions = new Actions(driver);

// Method 1: Explicit Chain
actions.clickAndHold(source)
       .moveToElement(target)
       .release()
       .build()
       .perform();

// Method 2: Shortcut
actions.dragAndDrop(source, target).perform();

⚠️ Common Pitfalls

HTML5 Drag and Drop: Sometimes the standard Selenium actions fail on modern HTML5 drag-and-drop elements due to browser implementation details. In such cases, you might need a JavaScript Executor workaround.
