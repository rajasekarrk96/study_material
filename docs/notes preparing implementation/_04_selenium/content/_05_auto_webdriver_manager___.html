05. Auto WebDriver Manager

📖 Introduction

Managing browser drivers manually is a pain. In the past, you had to download a specific version of chromedriver.exe or geckodriver.exe, place it in your project, and set a system property. If your browser automatically updated, your test would break instantly. Selenium Manager (and tools like WebDriverManager) solves this problem permanently.

💡 What You'll Learn

  • The "Old Way" of managing drivers manually
  • How Selenium Manager (Selenium 4.6+) works automatically
  • How to use the external WebDriverManager library (legacy projects)
  • Benefits of automated driver management

🔍 Detailed Explanation

The Old Manual Way (Before Selenium 4.6)

Historically, you had to manually download the driver executable that matched your browser version and set the path in your code:

System.setProperty("webdriver.chrome.driver", "/path/to/chromedriver.exe");
WebDriver driver = new ChromeDriver();

The Problem: When Chrome auto-updates from version 100 to 101, your version 100 driver stops working. You have to manually download the new driver. This is a maintenance nightmare.

The New Way: Selenium Manager (Selenium 4.6+)

Selenium now includes a tool called Selenium Manager out of the box. It detects your installed browser version, downloads the matching driver if needed, and caches it.

  • Zero Configuration: You simply instantiate the driver. Selenium handles the rest.
  • Cross-Browser: Works for Chrome, Firefox, Edge, etc.

WebDriverManager (Boni Garcia)

Before Selenium 4.6, the industry standard was an external library called WebDriverManager. You might see this in older codebases. It does essentially the same thing as Selenium Manager but requires an extra dependency.

💻 Code Examples

Here is how you handle drivers in modern Selenium:

Modern Approach (Selenium 4.6+)

// No setup needed! Selenium Manager handles it automatically.
WebDriver driver = new ChromeDriver();
driver.get("https://google.com");

Legacy Approach (WebDriverManager Library)

// Requires 'io.github.bonigarcia:webdrivermanager' dependency
WebDriverManager.chromedriver().setup();
WebDriver driver = new ChromeDriver();

⚠️ Common Pitfalls

Don't Mix Approaches: If you are using Selenium 4.6+, you do NOT need WebDriverManager.chromedriver().setup(). Remove the Boni Garcia dependency to keep your project clean.

Network Access: The first time you run a test, Selenium Manager needs internet access to download the driver. Ensure your firewall allows it.

🎯 Real-World Scenario

Practice Scenario

Goal: Verify automatic driver management.

  1. Create a new Test class.
  2. Do NOT set any `System.setProperty` specifics.
  3. Do NOT use `WebDriverManager.setup()`.
  4. Simply call `new ChromeDriver()` and open "https://google.com".
  5. Observe the console logs; you might see Selenium Manager activity.

📝 Interview Questions

Q1: What happens if the browser version and driver version do not match?

Answer: You will get a SessionNotCreatedException stating that "This version of ChromeDriver only supports Chrome version X". Automated managers verify this match to prevent errors.

Q2: Is WebDriverManager still relevant after Selenium 4.6?

Answer: Mostly no. Selenium Manager is now the standard. However, WebDriverManager offers advanced features like downloading specific browser versions for testing, which Selenium Manager is still evolving to support fully.
