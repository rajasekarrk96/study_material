25. Screenshots

📖 Introduction

A test automation framework without screenshots is blind. When a test fails overnight, you need to see exactly what the user saw. was it a 404 error? A popup blocking the button? A broken layout? Taking a screenshot at the moment of failure is the most critical debugging aid you have.

💡 What You'll Learn

  • The TakesScreenshot interface
  • Capturing the entire visible viewport
  • Capturing a specific WebElement (Selenium 4 feature)
  • Saving the screenshot to a file

🔍 Detailed Explanation

Typecasting to TakesScreenshot

The standard WebDriver instance needs to be "cast" (converted) to TakesScreenshot to access capture methods.

TakesScreenshot ts = (TakesScreenshot) driver;
File src = ts.getScreenshotAs(OutputType.FILE);

Saving the File

The screenshot starts as a temporary file in memory. You must move it to a permanent location (like a "Screenshots" folder) immediately using FileUtils or standard Java NIO.

💻 Code Examples

Capturing evidence:

Full Page Screenshot

// 1. Capture
File srcFile = ((TakesScreenshot)driver).getScreenshotAs(OutputType.FILE);

// 2. Define Destination
File destFile = new File("./screenshots/failure.png");

// 3. Save (Standard Java)
Files.copy(srcFile.toPath(), destFile.toPath());

Element Screenshot (Selenium 4+)

WebElement logo = driver.findElement(By.id("logo"));
File logoSrc = logo.getScreenshotAs(OutputType.FILE);
// Save logoSrc...

⚠️ Common Pitfalls

Overwrite Risk: If you name every screenshot "error.png", the next run will overwrite the previous one. Always append a Timestamp to the filename (e.g., "error_20231015_1200.png") to keep a history.

🎯 Real-World Scenario

Practice Scenario

Goal: Capture Failure Evidence.

  1. Try to login with bad credentials.
  2. Assert that an error message appears.
  3. Take a screenshot of the error message element itself (not the whole page).
  4. Save it as "login_error.png".

📝 Interview Questions

Q1: What interface allows taking screenshots?

Answer: TakesScreenshot.

Q2: Can you take a screenshot of the full long scrolling page?

Answer: Standard Selenium creates a viewport screenshot (what is visible). For full-page screenshots (scrolling automatically), automated tools like AShot or standard Firefox Driver full-page capabilities are often required.
