31. Extras

📖 Introduction

"It works on my machine" is the most dangerous phrase in software development. Modern automation runs on pipelines, not laptops. This section covers the advanced topics that turn a "script writer" into an "Automation Architect".

💡 What You'll Learn

  • CI/CD (Jenkins/GitHub Actions)
  • Headless Execution
  • Docker & Selenium Grid
  • Parallel Execution

🔍 Detailed Explanation

Jenkins (CI/CD)

A tool that watches your code repository. When you push changes, it automatically:

  1. Downloads the code.
  2. Runs mvn test.
  3. Generates Reports.
  4. Emails results.

Docker

Solves environment issues. You can run your tests inside a "container" that has Chrome and Java pre-installed, guaranteeing it works exactly the same everywhere.

💻 Code Examples

Running via Command Line (for Jenkins)

# Pass browser as a parameter
mvn test -Dbrowser=chrome -Dheadless=true

GitHub Actions Workflow (yaml)

name: Java CI
on: [push]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v2
    - name: Set up JDK 11
      uses: actions/setup-java@v2
      with:
        java-version: '11'
    - name: Test
      run: mvn test

⚠️ Common Pitfalls

Hardcoded Paths: If your code has C:\Users\Sedhu\Desktop\file.txt, it will fail on Jenkins. Always use relative paths.

Practice Scenario

Q1: Why do we use Jenkins?

Answer: To enable Continuous Testing. It allows nightly runs and regression testing on every code commit without human intervention.
