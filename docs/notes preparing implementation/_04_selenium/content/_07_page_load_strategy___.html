07. Page Load Strategy

📖 Introduction

By default, when you tell Selenium to go to a URL, it waits for the entire page to finish loading before it executes the next line of code. This includes all images, stylesheets, and third-party scripts. For modern, heavy websites, this can be agonizingly slow. PageLoadStrategy allows you to take control of this behavior and speed up your tests.

💡 What You'll Learn

  • The three types of Page Load Strategies: NORMAL, EAGER, and NONE
  • How to drastically reduce test execution time
  • Risks associated with aggressive load strategies

🔍 Detailed Explanation

The Three Strategies

  1. NORMAL (Default): Selenium waits for the document.readyState to be "complete". It waits for all resources (images, css, scripts) to load. It is the safest but slowest.
  2. EAGER: Selenium waits for document.readyState to be "interactive". Ideally, the HTML content is loaded and parsed, but images and sub-resources might still be loading. This is significantly faster and usually safe for text interaction.
  3. NONE: Selenium does not wait at all. It returns control immediately after the initial page request. This is the fastest but requires you to manage explicit waits for every single element you want to interact with.

💻 Code Examples

You set the strategy using ChromeOptions:

Using EAGER Strategy

ChromeOptions options = new ChromeOptions();
options.setPageLoadStrategy(PageLoadStrategy.EAGER);

WebDriver driver = new ChromeDriver(options);
long startTime = System.currentTimeMillis();

driver.get("https://www.amazon.com"); // Loads much faster

long endTime = System.currentTimeMillis();
System.out.println("Load Time: " + (endTime - startTime));

⚠️ Common Pitfalls

Stale Element Exceptions: If you use NONE or EAGER, the page might still be rendering when you try to find an element. You MUST use Explicit Waits (WebDriverWait) to ensure elements are present before interacting.

Inconsistent Flakiness: A test might pass on a fast network but fail on a slow one if you use aggressive load strategies without proper synchronization.

🎯 Real-World Scenario

Practice Scenario

Target URL: https://cnn.com (or any media-heavy site)

Goal: compare load times.

  1. Run a script with default (NORMAL) strategy and measure time to open CNN.
  2. Run the same script with EAGER strategy.
  3. Observe the significant time difference.
  4. Try to click a headline immediately after loading with EAGER to see if it works.

📝 Interview Questions

Q1: When should you use PageLoadStrategy.NONE?

Answer: Only when you are an expert at handling synchronization. It is useful for Single Page Applications (SPAs) where the 'load' event doesn't accurately reflect the application state, or when performance is the absolute highest priority.
