30. Reports

📖 Introduction

Running 100 tests is useless if you can't show the results to your manager. "Check the console logs" is not an acceptable answer. Automation frameworks need beautiful, html-based reports with charts, steps, and screenshots. Extent Reports (or Allure) is the standard for this.

💡 What You'll Learn

  • Why TestNG default reports are insufficient
  • Integrating Extent Reports
  • Attaching Screenshots to Reports
  • Logging steps (Pass/Fail/Info)

🔍 Detailed Explanation

Extent Reports Components

  • ExtentReports: The engine that generates the report.
  • ExtentSparkReporter: Defines the UI/Theme of the report.
  • ExtentTest: Represents a single test case in the report.

💻 Code Examples

// 1. Setup
ExtentSparkReporter spark = new ExtentSparkReporter("report.html");
ExtentReports extent = new ExtentReports();
extent.attachReporter(spark);

// 2. Create Test
ExtentTest test = extent.createTest("Login Test");

// 3. Log Steps
test.info("Starting Login");
test.pass("Entered Username");
test.fail("Login Failed");

// 4. Important! Write to file
extent.flush();

⚠️ Common Pitfalls

Flush: If you forget extent.flush() at the end of your suite (usually in @AfterSuite), the report file will be created but will be completely empty (0 bytes).

🎯 Real-World Scenario

Practice Scenario

Goal: Generate a Report.

  1. Add ExtentReports dependency to pom.xml.
  2. Create a Listener class.
  3. On "onTestFailure", capture a screenshot and attach it to the report.
  4. Run the suite and open the HTML file.

📝 Interview Questions

Q1: How do you share results with stakeholders?

Answer: We use Extent Reports hosted on a CI server (Jenkins), or Allure Reports. We email the link or summary to the team.
