Question 1: Move Lowercase Letters to Front

Problem Statement

Write a Java program to move all lowercase letters to the front of a given word while keeping the relative position of all letters (both uppercase and lowercase) the same.

Requirements:
  • All lowercase letters should appear first
  • Followed by all uppercase letters
  • Maintain the original order within each group
Examples:
  • "JaVaProGRam" → "aaroJVPGR"
  • "HeLLo" → "eoHLL"
  • "ABC" → "ABC" (no lowercase)
  • "abc" → "abc" (no uppercase)

Solution

class _03_ { public static void main(String[] args) { String str = "JaVaProGRam"; String lower = ""; String upper = ""; for (char c : str.toCharArray()) { if (Character.isLowerCase(c)) lower += c; else upper += c; } System.out.println(lower + upper); } }

Sample Output

Input: "JaVaProGRam" Output: aaroJVPGR Input: "HeLLo" Output: eoHLL Input: "ProgrammingLanguage" Output: rogramminganguagePLLPL Input: "ABC123" Output: ABC123 Input: "testTEST" Output: testTEST

Algorithm Explanation

Step-by-step process: 1. Initialize two empty strings: lower and upper 2. Convert input string to character array 3. For each character: - If lowercase → append to 'lower' string - If uppercase → append to 'upper' string 4. Concatenate: lower + upper 5. Return result Example: "JaVaProGRam" - Iteration 1: 'J' → upper = "J" - Iteration 2: 'a' → lower = "a" - Iteration 3: 'V' → upper = "JV" - Iteration 4: 'a' → lower = "aa" - Iteration 5: 'P' → upper = "JVP" - Iteration 6: 'r' → lower = "aar" - Iteration 7: 'o' → lower = "aaro" - Iteration 8: 'G' → upper = "JVPG" - Iteration 9: 'R' → upper = "JVPGR" - Iteration 10: 'a' → lower = "aaro" (wait, this is wrong in trace) - Result: "aaro" + "JVPGR" = "aaroJVPGR"

Alternative Approaches

// Approach 1: Using StringBuilder (more efficient) StringBuilder lower = new StringBuilder(); StringBuilder upper = new StringBuilder(); for (char c : str.toCharArray()) { if (Character.isLowerCase(c)) lower.append(c); else upper.append(c); } String result = lower.toString() + upper.toString(); // Approach 2: Using Streams (Java 8+) String result = str.chars() .mapToObj(c -> (char) c) .sorted((a, b) -> { boolean aLower = Character.isLowerCase(a); boolean bLower = Character.isLowerCase(b); if (aLower && !bLower) return -1; if (!aLower && bLower) return 1; return 0; }) .map(String::valueOf) .collect(Collectors.joining()); // Approach 3: Using Regex (extract and concatenate) String lower = str.replaceAll("[^a-z]", ""); String upper = str.replaceAll("[^A-Z]", ""); String result = lower + upper;
Key Concepts:
  • Character.isLowerCase(): Checks if character is lowercase
  • String Concatenation: Building strings by appending characters
  • Relative Order: Maintaining original sequence within groups
  • Two-Pass Algorithm: Separate lowercase and uppercase in one iteration
Performance Note: Using StringBuilder is more efficient than String concatenation in loops due to String immutability.

Question 2: Remove Leading Zeros from IP Address

Problem Statement

Write a Java program to remove leading zeros from a given IP address.

IP Address Format: Four octets separated by dots (e.g., 192.168.001.010)

Requirements:
  • Remove leading zeros from each octet
  • Keep at least one digit (don't remove all zeros)
  • Preserve the dot separators
Examples:
  • "192.168.001.010" → "192.168.1.10"
  • "010.020.030.040" → "10.20.30.40"
  • "000.000.000.000" → "0.0.0.0"

Solution

class _05_ { public static void main(String[] args) { String ip = "192.168.001.010"; String result = ip.replaceAll("\\b0+(\\d)", "$1"); System.out.println(result); } }

Sample Output

Input: "192.168.001.010" Output: 192.168.1.10 Input: "010.020.030.040" Output: 10.20.30.40 Input: "000.000.000.000" Output: 0.0.0.0 Input: "001.002.003.004" Output: 1.2.3.4 Input: "192.168.000.001" Output: 192.168.0.1

Regex Pattern Explanation

Pattern: \\b0+(\\d) Replacement: $1 Breakdown: - \\b : Word boundary (start of number) - 0+ : One or more leading zeros - (\\d) : Capturing group - matches one digit - $1 : Reference to captured digit How it works: 1. \\b ensures we're at the start of a number 2. 0+ matches all leading zeros 3. (\\d) captures the first non-zero digit (or last zero if all zeros) 4. Replace entire match with just the captured digit Examples: - "001" → \\b matches start, 0+ matches "00", (\\d) captures "1" → "1" - "010" → \\b matches start, 0+ matches "0", (\\d) captures "1" → "10" - "000" → \\b matches start, 0+ matches "00", (\\d) captures "0" → "0"

Understanding Word Boundaries

Word Boundary (\\b): - Matches position between word and non-word character - Word characters: [a-zA-Z0-9_] - Non-word characters: spaces, dots, special chars In IP address "192.168.001.010": - \\b matches before: 1, 1, 0 (in 001), 0 (in 010) - Ensures we only match at the start of each octet Without \\b: "1001" → "11" (wrong! would match "00" in middle) With \\b: "1001" → "1001" (correct! no match because "00" not at word boundary)

Alternative Approaches

// Approach 1: Split and process each octet String[] octets = ip.split("\\."); for (int i = 0; i < octets.length; i++) { octets[i] = String.valueOf(Integer.parseInt(octets[i])); } String result = String.join(".", octets); // Approach 2: Using Pattern and Matcher Pattern pattern = Pattern.compile("\\b0+(\\d)"); Matcher matcher = pattern.matcher(ip); String result = matcher.replaceAll("$1"); // Approach 3: Manual iteration StringBuilder result = new StringBuilder(); String[] parts = ip.split("\\."); for (int i = 0; i < parts.length; i++) { result.append(parts[i].replaceFirst("^0+(?=\\d)", "")); if (i < parts.length - 1) result.append("."); }
Key Concepts:
  • Word Boundary \\b: Matches position at start/end of word
  • Quantifier +: One or more occurrences
  • Capturing Group (): Saves matched digit for replacement
  • Backreference $1: Inserts captured group in replacement
Edge Cases:
  • "000" → "0" (keeps one zero)
  • "192.168.1.1" → "192.168.1.1" (no change if no leading zeros)
  • "00192.168.1.1" → "192.168.1.1" (works for first octet too)
Real-World Applications: IP address normalization, log file processing, network configuration
