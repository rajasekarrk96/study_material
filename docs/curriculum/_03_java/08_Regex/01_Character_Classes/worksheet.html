Question 1: Validate String with Specific Characters

Problem Statement

Write a Java program to check whether a string contains only a certain set of characters. The valid characters are:
  • Lowercase letters: a-z
  • Uppercase letters: A-Z
  • Digits: 0-9
  • Underscore: _
This is commonly used for validating usernames, variable names, or identifiers.

Solution

class _01_ { public static void main(String[] args) { String str = "Java_123"; if (str.matches("[a-zA-Z0-9_]+")) System.out.println("Valid string"); else System.out.println("Invalid string"); } }

Sample Output

Input: "Java_123" Output: Valid string Input: "Hello World" Output: Invalid string (contains space) Input: "test@123" Output: Invalid string (contains @) Input: "Valid_Variable_Name_123" Output: Valid string

Regex Pattern Explanation

[a-zA-Z0-9_]+ Breakdown: - [a-zA-Z0-9_] : Character class matching any single character from the set - a-z : Any lowercase letter - A-Z : Any uppercase letter - 0-9 : Any digit - _ : Underscore character - + : Quantifier - one or more occurrences Alternative: \\w+ (predefined class for word characters)
Key Concepts:
  • Character Class []: Matches any single character from the set
  • Range a-z: Matches any character from 'a' to 'z'
  • matches() method: Checks if entire string matches the pattern
  • Quantifier +: Requires at least one character
Common Use Cases: Username validation, identifier checking, alphanumeric verification

Question 2: Find Word with 'g' in Middle

Problem Statement

Write a Java program that matches a word containing 'g', but not at the start or end of the word. The letter 'g' must appear somewhere in the middle of the word.

Examples:
  • ✅ "string" - 'g' is in the middle
  • ✅ "programming" - 'g' appears in the middle
  • ❌ "go" - 'g' is at the start
  • ❌ "big" - 'g' is at the end
  • ❌ "test" - no 'g' present

Solution

class _02_ { public static void main(String[] args) { String word = "string"; if (word.matches(".+g.+")) System.out.println("Match found"); else System.out.println("No match"); } }

Sample Output

Input: "string" Output: Match found Input: "programming" Output: Match found Input: "go" Output: No match Input: "big" Output: No match Input: "regex" Output: No match

Regex Pattern Explanation

.+g.+ Breakdown: - .+ : One or more of any character (before 'g') - g : Literal character 'g' - .+ : One or more of any character (after 'g') This ensures: 1. At least one character before 'g' 2. The letter 'g' itself 3. At least one character after 'g' Therefore, 'g' cannot be at the start or end!
Key Concepts:
  • Dot (.): Matches any single character except newline
  • Quantifier +: One or more occurrences
  • Pattern .+g.+: Ensures 'g' has characters on both sides
  • Position Matching: Pattern implicitly enforces position constraints
Alternative Patterns:
  • ^.+g.+$ - Explicit start (^) and end ($) anchors
  • [^g]+g[^g]+ - More specific (non-g characters on both sides)
