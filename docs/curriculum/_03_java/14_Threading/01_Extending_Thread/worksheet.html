Question 1: Even and Odd Numbers with Two Threads

Problem Statement

Write a Java program that creates two threads to find and print even and odd numbers from 1 to 20. One thread should print all even numbers, and the other should print all odd numbers.

Solution

class EvenThread extends Thread { public void run() { for (int i = 2; i <= 20; i += 2) System.out.println("Even: " + i); } } class OddThread extends Thread { public void run() { for (int i = 1; i <= 20; i += 2) System.out.println("Odd: " + i); } } class _02_ { public static void main(String[] args) { new EvenThread().start(); new OddThread().start(); } }

Sample Output

Even: 2 Odd: 1 Even: 4 Odd: 3 Even: 6 Odd: 5 ... (Output order may vary due to thread scheduling)
Key Concepts:
  • Extending Thread: Both classes extend Thread and override run() method
  • start() Method: Starts a new thread and calls run() method
  • Concurrent Execution: Both threads run simultaneously
  • Thread Scheduling: Output order is non-deterministic
Note: The output order depends on the thread scheduler and may vary between runs.

Question 2: Concurrent Web Crawler Simulation

Problem Statement

Write a Java program to implement a concurrent web crawler that crawls multiple websites simultaneously using threads. This is a simplified simulation that demonstrates how multiple threads can process different websites concurrently.

Solution

class Crawler extends Thread { String site; Crawler(String s) { site = s; } public void run() { System.out.println("Crawling: " + site); } } class _03_ { public static void main(String[] args) { new Crawler("google.com").start(); new Crawler("openai.com").start(); new Crawler("github.com").start(); } }

Sample Output

Crawling: google.com Crawling: openai.com Crawling: github.com (Order may vary)
Key Concepts:
  • Parameterized Threads: Each thread receives a website URL via constructor
  • Concurrent Processing: Multiple websites crawled simultaneously
  • Real-World Application: Web crawlers use multithreading for efficiency
Extension Ideas: Add actual HTTP requests, implement depth-limited crawling, handle robots.txt

Question 3: join() Method - Wait for Thread Completion

Problem Statement

Write a Java program that demonstrates how to use the join() method to wait for a thread to complete its execution. The main thread should wait for the worker thread to finish before printing "Thread completed".

Solution

class JoinThread extends Thread { public void run() { for (int i = 1; i <= 5; i++) System.out.println(i); } } class _05_ { public static void main(String[] args) throws Exception { JoinThread t = new JoinThread(); t.start(); t.join(); System.out.println("Thread completed"); } }

Sample Output

1 2 3 4 5 Thread completed
Key Concepts:
  • join() Method: Makes calling thread wait until the thread completes
  • Thread Synchronization: Ensures sequential execution when needed
  • InterruptedException: join() throws this exception, must be handled
  • Use Case: When you need to wait for thread completion before proceeding
Without join(): "Thread completed" might print before the numbers.

Question 4: sleep() Method - Pause Thread Execution

Problem Statement

Write a Java program that demonstrates how to use the sleep() method to pause the execution of a thread. Print numbers 1 to 5 with a 1-second delay between each number.

Solution

class _06_ { public static void main(String[] args) throws Exception { for (int i = 1; i <= 5; i++) { System.out.println(i); Thread.sleep(1000); } } }

Sample Output

1 (1 second pause) 2 (1 second pause) 3 (1 second pause) 4 (1 second pause) 5
Key Concepts:
  • sleep() Method: Pauses thread execution for specified milliseconds
  • Static Method: Thread.sleep() is a static method
  • InterruptedException: Must be caught or declared
  • Use Cases: Delays, polling intervals, rate limiting
Important: sleep(1000) = 1 second (1000 milliseconds)

Question 5: yield() Method - Give Other Threads a Chance

Problem Statement

Write a Java program that demonstrates how to use the yield() method to give other threads a chance to run. Create two threads that print their names, using yield() to encourage thread switching.

Solution

class YieldThread extends Thread { public void run() { for (int i = 1; i <= 5; i++) { System.out.println(Thread.currentThread().getName()); Thread.yield(); } } } class _07_ { public static void main(String[] args) { YieldThread t1 = new YieldThread(); YieldThread t2 = new YieldThread(); t1.start(); t2.start(); } }

Sample Output

Thread-0 Thread-1 Thread-0 Thread-1 Thread-0 Thread-1 ... (Output may vary - yield() is a hint, not guaranteed)
Key Concepts:
  • yield() Method: Hints to scheduler that current thread is willing to yield CPU
  • Thread Scheduling: Scheduler may or may not honor the hint
  • Cooperative Multitasking: Threads voluntarily give up CPU time
  • Use Case: Preventing thread starvation in compute-intensive tasks
Note: yield() is rarely used in modern Java applications.

Question 6: isAlive() Method - Check Thread Status

Problem Statement

Write a Java program that demonstrates how to use the isAlive() method to check whether a thread is still running. Check the thread status before and after starting it.

Solution

class AliveThread extends Thread { public void run() { System.out.println("Thread running"); } } class _08_ { public static void main(String[] args) { AliveThread t = new AliveThread(); System.out.println(t.isAlive()); t.start(); System.out.println(t.isAlive()); } }

Sample Output

false Thread running true (or) false true Thread running
Key Concepts:
  • isAlive() Method: Returns true if thread has been started and not yet died
  • Thread States: NEW → RUNNABLE → TERMINATED
  • Before start(): Thread is in NEW state, isAlive() returns false
  • After start(): Thread is RUNNABLE or RUNNING, isAlive() returns true
Use Case: Checking if a thread has completed its task.
