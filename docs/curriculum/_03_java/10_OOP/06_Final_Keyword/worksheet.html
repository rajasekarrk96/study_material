Question 1: Final Variable - Constants

Problem Statement

Create a Constants class with a final variable PI initialized to 3.14. Attempt to change the value of PI in the main method and show that it results in a compile-time error.

Solution

class _01_ { public static void main(String[] args) { final double PI = 3.14; System.out.println("PI = " + PI); // PI = 3.14159; // ❌ Compile-time error: cannot assign a value to final variable PI } }

Sample Output

PI = 3.14
Key Concepts:
  • Final Variable: Once assigned, value cannot be changed
  • Compile-Time Error: Attempting to reassign causes compilation failure
  • Constants: Final variables are typically used for constants
  • Naming Convention: Constants are usually named in UPPERCASE
Error Message: "cannot assign a value to final variable PI"

Question 2: Final Method - Prevent Overriding

Problem Statement

Create an Animal class with a final method sleep() that prints "Animal is sleeping". Create a Dog class that extends Animal and tries to override the sleep() method. Demonstrate that overriding the final method is not allowed.

Solution

class Animal { final void sleep() { System.out.println("Animal is sleeping"); } } class Dog extends Animal { // void sleep() { // ❌ Compile-time error // System.out.println("Dog sleeping"); // } } class _02_ { public static void main(String[] args) { Dog d = new Dog(); d.sleep(); } }

Sample Output

Animal is sleeping
Key Concepts:
  • Final Method: Cannot be overridden in subclasses
  • Inheritance: Dog inherits sleep() but cannot override it
  • Method Binding: Final methods are bound at compile-time
  • Use Case: Critical methods that must maintain consistent behavior
Error Message: "sleep() in Dog cannot override sleep() in Animal; overridden method is final"

Question 3: Final Class - Prevent Inheritance

Problem Statement

Create a FinalClass with a final class modifier and attempt to extend it in another class. Show that extending a final class is not allowed.

Solution

final class FinalClass { void display() { System.out.println("Final class"); } } // class Child extends FinalClass { // ❌ Compile-time error // } class _03_ { public static void main(String[] args) { FinalClass f = new FinalClass(); f.display(); } }

Sample Output

Final class
Key Concepts:
  • Final Class: Cannot be extended or inherited
  • Immutability: Often used for immutable classes like String
  • Security: Prevents modification through inheritance
  • Examples: String, Integer, Math classes are final
Error Message: "cannot inherit from final FinalClass"

Question 4: Final Method Parameters

Problem Statement

Create a Calculator class with a method add(final int a, final int b) that adds two numbers. Show that the final parameters cannot be modified within the method.

Solution

class Calculator { int add(final int a, final int b) { // a = a + 1; // ❌ Compile-time error // b = b + 1; // ❌ Compile-time error return a + b; } } class _04_ { public static void main(String[] args) { Calculator c = new Calculator(); System.out.println("Sum = " + c.add(5, 10)); } }

Sample Output

Sum = 15
Key Concepts:
  • Final Parameters: Method parameters declared as final cannot be reassigned
  • Read-Only: Parameters can be read but not modified
  • Safety: Prevents accidental modification of input values
  • Best Practice: Useful for ensuring parameter integrity
Error Message: "final parameter a cannot be assigned"

Question 5: Final Method in Base Class

Problem Statement

Create a base class Base with a final method show() and a derived class Derived that tries to override the show() method. Demonstrate that overriding a final method is not permitted.

Solution

class Base { final void show() { System.out.println("Base show method"); } } class Derived extends Base { // void show() { // ❌ Compile-time error // System.out.println("Derived show"); // } } class _05_ { public static void main(String[] args) { Derived d = new Derived(); d.show(); } }

Sample Output

Base show method
Key Concepts:
  • Method Finality: Final methods in base class cannot be overridden
  • Polymorphism Limitation: Final methods prevent runtime polymorphism
  • Inheritance Chain: All subclasses use the base class implementation
  • Design Decision: Use final when method behavior must remain consistent
Error Message: "show() in Derived cannot override show() in Base; overridden method is final"
