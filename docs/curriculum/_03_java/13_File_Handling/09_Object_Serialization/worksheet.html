Question 1: Basic Object Serialization

Problem Statement

Write a Java program to serialize a Student object to a file. The Student class should have fields for id, name, and marks. Demonstrate how to save an object's state to a file.

Solution

import java.io.*; class Student implements Serializable { int id; String name; double marks; Student(int id, String name, double marks) { this.id = id; this.name = name; this.marks = marks; } } class SerializeStudent { public static void main(String[] args) { Student s = new Student(101, "Alice", 95.5); try (ObjectOutputStream oos = new ObjectOutputStream( new FileOutputStream("student.ser"))) { oos.writeObject(s); System.out.println("Student object serialized successfully"); } catch (Exception e) { e.printStackTrace(); } } }

Sample Output

Student object serialized successfully (Creates a file: student.ser containing the serialized object)
Key Concepts:
  • Serializable Interface: Marker interface (no methods) that enables serialization
  • ObjectOutputStream: Writes objects to an output stream
  • writeObject(): Serializes the object to the stream
  • .ser Extension: Common convention for serialized files
Requirements:
  • Class must implement Serializable
  • All non-transient fields must be serializable
  • Use try-with-resources for automatic stream closing

Question 2: Object Deserialization

Problem Statement

Write a Java program to deserialize the Student object from the file created in Question 1. Read the object from the file and display its data to verify successful deserialization.

Solution

import java.io.*; class Student implements Serializable { int id; String name; double marks; Student(int id, String name, double marks) { this.id = id; this.name = name; this.marks = marks; } void display() { System.out.println("ID: " + id); System.out.println("Name: " + name); System.out.println("Marks: " + marks); } } class DeserializeStudent { public static void main(String[] args) { try (ObjectInputStream ois = new ObjectInputStream( new FileInputStream("student.ser"))) { Student s = (Student) ois.readObject(); System.out.println("Student object deserialized successfully\n"); s.display(); } catch (Exception e) { e.printStackTrace(); } } }

Sample Output

Student object deserialized successfully ID: 101 Name: Alice Marks: 95.5
Key Concepts:
  • ObjectInputStream: Reads objects from an input stream
  • readObject(): Deserializes object from stream (returns Object)
  • Type Casting: Must cast to appropriate type after reading
  • ClassNotFoundException: Thrown if class definition not found
Important: The class definition must be available during deserialization

Question 3: Using transient Keyword

Problem Statement

Write a Java program demonstrating the transient keyword. Create a User class with username, password, and sessionId fields. Mark password and sessionId as transient so they are not serialized.

Solution

import java.io.*; class User implements Serializable { String username; transient String password; // Not serialized transient String sessionId; // Not serialized int loginCount; User(String username, String password, String sessionId, int loginCount) { this.username = username; this.password = password; this.sessionId = sessionId; this.loginCount = loginCount; } void display() { System.out.println("Username: " + username); System.out.println("Password: " + password); System.out.println("Session ID: " + sessionId); System.out.println("Login Count: " + loginCount); } } class TransientDemo { public static void main(String[] args) { User user = new User("john_doe", "secret123", "ABC-XYZ-789", 5); // Serialize try (ObjectOutputStream oos = new ObjectOutputStream( new FileOutputStream("user.ser"))) { oos.writeObject(user); System.out.println("Before Serialization:"); user.display(); } catch (Exception e) { e.printStackTrace(); } // Deserialize try (ObjectInputStream ois = new ObjectInputStream( new FileInputStream("user.ser"))) { User deserializedUser = (User) ois.readObject(); System.out.println("\nAfter Deserialization:"); deserializedUser.display(); } catch (Exception e) { e.printStackTrace(); } } }

Sample Output

Before Serialization: Username: john_doe Password: secret123 Session ID: ABC-XYZ-789 Login Count: 5 After Deserialization: Username: john_doe Password: null Session ID: null Login Count: 5
Key Concepts:
  • transient Keyword: Prevents field from being serialized
  • Default Values: Transient fields get default values after deserialization (null for objects, 0 for numbers)
  • Security: Use transient for sensitive data like passwords
  • Session Data: Temporary data that shouldn't be persisted
Common Use Cases:
  • Passwords and sensitive information
  • Cached or derived values
  • Thread references
  • Database connections

Question 4: Serializing Collections

Problem Statement

Write a Java program to serialize an ArrayList of Product objects. Create a Product class with id, name, and price fields. Serialize a list of products to a file and then deserialize it back.

Solution

import java.io.*; import java.util.ArrayList; class Product implements Serializable { int id; String name; double price; Product(int id, String name, double price) { this.id = id; this.name = name; this.price = price; } public String toString() { return "Product[ID=" + id + ", Name=" + name + ", Price=$" + price + "]"; } } class SerializeCollection { public static void main(String[] args) { ArrayList products = new ArrayList<>(); products.add(new Product(1, "Laptop", 999.99)); products.add(new Product(2, "Mouse", 25.50)); products.add(new Product(3, "Keyboard", 75.00)); // Serialize ArrayList try (ObjectOutputStream oos = new ObjectOutputStream( new FileOutputStream("products.ser"))) { oos.writeObject(products); System.out.println("Products serialized successfully\n"); } catch (Exception e) { e.printStackTrace(); } // Deserialize ArrayList try (ObjectInputStream ois = new ObjectInputStream( new FileInputStream("products.ser"))) { @SuppressWarnings("unchecked") ArrayList deserializedProducts = (ArrayList) ois.readObject(); System.out.println("Products deserialized successfully:\n"); for (Product p : deserializedProducts) { System.out.println(p); } } catch (Exception e) { e.printStackTrace(); } } }

Sample Output

Products serialized successfully Products deserialized successfully: Product[ID=1, Name=Laptop, Price=$999.99] Product[ID=2, Name=Mouse, Price=$25.5] Product[ID=3, Name=Keyboard, Price=$75.0]
Key Concepts:
  • Collection Serialization: ArrayList implements Serializable
  • Generic Type Casting: Need to cast to ArrayList<Product>
  • @SuppressWarnings: Suppress unchecked cast warning
  • All Elements Must Be Serializable: Product class must implement Serializable
Serializable Collections: ArrayList, LinkedList, HashMap, HashSet, TreeMap, TreeSet

Question 5: serialVersionUID and Version Control

Problem Statement

Write a Java program demonstrating the use of serialVersionUID for version control. Show what happens when you modify a class after serialization and the importance of maintaining a consistent serialVersionUID.

Solution

import java.io.*; class Employee implements Serializable { // Explicit serialVersionUID for version control private static final long serialVersionUID = 1L; int empId; String name; double salary; Employee(int empId, String name, double salary) { this.empId = empId; this.name = name; this.salary = salary; } void display() { System.out.println("Employee ID: " + empId); System.out.println("Name: " + name); System.out.println("Salary: $" + salary); } } class SerialVersionUIDDemo { public static void main(String[] args) { // Serialize Employee emp = new Employee(101, "Bob Smith", 75000); try (ObjectOutputStream oos = new ObjectOutputStream( new FileOutputStream("employee.ser"))) { oos.writeObject(emp); System.out.println("Employee serialized with serialVersionUID: " + Employee.serialVersionUID); } catch (Exception e) { e.printStackTrace(); } // Deserialize try (ObjectInputStream ois = new ObjectInputStream( new FileInputStream("employee.ser"))) { Employee deserializedEmp = (Employee) ois.readObject(); System.out.println("\nEmployee deserialized successfully:\n"); deserializedEmp.display(); } catch (Exception e) { e.printStackTrace(); } } }

Sample Output

Employee serialized with serialVersionUID: 1 Employee deserialized successfully: Employee ID: 101 Name: Bob Smith Salary: $75000.0

Understanding serialVersionUID

// Without explicit serialVersionUID (NOT RECOMMENDED) class Employee implements Serializable { // JVM generates serialVersionUID automatically // Changes to class structure will change the UID int empId; String name; } // With explicit serialVersionUID (RECOMMENDED) class Employee implements Serializable { private static final long serialVersionUID = 1L; int empId; String name; // Can add new fields without breaking compatibility double salary; // Added later } // What happens without serialVersionUID: // 1. Serialize Employee with fields: empId, name // 2. Add new field: salary // 3. Try to deserialize → InvalidClassException! // With serialVersionUID: // 1. Serialize Employee (serialVersionUID = 1L) // 2. Add new field: salary (keep serialVersionUID = 1L) // 3. Deserialize → Works! (new field gets default value)
Key Concepts:
  • serialVersionUID: Unique identifier for Serializable class version
  • Version Control: Ensures compatibility between serialized and deserialized versions
  • InvalidClassException: Thrown when serialVersionUID doesn't match
  • Best Practice: Always declare explicit serialVersionUID
When to Change serialVersionUID:
  • Incompatible changes (removing fields, changing field types)
  • When you want to break backward compatibility
When to Keep Same serialVersionUID:
  • Adding new fields (backward compatible)
  • Adding methods
  • Changing method implementations
