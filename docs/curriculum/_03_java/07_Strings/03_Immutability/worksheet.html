Question 1: Demonstrating String Immutability

Problem Statement

Write a Java program that demonstrates string immutability by: • Creating a string • Applying various string methods (toUpperCase, toLowerCase, replace, concat) • Showing that the original string remains unchanged

Solution

import java.util.Scanner; class _01_ { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a string: "); String original = sn.nextLine(); System.out.println("\nOriginal string: " + original); String upper = original.toUpperCase(); System.out.println("After toUpperCase(): " + original); System.out.println("New string (upper): " + upper); String lower = original.toLowerCase(); System.out.println("\nAfter toLowerCase(): " + original); System.out.println("New string (lower): " + lower); String replaced = original.replace('a', 'X'); System.out.println("\nAfter replace('a', 'X'): " + original); System.out.println("New string (replaced): " + replaced); String concatenated = original.concat(" World"); System.out.println("\nAfter concat(\" World\"): " + original); System.out.println("New string (concatenated): " + concatenated); System.out.println("\n✓ Original string never changed: " + original); sn.close(); } }

Sample Output

Enter a string: Java Original string: Java After toUpperCase(): Java New string (upper): JAVA After toLowerCase(): Java New string (lower): java After replace('a', 'X'): Java New string (replaced): JXvX After concat(" World"): Java New string (concatenated): Java World ✓ Original string never changed: Java
Key Concepts:
  • Immutability: Original string object never changes
  • New Objects: Each method creates a new String object
  • Return Values: Methods return new strings, don't modify original
  • Memory: Multiple string objects created in memory
Important: str.toUpperCase() doesn't change str, it returns a NEW string

Question 2: String vs StringBuilder Performance

Problem Statement

Write a Java program that compares string concatenation using: • String (immutable) - creates new objects • StringBuilder (mutable) - modifies same object Show the difference in object creation.

Solution

import java.util.Scanner; class _02_ { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter number of concatenations: "); int n = sn.nextInt(); // Using String (immutable) String str = ""; System.out.println("\nUsing String (immutable):"); for (int i = 1; i <= n; i++) { str = str + i + " "; // Creates new String object each time System.out.println("Step " + i + ": " + str); } // Using StringBuilder (mutable) StringBuilder sb = new StringBuilder(); System.out.println("\nUsing StringBuilder (mutable):"); for (int i = 1; i <= n; i++) { sb.append(i).append(" "); // Modifies same object System.out.println("Step " + i + ": " + sb.toString()); } System.out.println("\n✓ String creates " + n + " new objects"); System.out.println("✓ StringBuilder modifies 1 object " + n + " times"); sn.close(); } }

Sample Output

Enter number of concatenations: 3 Using String (immutable): Step 1: 1 Step 2: 1 2 Step 3: 1 2 3 Using StringBuilder (mutable): Step 1: 1 Step 2: 1 2 Step 3: 1 2 3 ✓ String creates 3 new objects ✓ StringBuilder modifies 1 object 3 times
Key Concepts:
  • Performance Impact: String concatenation creates many objects
  • StringBuilder Advantage: Mutable, modifies same object
  • Memory Efficiency: StringBuilder uses less memory for loops
  • Best Practice: Use StringBuilder for repeated concatenations
Rule: Use String for simple operations, StringBuilder for loops/heavy manipulation

Question 3: Reference vs Value in Strings

Problem Statement

Write a Java program that demonstrates: • Creating two string variables pointing to same object • Modifying one variable creates a new object • The other variable still points to original object Use == and equals() to show the difference.

Solution

import java.util.Scanner; class _03_ { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a string: "); String str1 = sn.nextLine(); String str2 = str1; // Both point to same object System.out.println("\nInitial state:"); System.out.println("str1: " + str1); System.out.println("str2: " + str2); System.out.println("str1 == str2: " + (str1 == str2)); // true (same reference) System.out.println("str1.equals(str2): " + str1.equals(str2)); // true (same value) // "Modify" str1 (actually creates new object) str1 = str1.toUpperCase(); System.out.println("\nAfter str1.toUpperCase():"); System.out.println("str1: " + str1); System.out.println("str2: " + str2); // Unchanged! System.out.println("str1 == str2: " + (str1 == str2)); // false (different references) System.out.println("str1.equals(str2): " + str1.equals(str2)); // false (different values) System.out.println("\n✓ str2 unchanged because strings are immutable"); System.out.println("✓ str1 now points to a NEW string object"); sn.close(); } }

Sample Output

Enter a string: hello Initial state: str1: hello str2: hello str1 == str2: true str1.equals(str2): true After str1.toUpperCase(): str1: HELLO str2: hello str1 == str2: false str1.equals(str2): false ✓ str2 unchanged because strings are immutable ✓ str1 now points to a NEW string object
Key Concepts:
  • Reference Assignment: str2 = str1 copies reference, not value
  • Immutability Protection: Changing str1 doesn't affect str2
  • == vs equals(): == checks reference, equals() checks value
  • New Object Creation: toUpperCase() creates new object for str1
Safety: Immutability prevents unintended modifications through references

Question 4: Method Parameter Immutability

Problem Statement

Write a Java program with a method that attempts to modify a string parameter. Demonstrate that the original string in main() remains unchanged because strings are immutable and Java is pass-by-value.

Solution

import java.util.Scanner; class _04_ { public static void modifyString(String str) { System.out.println("Inside method - before: " + str); str = str.toUpperCase(); // Creates new object, local variable points to it System.out.println("Inside method - after: " + str); } public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a string: "); String original = sn.nextLine(); System.out.println("\nBefore method call: " + original); modifyString(original); System.out.println("After method call: " + original); System.out.println("\n✓ Original string unchanged!"); System.out.println("✓ Method received a COPY of the reference"); System.out.println("✓ Method's modification created NEW object"); System.out.println("✓ Original reference still points to original object"); sn.close(); } }

Sample Output

Enter a string: java Before method call: java Inside method - before: java Inside method - after: JAVA After method call: java ✓ Original string unchanged! ✓ Method received a COPY of the reference ✓ Method's modification created NEW object ✓ Original reference still points to original object
Key Concepts:
  • Pass-by-Value: Method gets copy of reference, not original reference
  • Immutability + Pass-by-Value: Combination prevents external modification
  • Local Scope: Method's str variable is local, doesn't affect caller
  • Security: Methods can't accidentally modify caller's strings
Contrast: If strings were mutable, method could change original string

Question 5: Chaining String Methods

Problem Statement

Write a Java program that demonstrates method chaining with strings. Show that: • Each method returns a new string • Methods can be chained together • The original string remains unchanged Apply multiple transformations in one statement.

Solution

import java.util.Scanner; class _05_ { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a string: "); String original = sn.nextLine(); System.out.println("\nOriginal: " + original); // Method chaining - each method returns new String String result = original.trim() .toLowerCase() .replace(' ', '_') .concat("_processed"); System.out.println("\nAfter chaining:"); System.out.println(" .trim()"); System.out.println(" .toLowerCase()"); System.out.println(" .replace(' ', '_')"); System.out.println(" .concat(\"_processed\")"); System.out.println("\nResult: " + result); System.out.println("Original: " + original); System.out.println("\n✓ Each method created a new String object"); System.out.println("✓ Total new objects created: 4"); System.out.println("✓ Original string unchanged: " + original); sn.close(); } }

Sample Output

Enter a string: Hello World Original: Hello World After chaining: .trim() .toLowerCase() .replace(' ', '_') .concat("_processed") Result: hello_world_processed Original: Hello World ✓ Each method created a new String object ✓ Total new objects created: 4 ✓ Original string unchanged: Hello World
Key Concepts:
  • Method Chaining: Possible because each method returns a String
  • Intermediate Objects: Each chained method creates temporary object
  • Fluent API: Chaining makes code more readable
  • Memory Cost: Multiple temporary objects created and discarded
Efficiency: For heavy chaining, consider StringBuilder for better performance
