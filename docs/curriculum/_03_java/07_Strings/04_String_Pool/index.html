String Pool in Java

Understanding Java's String Pool memory optimization.

What is String Pool?

String Pool is a special memory region where Java stores string literals. When you create a string literal, Java checks if it already exists in the pool. If yes, it reuses the existing object.

💡 Memory Optimization: String Pool saves memory by storing only one copy of each unique string literal.

String Pool Example

public class StringPoolDemo {
    public static void main(String[] args) {
        // String literals - stored in pool
        String s1 = "Hello";
        String s2 = "Hello";
        
        // new keyword - creates object in heap
        String s3 = new String("Hello");
        
        // == compares references
        System.out.println(s1 == s2);  // true (same reference in pool)
        System.out.println(s1 == s3);  // false (different objects)
        
        // equals() compares content
        System.out.println(s1.equals(s3));  // true (same content)
    }
}

intern() Method

public class InternDemo {
    public static void main(String[] args) {
        String s1 = new String("Java");
        String s2 = s1.intern();  // Adds to pool if not exists
        String s3 = "Java";
        
        System.out.println(s1 == s2);  // false
        System.out.println(s2 == s3);  // true (both from pool)
    }
}

true
