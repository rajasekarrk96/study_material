String Introduction

Section 1: What are Strings?

A String is a sequence of characters (text). It's one of the most commonly used data types in Java. Strings are immutable, meaning once created, they cannot be changed. Every string in Java is an object of the String class.

Key Characteristics

  • Sequence of characters: A String contains letters, numbers, symbols
  • Immutable: Once created, the value cannot be changed
  • Reference type: Strings are objects, not primitive types
  • Widely used: Almost every program uses strings

💡 String vs Char

String represents multiple characters and uses double quotes: "Hello"
char represents a single character and uses single quotes: 'A'

🎯 Real-World Example

Strings are used for names, addresses, messages, usernames, passwords - basically any text data in programs.

Section 2: String Creation and Declaration

Strings can be created in multiple ways in Java:

Method 1: String Literal

String name = "Alice"; String greeting = "Hello World"; String empty = "";

Method 2: Using new Keyword

String str1 = new String("Java"); String str2 = new String("Programming"); String str3 = new String(); // Empty string

Method 3: String Concatenation

String firstName = "John"; String lastName = "Doe"; String fullName = firstName + " " + lastName; // Concatenation System.out.println(fullName); // Output: John Doe

Complete Example

public class StringCreation { public static void main(String[] args) { // Using literals String str1 = "Java"; String str2 = "Programming"; // Using new keyword String str3 = new String("Hello"); // Concatenation String combined = str1 + " " + str2; System.out.println(combined); // Output: Java Programming // String from variables int age = 25; String message = "I am " + age + " years old"; System.out.println(message); // Output: I am 25 years old } }

Output

Java Programming
I am 25 years old

📝 String Literal vs new Keyword

"Hello" creates string in String pool (memory efficient)
new String("Hello") creates string in heap (creates new object each time)

Section 3: String Immutability Concept

Immutability means once a String is created, it cannot be modified. Any operation that appears to modify a string actually creates a new string object. This is crucial for understanding string behavior in Java.

Immutability Explained

String original = "Hello"; String modified = original.toUpperCase(); System.out.println(original); // Output: Hello (unchanged) System.out.println(modified); // Output: HELLO (new string)

Example: Demonstrating Immutability

public class Immutability { public static void main(String[] args) { String str = "Java"; System.out.println("Original: " + str); // These methods return new strings, don't modify original String upper = str.toUpperCase(); String lower = str.toLowerCase(); String replaced = str.replace('a', 'o'); System.out.println("Uppercase: " + upper); // JAVA System.out.println("Lowercase: " + lower); // java System.out.println("Replaced: " + replaced); // Jovo System.out.println("Original still: " + str); // Java (unchanged) } }

Impact of Immutability

Advantage Disadvantage
Thread-safe by nature Creates new objects frequently
Can be cached safely Memory overhead
Simple to use Performance impact with many operations

🔍 Why Immutable?

Immutability makes strings safe for concurrent use, enables caching, and simplifies code reasoning.

Section 4: String Pool and Memory

Java uses a special memory region called the String Pool to optimize string storage:

What is String Pool?

String str1 = "Hello"; // Created in String Pool String str2 = "Hello"; // References same object in Pool String str3 = new String("Hello"); // Created in Heap (not in Pool) System.out.println(str1 == str2); // true (same reference) System.out.println(str1 == str3); // false (different objects)

String Pool Behavior

public class StringPoolDemo { public static void main(String[] args) { // Both reference same object in String Pool String a = "Java"; String b = "Java"; System.out.println(a == b); // true System.out.println(a.equals(b)); // true // Different objects in heap String c = new String("Java"); System.out.println(a == c); // false (different objects) System.out.println(a.equals(c)); // true (same content) // Using intern() to add to pool String d = c.intern(); System.out.println(a == d); // true (now in pool) } }

false
true
true

💡 Memory Efficiency

String Pool saves memory by storing only one copy of each unique string literal. This is why using string literals is preferred over new String().

Section 5: String vs Char vs StringBuffer

Understanding the differences between these string-related types helps choose the right one for your needs:

String vs Char

Feature String char
Definition Sequence of characters Single character
Quotes Double quotes: "Hello" Single quotes: 'A'
Type Reference (Object) Primitive
Size Variable Always 1 character

String vs StringBuffer

// String - Immutable (creates new object) String str = "Hello"; str = str + " World"; // Creates new String object // StringBuffer - Mutable (modifies same object) StringBuffer sb = new StringBuffer("Hello"); sb.append(" World"); // Modifies same object

Comparison Table

Aspect String StringBuffer
Mutability Immutable Mutable
Thread Safety Always safe Synchronized (thread-safe)
Performance Slow for modifications Fast for modifications
Use Case Fixed strings Frequent modifications

Example: When to Use Each

public class StringComparison { public static void main(String[] args) { // Use String for fixed values String name = "John"; System.out.println("Name: " + name); // Use StringBuffer for frequent modifications StringBuffer sb = new StringBuffer(); for (int i = 1; i <= 5; i++) { sb.append(i).append(", "); } System.out.println("Numbers: " + sb); // String concatenation (less efficient) String result = ""; for (int i = 1; i <= 5; i++) { result += i + ", "; // Creates new object each iteration } System.out.println("Result: " + result); } }

⚠️ Performance Tip

When doing many string concatenations in a loop, use StringBuffer or StringBuilder instead of String. It's much more efficient.
