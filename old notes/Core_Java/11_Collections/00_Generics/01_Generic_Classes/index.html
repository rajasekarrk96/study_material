Generic Classes

Section 1: What are Generics?

Generics enable types (classes and interfaces) to be parameters when defining classes, interfaces, and methods. They provide stronger type checks at compile time and eliminate the need for casting.

// Without Generics (before Java 5) ArrayList list = new ArrayList(); list.add("Hello"); String s = (String) list.get(0); // Cast required // With Generics (Java 5+) ArrayList<String> list = new ArrayList<String>(); list.add("Hello"); String s = list.get(0); // No cast needed!

Section 2: Generic Class Syntax

A generic class is defined with type parameters in angle brackets after the class name.

class Box<T> { private T value; public void set(T value) { this.value = value; } public T get() { return value; } } // Usage Box<Integer> intBox = new Box<Integer>(); intBox.set(10); Integer num = intBox.get(); Box<String> strBox = new Box<String>(); strBox.set("Hello"); String text = strBox.get();

Section 3: Type Parameters

Common naming conventions for type parameters:

Parameter Meaning Example
T Type Box<T>
E Element List<E>
K Key Map<K,V>
V Value Map<K,V>
N Number Calculator<N>

Section 4: Benefits of Generics

  • Type Safety: Compile-time type checking prevents ClassCastException
  • No Casting: Eliminates explicit type casting
  • Code Reusability: Write once, use with any type
  • Better Documentation: Code is more readable and self-documenting
// Type Safety Example Box<Integer> box = new Box<>(); box.set(10); // OK box.set("Hello"); // Compile Error! Type mismatch

Section 5: Multiple Type Parameters

class Pair<K, V> { private K key; private V value; public Pair(K key, V value) { this.key = key; this.value = value; } public K getKey() { return key; } public V getValue() { return value; } } // Usage Pair<String, Integer> pair = new Pair<>("Age", 25); System.out.println(pair.getKey() + ": " + pair.getValue());
