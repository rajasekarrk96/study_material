Final Keyword

Final Keyword in Java

Restrict modification and inheritance using the final keyword.

What is the Final Keyword?

The final keyword in Java is used to restrict the user from modifying a variable, method, or class. Once declared as final, they cannot be changed, overridden, or inherited respectively.

Uses of Final Keyword:
Applied To Effect Use Case
Variable Creates a constant (cannot be reassigned) Mathematical constants, configuration values
Method Prevents method overriding Critical methods that shouldn't be changed
Class Prevents inheritance Immutable classes, security-sensitive classes

1. Final Variables

A final variable can be assigned only once. Once assigned, its value cannot be changed.

Example 1: Final Local Variable
public class FinalVariableDemo {
    public static void main(String[] args) {
        final int MAX_VALUE = 100;
        System.out.println("Max Value: " + MAX_VALUE);
        
        // MAX_VALUE = 200;  // ERROR: Cannot assign a value to final variable
    }
}
Max Value: 100
Example 2: Final Instance Variable
class Student {
    final String SCHOOL_NAME = "ABC School";  // Initialized at declaration
    final int rollNumber;  // Blank final variable
    
    Student(int roll) {
        rollNumber = roll;  // Must be initialized in constructor
    }
    
    void display() {
        System.out.println("School: " + SCHOOL_NAME);
        System.out.println("Roll Number: " + rollNumber);
    }
}

public class FinalInstanceDemo {
    public static void main(String[] args) {
        Student s = new Student(101);
        s.display();
        
        // s.rollNumber = 102;  // ERROR: Cannot assign to final variable
    }
}
School: ABC School
Roll Number: 101
Example 3: Final Static Variable (Constants)
class MathConstants {
    static final double PI = 3.14159;
    static final double E = 2.71828;
    
    static void displayConstants() {
        System.out.println("PI = " + PI);
        System.out.println("E = " + E);
    }
}

public class FinalStaticDemo {
    public static void main(String[] args) {
        MathConstants.displayConstants();
        System.out.println("Circle Area: " + (MathConstants.PI * 5 * 5));
    }
}
PI = 3.14159
E = 2.71828
Circle Area: 78.53975
💡 Naming Convention: Final variables (constants) are typically named in UPPERCASE with underscores (e.g., MAX_VALUE, PI).

2. Final Methods

A final method cannot be overridden by subclasses. This ensures the method's behavior remains consistent across the inheritance hierarchy.

Example 1: Final Method Cannot Be Overridden
class Parent {
    final void display() {
        System.out.println("This is a final method in Parent class");
    }
    
    void show() {
        System.out.println("This is a normal method");
    }
}

class Child extends Parent {
    // void display() {  // ERROR: Cannot override final method
    //     System.out.println("Trying to override");
    // }
    
    @Override
    void show() {
        System.out.println("Overridden show method in Child");
    }
}

public class FinalMethodDemo {
    public static void main(String[] args) {
        Child c = new Child();
        c.display();
        c.show();
    }
}
This is a final method in Parent class
Overridden show method in Child
Example 2: Final Method for Security
class BankAccount {
    private double balance;
    
    BankAccount(double balance) {
        this.balance = balance;
    }
    
    // Critical method that should not be overridden
    final double getBalance() {
        return balance;
    }
    
    void deposit(double amount) {
        balance += amount;
    }
}

class SavingsAccount extends BankAccount {
    SavingsAccount(double balance) {
        super(balance);
    }
    
    // Cannot override getBalance() - it's final
    
    @Override
    void deposit(double amount) {
        super.deposit(amount);
        System.out.println("Deposited: " + amount);
    }
}

public class BankDemo {
    public static void main(String[] args) {
        SavingsAccount acc = new SavingsAccount(1000);
        acc.deposit(500);
        System.out.println("Balance: " + acc.getBalance());
    }
}
Deposited: 500.0
Balance: 1500.0

3. Final Classes

A final class cannot be extended (inherited). This is used when you want to prevent inheritance for security or design reasons.

Example 1: Final Class Cannot Be Inherited
final class ImmutableClass {
    private final int value;
    
    ImmutableClass(int value) {
        this.value = value;
    }
    
    int getValue() {
        return value;
    }
}

// class ExtendedClass extends ImmutableClass {  // ERROR: Cannot inherit from final class
// }

public class FinalClassDemo {
    public static void main(String[] args) {
        ImmutableClass obj = new ImmutableClass(100);
        System.out.println("Value: " + obj.getValue());
    }
}
Value: 100
Example 2: Real-World Final Classes
// String class is final in Java
final class MyString {
    private final String value;
    
    MyString(String value) {
        this.value = value;
    }
    
    String getValue() {
        return value;
    }
    
    int length() {
        return value.length();
    }
}

public class StringDemo {
    public static void main(String[] args) {
        MyString str = new MyString("Hello");
        System.out.println("String: " + str.getValue());
        System.out.println("Length: " + str.length());
        
        // Real Java String class is also final
        String javaStr = "World";
        System.out.println("Java String: " + javaStr);
    }
}
String: Hello
Length: 5
Java String: World
📌 Important Java Classes that are Final:
  • String - Immutability and security
  • Integer, Double, Boolean - Wrapper classes
  • Math - Utility class
  • System - System utilities

Practical Examples

Example 1: Configuration Class with Final Variables
class AppConfig {
    static final String APP_NAME = "MyApp";
    static final String VERSION = "1.0.0";
    static final int MAX_USERS = 1000;
    static final boolean DEBUG_MODE = true;
    
    static void displayConfig() {
        System.out.println("Application: " + APP_NAME);
        System.out.println("Version: " + VERSION);
        System.out.println("Max Users: " + MAX_USERS);
        System.out.println("Debug Mode: " + DEBUG_MODE);
    }
}

public class ConfigDemo {
    public static void main(String[] args) {
        AppConfig.displayConfig();
    }
}
Application: MyApp
Version: 1.0.0
Max Users: 1000
Debug Mode: true
Example 2: Final Parameters
class Calculator {
    int add(final int a, final int b) {
        // a = 10;  // ERROR: Cannot assign to final parameter
        return a + b;
    }
    
    void display(final String message) {
        // message = "Changed";  // ERROR: Cannot modify final parameter
        System.out.println(message);
    }
}

public class FinalParameterDemo {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        System.out.println("Sum: " + calc.add(5, 3));
        calc.display("Hello, Final Parameters!");
    }
}
Sum: 8
Hello, Final Parameters!
