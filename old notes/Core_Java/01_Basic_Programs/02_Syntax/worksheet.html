Question 1: Single-Line Comment Program

Problem Statement

Write a Java program that prints "Learning Java Syntax" and includes a single-line comment explaining what the program does.

Sample Input and Output

Sample Input 1:
(No input required)

Sample Output 1:
Learning Java Syntax

Sample Input 2:
(No input required)

Sample Output 2:
Learning Java Syntax
Show Solution

Solution (Java Program)

public class CommentExample { public static void main(String[] args) { // This program demonstrates single-line comments System.out.println("Learning Java Syntax"); } }

Step-by-Step Explanation

  1. Single-Line Comment: // is used for single-line comments. Everything after // on that line is ignored by the compiler.
  2. Class Declaration: Defines a public class named CommentExample.
  3. Main Method: The entry point of the program.
  4. Print Statement: Outputs the text to the console.
  5. Comments Purpose: Comments help explain code to other programmers and to yourself when you revisit code later.

Question 2: Multi-Line Comment Program

Problem Statement

Write a Java program that calculates the area of a rectangle. Include a multi-line comment at the beginning explaining the program's purpose.

Sample Input and Output

Sample Input 1:
5
10

Sample Output 1:
Area of rectangle: 50

Sample Input 2:
7
8

Sample Output 2:
Area of rectangle: 56
Show Solution

Solution (Java Program)

import java.util.Scanner; /* * This program calculates the area of a rectangle * Formula: Area = length * width * Author: Java Student */ public class RectangleArea { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter length: "); int length = sn.nextInt(); System.out.print("Enter width: "); int width = sn.nextInt(); int area = length * width; System.out.println("Area of rectangle: " + area); sn.close(); } }

Step-by-Step Explanation

  1. Multi-Line Comment: /* ... */ is used for multi-line comments. Everything between /* and */ is ignored by the compiler.
  2. Import Scanner: Imports the Scanner class for user input.
  3. Read Dimensions: Reads length and width from the user.
  4. Calculate Area: Multiplies length and width to get the area.
  5. Display Result: Prints the calculated area.
  6. Documentation: Multi-line comments are often used for documentation at the beginning of classes or methods.

Question 3: Proper Indentation Practice

Problem Statement

Write a Java program that takes a number and prints whether it's positive or negative. Use proper indentation to make the code readable.

Sample Input and Output

Sample Input 1:
15

Sample Output 1:
The number is positive

Sample Input 2:
-8

Sample Output 2:
The number is negative
Show Solution

Solution (Java Program)

import java.util.Scanner; public class CheckNumber { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a number: "); int number = sn.nextInt(); if (number > 0) { System.out.println("The number is positive"); } else if (number < 0) { System.out.println("The number is negative"); } else { System.out.println("The number is zero"); } sn.close(); } }

Step-by-Step Explanation

  1. Proper Indentation: Notice how code inside braces { } is indented. This makes the code structure clear and readable.
  2. Read Number: Gets an integer input from the user.
  3. Conditional Check: Uses if-else statements to check if the number is positive, negative, or zero.
  4. Code Blocks: Each condition has its own code block enclosed in braces.
  5. Readability: Proper indentation is crucial for understanding code structure, especially in larger programs.
  6. Best Practice: Always use consistent indentation (typically 4 spaces or 1 tab).

Question 4: Variable Naming Conventions

Problem Statement

Write a Java program that calculates the total price of items including tax. Use proper variable naming conventions (camelCase for variables).

Sample Input and Output

Sample Input 1:
100
10

Sample Output 1:
Total Price: 110.0

Sample Input 2:
500
5

Sample Output 2:
Total Price: 525.0
Show Solution

Solution (Java Program)

import java.util.Scanner; public class CalculatePrice { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter item price: "); double itemPrice = sn.nextDouble(); System.out.print("Enter tax percentage: "); double taxPercentage = sn.nextDouble(); double taxAmount = (itemPrice * taxPercentage) / 100; double totalPrice = itemPrice + taxAmount; System.out.println("Total Price: " + totalPrice); sn.close(); } }

Step-by-Step Explanation

  1. CamelCase Naming: Variable names like itemPrice, taxPercentage, and totalPrice follow camelCase convention (first word lowercase, subsequent words capitalized).
  2. Descriptive Names: Variable names clearly describe what they store, making the code self-documenting.
  3. Read Input: Uses nextDouble() to read decimal numbers.
  4. Calculate Tax: Computes tax amount as a percentage of the item price.
  5. Calculate Total: Adds tax to the original price.
  6. Naming Rules: Variables must start with a letter, $, or _. They cannot start with a number or contain spaces.

Question 5: Semicolon and Statement Termination

Problem Statement

Write a Java program that demonstrates proper use of semicolons by calculating the average of three numbers.

Sample Input and Output

Sample Input 1:
10
20
30

Sample Output 1:
Average: 20.0

Sample Input 2:
15
25
35

Sample Output 2:
Average: 25.0
Show Solution

Solution (Java Program)

import java.util.Scanner; public class CalculateAverage { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter first number: "); int num1 = sn.nextInt(); System.out.print("Enter second number: "); int num2 = sn.nextInt(); System.out.print("Enter third number: "); int num3 = sn.nextInt(); double sum = num1 + num2 + num3; double average = sum / 3.0; System.out.println("Average: " + average); sn.close(); } }

Step-by-Step Explanation

  1. Semicolons: Notice that every statement ends with a semicolon (;). This is mandatory in Java.
  2. Statement Termination: The semicolon tells the compiler where one statement ends and another begins.
  3. Read Three Numbers: Reads three integer values from the user.
  4. Calculate Sum: Adds all three numbers together.
  5. Calculate Average: Divides the sum by 3.0 (using 3.0 instead of 3 ensures we get a decimal result).
  6. Missing Semicolon Error: Forgetting a semicolon is one of the most common syntax errors in Java and will prevent your program from compiling.
