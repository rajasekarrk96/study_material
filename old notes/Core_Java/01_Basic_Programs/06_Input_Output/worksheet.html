Question 1: Reading Different Data Types

Problem Statement

Write a Java program that reads different types of input (String, int, double) and displays them in a formatted manner.

Sample Input and Output

Sample Input 1:
Alice
25
5.9

Sample Output 1:
Name: Alice
Age: 25 years
Height: 5.9 feet

Sample Input 2:
Bob
30
6.1

Sample Output 2:
Name: Bob
Age: 30 years
Height: 6.1 feet
Show Solution

Solution (Java Program)

import java.util.Scanner; public class ReadMultipleTypes { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter name: "); String name = sn.nextLine(); System.out.print("Enter age: "); int age = sn.nextInt(); System.out.print("Enter height: "); double height = sn.nextDouble(); System.out.println("Name: " + name); System.out.println("Age: " + age + " years"); System.out.println("Height: " + height + " feet"); sn.close(); } }

Step-by-Step Explanation

  1. Scanner provides type-specific input methods: Each data type has its own method - nextLine() for strings, nextInt() for integers, nextDouble() for decimals. This type safety prevents runtime errors from invalid input.
  2. Reading order matters for mixed input: We read the string first using nextLine() to avoid the newline character issue. If we read numbers first, we'd need an extra nextLine() call to consume the leftover newline.
  3. Formatted output enhances readability: By adding units like "years" and "feet", we make the output self-explanatory. This is crucial for user-facing applications where context matters.
  4. Resource management with close(): Calling sn.close() releases system resources. While not critical in small programs, it's good practice and prevents resource leaks in larger applications.

Question 2: Formatted Output with printf

Problem Statement

Write a Java program that displays a student's information using printf for formatted output with specific decimal places and alignment.

Sample Input and Output

Sample Input 1:
John
85.678
92.345

Sample Output 1:
Student: John
Math Score: 85.68
Science Score: 92.35
Average: 89.01

Sample Input 2:
Sarah
78.456
88.234

Sample Output 2:
Student: Sarah
Math Score: 78.46
Science Score: 88.23
Average: 83.35
Show Solution

Solution (Java Program)

import java.util.Scanner; public class FormattedOutput { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter student name: "); String name = sn.nextLine(); System.out.print("Enter math score: "); double math = sn.nextDouble(); System.out.print("Enter science score: "); double science = sn.nextDouble(); double average = (math + science) / 2; System.out.printf("Student: %s%n", name); System.out.printf("Math Score: %.2f%n", math); System.out.printf("Science Score: %.2f%n", science); System.out.printf("Average: %.2f%n", average); sn.close(); } }

Step-by-Step Explanation

  1. printf enables precise formatting control: Unlike println which just converts values to strings, printf lets us specify exact formatting - number of decimal places, padding, alignment, etc.
  2. Format specifiers define output structure: %s means string, %.2f means floating-point with 2 decimal places, %n is a platform-independent newline. These placeholders are replaced by the actual values in order.
  3. Decimal precision prevents misleading accuracy: Showing 85.678 as 85.68 is more appropriate for scores. The extra digits suggest false precision - we're not measuring to thousandths of a point.
  4. Consistent formatting improves professionalism: All scores display with exactly 2 decimal places, creating a clean, aligned output. This attention to detail makes applications feel polished and trustworthy.

Question 3: Handling Input Buffer Issues

Problem Statement

Write a Java program that correctly handles mixed input (numbers followed by strings) by managing the input buffer properly.

Sample Input and Output

Sample Input 1:
101
Computer Science
3.8

Sample Output 1:
Student ID: 101
Course: Computer Science
GPA: 3.8

Sample Input 2:
205
Mechanical Engineering
3.5

Sample Output 2:
Student ID: 205
Course: Mechanical Engineering
GPA: 3.5
Show Solution

Solution (Java Program)

import java.util.Scanner; public class BufferHandling { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter student ID: "); int id = sn.nextInt(); sn.nextLine(); // Consume the newline System.out.print("Enter course name: "); String course = sn.nextLine(); System.out.print("Enter GPA: "); double gpa = sn.nextDouble(); System.out.println("Student ID: " + id); System.out.println("Course: " + course); System.out.println("GPA: " + gpa); sn.close(); } }

Step-by-Step Explanation

  1. The newline character causes input issues: When you type "101" and press Enter, nextInt() reads "101" but leaves the Enter key (newline character) in the buffer. If we immediately call nextLine(), it reads that leftover newline as an empty string.
  2. Consuming the newline prevents skipped input: The extra sn.nextLine() after nextInt() reads and discards that leftover newline character, clearing the buffer for the next actual input.
  3. This pattern applies to all numeric inputs: Whenever you read numbers (nextInt, nextDouble, etc.) before strings (nextLine), you need to consume the newline. This is one of the most common Scanner pitfalls.
  4. Understanding the buffer is key: Scanner doesn't automatically clear input - it reads character by character from a buffer. Knowing what's left in the buffer after each read operation helps you write robust input code.

Question 4: Reading Multiple Words

Problem Statement

Write a Java program that demonstrates the difference between next() and nextLine() when reading multi-word input.

Sample Input and Output

Sample Input 1:
Hello World
Java Programming

Sample Output 1:
Using next(): Hello
Using nextLine(): World
Full line: Java Programming

Sample Input 2:
Good Morning
Happy Coding

Sample Output 2:
Using next(): Good
Using nextLine(): Morning
Full line: Happy Coding
Show Solution

Solution (Java Program)

import java.util.Scanner; public class NextVsNextLine { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter two words: "); String word1 = sn.next(); String remaining = sn.nextLine(); System.out.print("Enter a full sentence: "); String sentence = sn.nextLine(); System.out.println("Using next(): " + word1); System.out.println("Using nextLine(): " + remaining.trim()); System.out.println("Full line: " + sentence); sn.close(); } }

Step-by-Step Explanation

  1. next() reads until whitespace: It stops at the first space, tab, or newline. So "Hello World" with next() only captures "Hello", leaving " World" in the buffer.
  2. nextLine() reads until newline: It captures everything up to and including the Enter key, then discards the newline. This is why it can read multi-word input like "Java Programming".
  3. Combining both requires understanding buffer state: After next() reads "Hello", the buffer contains " World\n". When we call nextLine(), it reads " World" (with leading space). We use trim() to remove that space.
  4. Choose the right method for your needs: Use next() when you want single words (like parsing commands). Use nextLine() when you need full sentences or phrases. Mixing them requires careful buffer management.

Question 5: Creating a Simple Receipt

Problem Statement

Write a Java program that takes item details and creates a formatted receipt with proper alignment and calculations.

Sample Input and Output

Sample Input 1:
Laptop
999.99
2

Sample Output 1:
========== RECEIPT ==========
Item: Laptop
Price: $999.99
Quantity: 2
Total: $1999.98
============================

Sample Input 2:
Mouse
25.50
3

Sample Output 2:
========== RECEIPT ==========
Item: Mouse
Price: $25.50
Quantity: 3
Total: $76.50
============================
Show Solution

Solution (Java Program)

import java.util.Scanner; public class Receipt { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter item name: "); String item = sn.nextLine(); System.out.print("Enter price: "); double price = sn.nextDouble(); System.out.print("Enter quantity: "); int quantity = sn.nextInt(); double total = price * quantity; System.out.println("========== RECEIPT =========="); System.out.println("Item: " + item); System.out.printf("Price: $%.2f%n", price); System.out.println("Quantity: " + quantity); System.out.printf("Total: $%.2f%n", total); System.out.println("============================"); sn.close(); } }

Step-by-Step Explanation

  1. Structured output creates professional appearance: The header and footer lines (====) frame the receipt, making it visually distinct and easy to read. This simple formatting dramatically improves user experience.
  2. Consistent decimal formatting for currency: Using %.2f ensures all monetary values show exactly 2 decimal places, which is standard for currency. $25.5 becomes $25.50, maintaining professional presentation.
  3. Calculation happens before display: We compute the total first, then display all information together. This separation of logic (calculation) from presentation (output) is a fundamental programming principle.
  4. Real-world application demonstrates concepts: This receipt program combines multiple I/O concepts - reading different types, formatting output, performing calculations. It shows how basic I/O operations build into practical applications.
