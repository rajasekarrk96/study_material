Question 1: Basic Arithmetic Calculator

Problem Statement

Write a Java program that performs all basic arithmetic operations (+, -, *, /, %) on two numbers and displays the results.

Sample Input and Output

Sample Input 1:
20
3

Sample Output 1:
Addition: 23
Subtraction: 17
Multiplication: 60
Division: 6
Modulus: 2

Sample Input 2:
15
4

Sample Output 2:
Addition: 19
Subtraction: 11
Multiplication: 60
Division: 3
Modulus: 3
Show Solution

Solution (Java Program)

import java.util.Scanner; public class ArithmeticOps { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter first number: "); int a = sn.nextInt(); System.out.print("Enter second number: "); int b = sn.nextInt(); System.out.println("Addition: " + (a + b)); System.out.println("Subtraction: " + (a - b)); System.out.println("Multiplication: " + (a * b)); System.out.println("Division: " + (a / b)); System.out.println("Modulus: " + (a % b)); sn.close(); } }

Step-by-Step Explanation

  1. Arithmetic operators perform mathematical calculations: Each operator has a specific purpose - addition combines values, subtraction finds difference, multiplication scales values, division splits into equal parts, and modulus finds the remainder.
  2. Integer division truncates decimals: When dividing 20 by 3, we get 6 (not 6.67) because both operands are integers. The fractional part is discarded.
  3. Modulus reveals remainder information: The % operator is particularly useful for determining divisibility, cycling through ranges, or extracting digits from numbers.
  4. Order of operations matters: Java follows PEMDAS rules - parentheses, then multiplication/division/modulus (left to right), then addition/subtraction (left to right).

Question 2: Increment and Decrement Operators

Problem Statement

Write a Java program that demonstrates the difference between pre-increment (++x) and post-increment (x++) operators.

Sample Input and Output

Sample Input 1:
5

Sample Output 1:
Original: 5
Post-increment: 5
After post: 6
Pre-increment: 7
After pre: 7

Sample Input 2:
10

Sample Output 2:
Original: 10
Post-increment: 10
After post: 11
Pre-increment: 12
After pre: 12
Show Solution

Solution (Java Program)

import java.util.Scanner; public class IncrementDemo { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a number: "); int x = sn.nextInt(); System.out.println("Original: " + x); System.out.println("Post-increment: " + (x++)); System.out.println("After post: " + x); System.out.println("Pre-increment: " + (++x)); System.out.println("After pre: " + x); sn.close(); } }

Step-by-Step Explanation

  1. Post-increment (x++) uses then increments: The current value is used in the expression first, then the variable is incremented. This is why we see the original value (5) printed, but x becomes 6 afterward.
  2. Pre-increment (++x) increments then uses: The variable is incremented first, then the new value is used in the expression. So x becomes 7 before being printed.
  3. The timing difference is crucial: In complex expressions, choosing pre vs post increment can completely change the result. For example, array[i++] vs array[++i] access different elements.
  4. Both permanently modify the variable: Regardless of pre or post, the variable's value increases by 1. The difference is only in when that new value becomes available to the surrounding expression.

Question 3: Relational and Logical Operators

Problem Statement

Write a Java program that checks if a number is within a specific range (10-50) using relational and logical operators.

Sample Input and Output

Sample Input 1:
25

Sample Output 1:
Is 25 >= 10? true
Is 25 <= 50? true
Is 25 in range [10-50]? true

Sample Input 2:
75

Sample Output 2:
Is 75 >= 10? true
Is 75 <= 50? false
Is 75 in range [10-50]? false
Show Solution

Solution (Java Program)

import java.util.Scanner; public class RangeCheck { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a number: "); int num = sn.nextInt(); boolean greaterThan10 = num >= 10; boolean lessThan50 = num <= 50; boolean inRange = greaterThan10 && lessThan50; System.out.println("Is " + num + " >= 10? " + greaterThan10); System.out.println("Is " + num + " <= 50? " + lessThan50); System.out.println("Is " + num + " in range [10-50]? " + inRange); sn.close(); } }

Step-by-Step Explanation

  1. Relational operators create boolean conditions: Operators like >=, <=,==, !=compare values and return true or false. They form the building blocks of decision-making in programs.
  2. Logical AND (&&) requires both conditions: For a number to be in range, it must satisfy BOTH conditions simultaneously - it must be >= 10 AND <= 50. If either is false, the whole expression is false.
  3. Breaking down complex conditions improves clarity: By storing intermediate boolean results in variables, we make the logic easier to understand and debug. This is better than writing one giant expression.
  4. Short-circuit evaluation optimizes performance: The && operator stops checking as soon as it finds a false condition. If num < 10, Java won't even check if num <=50 because the result is already determined to be false.

Question 4: Ternary Operator

Problem Statement

Write a Java program that uses the ternary operator to find the maximum of two numbers.

Sample Input and Output

Sample Input 1:
45
32

Sample Output 1:
Maximum: 45

Sample Input 2:
18
27

Sample Output 2:
Maximum: 27
Show Solution

Solution (Java Program)

import java.util.Scanner; public class TernaryMax { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter first number: "); int a = sn.nextInt(); System.out.print("Enter second number: "); int b = sn.nextInt(); int max = (a > b) ? a : b; System.out.println("Maximum: " + max); sn.close(); } }

Step-by-Step Explanation

  1. Ternary operator is a compact if-else: The syntax (condition) ? valueIfTrue : valueIfFalse replaces a full if-else block, making code more concise for simple decisions.
  2. The condition determines which value is returned: If a > b evaluates to true, the expression returns a. Otherwise, it returns b. Only one of the two values is ever evaluated.
  3. Best used for simple value selection: Ternary operators excel at choosing between two values based on a condition. For complex logic with multiple statements, traditional if-else is clearer.
  4. Ternary operators can be nested but avoid overuse: While you can nest ternary operators like a > b ? a : (b > c ? b : c), this quickly becomes unreadable. Use if-else for complex scenarios.

Question 5: Compound Assignment Operators

Problem Statement

Write a Java program that demonstrates compound assignment operators (+=, -=, *=, /=, %=) by performing cumulative operations on a number.

Sample Input and Output

Sample Input 1:
100
5

Sample Output 1:
Initial: 100
After += 5: 105
After -= 5: 100
After *= 5: 500
After /= 5: 100
After %= 5: 0

Sample Input 2:
50
3

Sample Output 2:
Initial: 50
After += 3: 53
After -= 3: 50
After *= 3: 150
After /= 3: 50
After %= 3: 2
Show Solution

Solution (Java Program)

import java.util.Scanner; public class CompoundOps { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter starting value: "); int value = sn.nextInt(); System.out.print("Enter operand: "); int operand = sn.nextInt(); System.out.println("Initial: " + value); value += operand; System.out.println("After += " + operand + ": " + value); value -= operand; System.out.println("After -= " + operand + ": " + value); value *= operand; System.out.println("After *= " + operand + ": " + value); value /= operand; System.out.println("After /= " + operand + ": " + value); value %= operand; System.out.println("After %= " + operand + ": " + value); sn.close(); } }

Step-by-Step Explanation

  1. Compound operators combine operation with assignment: value += 5 is shorthand for value = value + 5. This pattern reduces code repetition and makes the intent clearer - we're modifying the variable in place.
  2. Each operation builds on the previous result: The value changes cumulatively through the program. Starting at 100, adding 5 gives 105, then subtracting 5 returns to 100, then multiplying by 5 gives 500, and so on.
  3. Compound operators are more efficient: Beyond being shorter to write, compound operators can be slightly faster because the variable is only evaluated once. In array[i] += 5, the index i is calculated once, whereas array[i] = array[i] + 5 calculates it twice.
  4. Understanding the flow is key: Each operation permanently modifies the variable, so the order matters. If we did *= before +=, we'd get different results. This demonstrates how state changes accumulate in programs.
