Question 1: Synchronized Counter - Thread-Safe Shared Resource

Problem Statement

Write a Java program that creates two threads and demonstrates how to synchronize their access to a shared resource. Both threads will increment a shared counter 1000 times. Without synchronization, the final count would be incorrect due to race conditions.

Solution

class Counter { int count = 0; synchronized void increment() { count++; } } class MyThread extends Thread { Counter c; MyThread(Counter c) { this.c = c; } public void run() { for (int i = 0; i < 1000; i++) c.increment(); } } class _01_ { public static void main(String[] args) throws Exception { Counter c = new Counter(); MyThread t1 = new MyThread(c); MyThread t2 = new MyThread(c); t1.start(); t2.start(); t1.join(); t2.join(); System.out.println("Count = " + c.count); } }

Sample Output

Count = 2000

Without Synchronization

// If increment() is NOT synchronized: void increment() { count++; // NOT thread-safe! } // Possible output (WRONG): Count = 1847 // Lost updates due to race condition! Count = 1923 Count = 1999 // Rarely 2000
Key Concepts:
  • synchronized Keyword: Ensures only one thread can execute the method at a time
  • Race Condition: Multiple threads accessing shared data simultaneously
  • Thread Safety: Protecting shared resources from concurrent modification
  • Monitor Lock: Each object has an intrinsic lock (monitor)
How it Works:
  1. Thread acquires lock on Counter object
  2. Executes increment() method
  3. Releases lock
  4. Other thread can now acquire lock

Question 2: wait() and notify() - Inter-Thread Communication

Problem Statement

Write a Java program that demonstrates how to use the wait() and notify() methods to coordinate the activities of multiple threads. One thread will send a message and wait, while another thread will receive and notify.

Solution

class Message { synchronized void send() throws InterruptedException { System.out.println("Sending message..."); wait(); System.out.println("Message sent"); } synchronized void receive() { System.out.println("Receiving message..."); notify(); } } class _04_ { public static void main(String[] args) { Message m = new Message(); new Thread(() -> { try { m.send(); } catch (Exception e) {} }).start(); new Thread(() -> { m.receive(); }).start(); } }

Sample Output

Sending message... Receiving message... Message sent

Understanding wait() and notify()

// wait() - Releases lock and waits for notification synchronized void method1() throws InterruptedException { wait(); // Thread releases lock and waits } // notify() - Wakes up one waiting thread synchronized void method2() { notify(); // Wakes up waiting thread } // notifyAll() - Wakes up all waiting threads synchronized void method3() { notifyAll(); // Wakes up all waiting threads }
Key Concepts:
  • wait(): Causes current thread to wait until another thread calls notify()
  • notify(): Wakes up a single thread waiting on the object's monitor
  • Must be synchronized: wait() and notify() must be called from synchronized context
  • Releases Lock: wait() releases the lock, allowing other threads to acquire it
Common Use Cases: Producer-Consumer pattern, thread coordination, resource pooling

Question 3: interrupt() Method - Interrupting Running Threads

Problem Statement

Write a Java program that demonstrates how to use the interrupt() method to interrupt a running thread. The thread will be sleeping for 5 seconds, but we'll interrupt it immediately.

Solution

class InterruptThread extends Thread { public void run() { try { Thread.sleep(5000); } catch (InterruptedException e) { System.out.println("Thread interrupted"); } } } class _09_ { public static void main(String[] args) { InterruptThread t = new InterruptThread(); t.start(); t.interrupt(); } }

Sample Output

Thread interrupted

Interrupt Handling Patterns

// Pattern 1: Catch and handle public void run() { try { Thread.sleep(5000); } catch (InterruptedException e) { System.out.println("Interrupted!"); return; // Exit gracefully } } // Pattern 2: Check interrupt status public void run() { while (!Thread.interrupted()) { // Do work System.out.println("Working..."); } System.out.println("Interrupted, cleaning up..."); } // Pattern 3: Propagate exception public void run() throws InterruptedException { Thread.sleep(5000); // Let caller handle }
Key Concepts:
  • interrupt(): Sets the interrupt flag on the target thread
  • InterruptedException: Thrown when thread is interrupted during blocking operation
  • Graceful Shutdown: Allows threads to clean up resources before terminating
  • Cooperative: Thread must check interrupt status or catch exception
When to Use: Canceling long-running tasks, shutting down thread pools, timeout handling

Question 4: Executor Framework - Thread Pool Management

Problem Statement

Write a Java program that uses the Executor framework to create a thread pool and submit tasks to it. Create a fixed thread pool with 3 threads and submit 5 tasks to demonstrate task queuing and execution.

Solution

import java.util.concurrent.ExecutorService; import java.util.concurrent.Executors; class Task implements Runnable { public void run() { System.out.println("Task executed by " + Thread.currentThread().getName()); } } class _10_ { public static void main(String[] args) { ExecutorService es = Executors.newFixedThreadPool(3); for (int i = 1; i <= 5; i++) es.execute(new Task()); es.shutdown(); } }

Sample Output

Task executed by pool-1-thread-1 Task executed by pool-1-thread-2 Task executed by pool-1-thread-3 Task executed by pool-1-thread-1 Task executed by pool-1-thread-2

Executor Framework Overview

// Different types of thread pools: // 1. Fixed Thread Pool (fixed number of threads) ExecutorService fixed = Executors.newFixedThreadPool(3); // 2. Cached Thread Pool (creates threads as needed) ExecutorService cached = Executors.newCachedThreadPool(); // 3. Single Thread Executor (single worker thread) ExecutorService single = Executors.newSingleThreadExecutor(); // 4. Scheduled Thread Pool (delayed/periodic tasks) ScheduledExecutorService scheduled = Executors.newScheduledThreadPool(2); // Submitting tasks: executor.execute(runnable); // Fire and forget Future future = executor.submit(runnable); // Get Future Future result = executor.submit(callable); // Get result // Shutdown: executor.shutdown(); // Graceful shutdown executor.shutdownNow(); // Immediate shutdown

Advantages of Executor Framework

Benefits:
  • Thread Reuse: Threads are reused, reducing overhead
  • Resource Management: Limits number of concurrent threads
  • Task Queuing: Automatically queues tasks when all threads are busy
  • Simplified API: Easier than manual thread management
Key Methods:
  • execute(Runnable) - Submit task without return value
  • submit(Callable) - Submit task with return value (Future)
  • shutdown() - Gracefully shutdown (waits for tasks to complete)
  • shutdownNow() - Immediate shutdown (interrupts running tasks)
Real-World Use: Web servers, background processing, parallel computations
