Question 1: Movie Ticket Price

Problem Statement

Calculate movie ticket price based on age: Child (0-12) = $5, Teen (13-17) = $8, Adult (18-64) = $12, Senior (65+) = $7.

Sample Input and Output

Sample Input 1:
10

Sample Output 1:
Ticket Price: $5

Sample Input 2:
70

Sample Output 2:
Ticket Price: $7
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class MovieTicket { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter age: "); int age = sn.nextInt(); if (age <= 12) { System.out.println("Ticket Price: $5"); } else if (age <= 17) { System.out.println("Ticket Price: $8"); } else if (age <= 64) { System.out.println("Ticket Price: $12"); } else { System.out.println("Ticket Price: $7"); } sn.close(); } }

Step-by-Step Explanation

  1. Sequential range checking is efficient: By ordering conditions from low to high (or high to low), you can use simple comparisons. Here, <=12, <=17, <=64 works because earlier conditions have already been eliminated.
  2. Inclusive vs exclusive boundaries matter: Using <= means the upper value is included in that range. Age 12 gets child price, age 13 gets teen price. Perfect for categorizing continuous values into discrete groups.
  3. Final else handles unbounded range: Ages 65+ don't need an explicit condition. The else automatically catches everything not handled by previous conditions, simplifying code and preventing gaps.
  4. Real pricing systems use this pattern: Age-based pricing, tax brackets, shipping costs - all use ladder structures. Understanding this pattern prepares you for real-world business logic implementation.

Question 2: Water State

Problem Statement

Determine water state based on temperature: <=0="Solid (Ice)" , 1-99="Liquid (Water)" ,>=100 = "Gas (Steam)".

Sample Input and Output

Sample Input 1:
-5

Sample Output 1:
Solid (Ice)

Sample Input 2:
105

Sample Output 2:
Gas (Steam)
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class WaterState { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter temperature (°C): "); int temp = sn.nextInt(); if (temp <= 0) { System.out.println("Solid (Ice)"); } else if (temp < 100) { System.out.println("Liquid (Water)"); } else { System.out.println("Gas (Steam)"); } sn.close(); } }

Step-by-Step Explanation

  1. Scientific ranges map to code logic: Physical properties of water at different temperatures translate directly to if-else conditions. Programming often models real-world phenomena this way.
  2. < vs <=creates precise boundaries: Using temp < 100 for liquid means 100 is NOT liquid (it's steam). This precision prevents edge case bugs where a value might fall into wrong category.
  3. Three state transitions require two checks: With three categories, you need only two explicit conditions. First divides solid/not-solid, second divides liquid/gas. The ladder structure handles the rest automatically.
  4. Consider temperature scale assumptions: This code assumes Celsius. For production, you'd specify units clearly or accept Fahrenheit too. Good programming anticipates these ambiguities.
