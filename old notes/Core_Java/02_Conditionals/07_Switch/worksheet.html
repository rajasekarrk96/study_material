Question 1: Calculator

Problem Statement

Create a simple calculator that takes two numbers and an operator (+, -, *, /) and performs the operation.

Sample Input and Output

Sample Input 1:
10 5 +

Sample Output 1:
Result: 15

Sample Input 2:
20 4 *

Sample Output 2:
Result: 80
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class Calculator { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter num1 num2 operator: "); int num1 = sn.nextInt(); int num2 = sn.nextInt(); char op = sn.next().charAt(0); switch (op) { case '+': System.out.println("Result: " + (num1 + num2)); break; case '-': System.out.println("Result: " + (num1 - num2)); break; case '*': System.out.println("Result: " + (num1 * num2)); break; case '/': System.out.println("Result: " + (num1 / num2)); break; default: System.out.println("Invalid operator"); } sn.close(); } }

Step-by-Step Explanation

  1. Switch is perfect for discrete values: When checking one variable against multiple specific values (like operators), switch is cleaner than multiple if-else statements. Makes code more readable and maintainable.
  2. Break prevents fall-through: Without break, execution continues to next case. Here, after performing addition, break exits the switch. Forgetting break is a common bug that causes unexpected behavior.
  3. Default handles unexpected input: Like else in if-else, default catches any value not explicitly handled. Always include default for completeness and error handling, even if you think you've covered all cases.
  4. Char switch works with single characters: Switch works with int, char, String (Java 7+), and enums. For characters like operators, it's more intuitive than comparing ASCII values or using multiple ifs.

Question 2: English to Spanish Numbers

Problem Statement

Convert digit (1-5) to Spanish word: 1="Uno", 2="Dos", 3="Tres", 4="Cuatro", 5="Cinco".

Sample Input and Output

Sample Input 1:
3

Sample Output 1:
Tres

Sample Input 2:
8

Sample Output 2:
Not in range
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class SpanishNumbers { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a digit (1-5): "); int num = sn.nextInt(); switch (num) { case 1: System.out.println("Uno"); break; case 2: System.out.println("Dos"); break; case 3: System.out.println("Tres"); break; case 4: System.out.println("Cuatro"); break; case 5: System.out.println("Cinco"); break; default: System.out.println("Not in range"); } sn.close(); } }

Step-by-Step Explanation

  1. Switch makes lookup tables clear: This pattern (input value → output value mapping) is common in programming. Switch statement makes these mappings explicit and easy to read, like a mini-dictionary.
  2. Each case is independent: Unlike if-else chains where order matters for ranges, switch cases can be in any order. Each case is checked for exact equality, not evaluated sequentially.
  3. Extensibility is straightforward: Adding case 6: "Seis" is trivial. With if-else, you'd add another else-if. Switch scales better for many discrete values - imagine handling 1-100!
  4. Alternative: arrays/maps for larger datasets: For 1-100 translations, an array would be more efficient. Switch is best for moderate numbers of cases (2-20). Beyond that, data structures work better.

Question 3: Menu System

Problem Statement

Create a menu: 1=View Profile, 2=Settings, 3=Logout, 4=Exit. Print corresponding action.

Sample Input and Output

Sample Input 1:
2

Sample Output 1:
Opening Settings...

Sample Input 2:
5

Sample Output 2:
Invalid choice
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class MenuSystem { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.println("Menu: 1-Profile 2-Settings 3-Logout 4-Exit"); System.out.print("Enter choice: "); int choice = sn.nextInt(); switch (choice) { case 1: System.out.println("Viewing Profile..."); break; case 2: System.out.println("Opening Settings..."); break; case 3: System.out.println("Logging out..."); break; case 4: System.out.println("Exiting application..."); break; default: System.out.println("Invalid choice"); } sn.close(); } }

Step-by-Step Explanation

  1. Menu systems naturally use switch: User interfaces with numbered options are a classic switch use case. The structure mirrors how users think: "choose option 1, 2, 3..." making code intuitive.
  2. Compare to if-else alternative: Would need if(choice==1) else if(choice==2) etc. Switch is more concise and shows at a glance that we're comparing one variable to multiple constants.
  3. Real menus would loop: Production code wraps this in a while loop so menu redisplays after each action (except logout/exit). This example shows the core logic; loops extend it for usability.
  4. Break is essential here: Without break after case 1, execution would "fall through" to case 2, 3, 4, printing all messages. This fall-through can be intentional (grouping cases) but usually it's a bug.
