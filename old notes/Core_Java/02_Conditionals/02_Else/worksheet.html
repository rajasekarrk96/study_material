Question 1: Check Even or Odd

Problem Statement

Write a Java program that checks if a number is even or odd. Print "Even" if the number is even, otherwise print "Odd".

Sample Input and Output

Sample Input 1:
24

Sample Output 1:
Even

Sample Input 2:
17

Sample Output 2:
Odd
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class EvenOdd { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a number: "); int num = sn.nextInt(); if (num % 2 == 0) { System.out.println("Even"); } else { System.out.println("Odd"); } sn.close(); } }

Step-by-Step Explanation

  1. The else clause provides an alternative path: While a standalone if executes code only when the condition is true, adding else ensures something always happens. This is perfect for binary decisions like even/odd, yes/no, pass/fail.
  2. Exactly one block will execute: Either the if block runs (when num % 2 == 0) or the else block runs (when num % 2 != 0). They never both execute. This mutual exclusivity is fundamental to if-else logic.
  3. Else eliminates silent failures: Without else, odd numbers would produce no output. Adding else guarantees user feedback for all inputs, improving user experience and making debugging easier.
  4. The else block needs no condition: It automatically handles all cases not caught by the if. This "catch-all" behavior makes code simpler and prevents logical gaps in your program.

Question 2: Check Positive or Negative

Problem Statement

Write a Java program that checks if a number is positive or negative. Print "Positive" if the number is greater than or equal to 0, otherwise print "Negative".

Sample Input and Output

Sample Input 1:
15

Sample Output 1:
Positive

Sample Input 2:
-8

Sample Output 2:
Negative
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class PositiveNegative { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter a number: "); int num = sn.nextInt(); if (num >= 0) { System.out.println("Positive"); } else { System.out.println("Negative"); } sn.close(); } }

Step-by-Step Explanation

  1. Choosing >= instead of > matters: This code treats 0 as positive. If the requirement were different (0 as negative or neither), you'd adjust the condition. Understanding these boundary cases is critical for correct logic.
  2. If-else perfectly models binary classification: When data falls into exactly two categories (positive/negative, on/off, open/closed), if-else is the ideal structure. It's clearer than multiple ifs and prevents logical errors.
  3. The else guarantees complete coverage: Since every number is either >= 0 or < 0, this if-else handles all possible inputs. There's no third case, no edge case left unhandled.
  4. Boolean complement is implicit: The else condition is implicitly "not (num >= 0)" which equals "num < 0". You don't write this condition explicitly, making code more concise and less error-prone.

Question 3: Check Voting Eligibility

Problem Statement

Write a Java program that checks if a person can vote. If age is 18 or above, print "Eligible to vote", otherwise print "Not eligible to vote".

Sample Input and Output

Sample Input 1:
20

Sample Output 1:
Eligible to vote

Sample Input 2:
16

Sample Output 2:
Not eligible to vote
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class VotingEligibility { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter your age: "); int age = sn.nextInt(); if (age >= 18) { System.out.println("Eligible to vote"); } else { System.out.println("Not eligible to vote"); } sn.close(); } }

Step-by-Step Explanation

  1. Else improves user feedback dramatically: Compare to a standalone if statement that prints only for eligible voters. With else, ineligible users get explicit feedback explaining why they saw no positive message.
  2. Real-world rules often require binary responses: Access control, permissions, eligibility checks - these require yes/no answers. If-else directly models this requirement without additional complexity.
  3. Negative feedback is often more important: When users don't qualify, telling them why is crucial. The else block delivers this negative feedback, preventing confusion and improving usability.
  4. Consider the real-world implications: In production, you'd validate that age is reasonable (not negative, not impossibly high). The logic shown here assumes valid input, but robust applications add error handling.
