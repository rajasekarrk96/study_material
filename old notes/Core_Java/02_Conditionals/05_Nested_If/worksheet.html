Question 1: Driving License Eligibility

Problem Statement

Check if person can get a driving license: Age must be 18+ AND must pass written test (score >= 70).

Sample Input and Output

Sample Input 1:
20 (age)
75 (score)

Sample Output 1:
Eligible for license

Sample Input 2:
16 (age)
80 (score)

Sample Output 2:
Not eligible - too young
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class DrivingLicense { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter age: "); int age = sn.nextInt(); System.out.print("Enter test score: "); int score = sn.nextInt(); if (age >= 18) { if (score >= 70) { System.out.println("Eligible for license"); } else { System.out.println("Not eligible - failed test"); } } else { System.out.println("Not eligible - too young"); } sn.close(); } }

Step-by-Step Explanation

  1. Nesting creates dependent conditions: The inner if (score check) only runs if outer if (age check) passes. This models real-world logic where certain checks only matter after prerequisites are met.
  2. Different error messages provide clarity: Failed age check and failed test give different messages. Nesting allows specific feedback at each decision point rather than generic "not eligible".
  3. Avoids unnecessary checks: If person is 16, we never evaluate their test score. Nesting is more efficient than checking both conditions always, and models real processes better.
  4. Could use &&, but nesting is clearer: While (age>=18 && score>=70) works, nesting provides better structure for different failure messages and makes the decision hierarchy explicit.

Question 2: Triangle Validity

Problem Statement

Check if three sides can form a valid triangle. Rule: sum of any two sides must be greater than the third side (must check all three combinations).

Sample Input and Output

Sample Input 1:
3 4 5

Sample Output 1:
Valid triangle

Sample Input 2:
1 2 5

Sample Output 2:
Not a valid triangle
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class TriangleValidity { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter three sides: "); int a = sn.nextInt(); int b = sn.nextInt(); int c = sn.nextInt(); if (a + b > c) { if (b + c > a) { if (a + c > b) { System.out.println("Valid triangle"); } else { System.out.println("Not a valid triangle"); } } else { System.out.println("Not a valid triangle"); } } else { System.out.println("Not a valid triangle"); } sn.close(); } }

Step-by-Step Explanation

  1. Three-level nesting checks three conditions: Each nesting level represents one triangle inequality. All three must be true for a valid triangle, making this deep nesting appropriate.
  2. Alternative: use && operator: Could write if((a+b>c) && (b+c>a) && (a+c>b)). Nested version is more verbose but shows step-by-step validation process clearly.
  3. Geometric rules become code logic: Mathematical triangle inequality theorem translates directly to nested if structure. Programming often implements mathematical or scientific principles this way.
  4. Consider avoiding deep nesting: Three levels is getting complex. In production, you'd likely use && for readability. Nesting is best for 2 levels max; beyond that, other patterns are clearer.
