Question 1: Multiplication Table

Problem Statement

Print multiplication table of a given number from 1 to 10 using for loop.

Sample Input and Output

Sample Input 1:
5

Sample Output 1:
5 x 1 = 5
5 x 2 = 10
5 x 3 = 15
...
5 x 10 = 50
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class MultiplicationTable { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter number: "); int num = sn.nextInt(); for (int i = 1; i <= 10; i++) { System.out.println(num + " x " + i + " = " + (num * i)); } sn.close(); } }

Step-by-Step Explanation

  1. For loop excels at known iterations: We know exactly how many times to loop (10 times for table 1-10). For loops are perfect when iteration count is predetermined.
  2. Three-part structure clarity: Initialization (int i=1), condition (i <=10), increment (i++) are all visible in one line. This makes the loop's behavior immediately obvious.
  3. Loop variable scope: Variable i exists only within the for loop. After loop ends, i is gone. This prevents variable pollution in your code.
  4. Classic counting pattern: Starting at 1 and counting to N is the most common for loop pattern. Master this and you can handle array iterations, mathematical series, and more.

Question 2: Pattern Printing (Stars)

Problem Statement

Print a right triangle pattern of stars. For N=5, print 1 star, then 2, then 3, up to 5.

Sample Input and Output

Sample Input 1:
4

Sample Output 1:
*
**
***
****
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class StarPattern { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter N: "); int n = sn.nextInt(); for (int i = 1; i <= n; i++) { for (int j = 1; j <= i; j++) { System.out.print("*"); } System.out.println(); } sn.close(); } }

Step-by-Step Explanation

  1. Nested loops for 2D patterns: Outer loop controls rows (i from 1 to n), inner loop controls columns/stars per row (j from 1 to i). This row-column thinking is fundamental.
  2. Inner loop depends on outer: Key insight - j goes from 1 to i (not n). First row has 1 star (i=1, j=1), second row has 2 stars (i=2, j=1,2). The dependency creates the triangle.
  3. print vs println distinction: Inner loop uses print() to put stars on same line. After inner loop completes, println() moves to next line. Understanding output control is crucial.
  4. Pattern printing develops logic: These problems build algorithmic thinking. Same nested structure applies to number patterns, pyramids, diamonds. Master the template, not just this specific pattern.

Question 3: Check Prime Number

Problem Statement

Check if a number is prime. A prime number is only divisible by 1 and itself.

Sample Input and Output

Sample Input 1:
7

Sample Output 1:
7 is Prime

Sample Input 2:
12

Sample Output 2:
12 is Not Prime
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class PrimeCheck { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter number: "); int num = sn.nextInt(); boolean isPrime = true; if (num <= 1) { isPrime = false; } else { for (int i = 2; i <= num/2; i++) { if (num % i == 0) { isPrime = false; break; } } } System.out.println(num + (isPrime ? " is Prime" : " is Not Prime")); sn.close(); } }

Step-by-Step Explanation

  1. Flag variable pattern: isPrime starts true (assume prime), then we look for evidence it's not. If we find a divisor, set flag false and break. This innocent-until-proven-guilty approach is common.
  2. Optimization - check to N/2: No need to check beyond num/2 because larger numbers can't divide evenly into num. For 12, checking 2,3,4,5,6 is enough. This halves work.
  3. Early exit with break: Once we find one divisor, we know it's not prime. break immediately exits the loop, saving unnecessary iterations. Efficiency matters in algorithms.
  4. Edge case handling: Numbers ≤ 1 aren't prime by definition. Always handle edge cases before main logic. 2 is the smallest prime and is correctly handled (loop never runs, stays true).

Question 4: Sum of Even Numbers in Range

Problem Statement

Find sum of all even numbers between 1 and N using for loop.

Sample Input and Output

Sample Input 1:
10

Sample Output 1:
Sum of even numbers: 30
(2+4+6+8+10=30)
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class SumEven { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter N: "); int n = sn.nextInt(); int sum = 0; for (int i = 2; i <= n; i += 2) { sum += i; } System.out.println("Sum of even numbers: " + sum); sn.close(); } }

Step-by-Step Explanation

  1. Smart loop increment saves checks: Starting at 2 and incrementing by 2 (i += 2) only visits even numbers. More efficient than checking if (i % 2 == 0) every iteration.
  2. Increment can be any value: Usually we use i++, but i += 2 (or i += 5, i *= 2, etc.) is valid. For loops are flexible - increment doesn't have to be +1.
  3. Accumulator initialized outside loop: sum starts at 0 before loop. If initialized inside, it would reset to 0 each iteration, losing accumulated value. Scope placement matters.
  4. Alternative approach exists: Could loop 1 to n with if (i%2==0) sum+=i. Both work, but i+=2 is cleaner and faster. Multiple solutions exist - choose the most elegant.

Question 5: Factorial with For Loop

Problem Statement

Calculate factorial of N using for loop. N! = N × (N-1) × ... × 2 × 1.

Sample Input and Output

Sample Input 1:
6

Sample Output 1:
Factorial of 6 = 720
▼ Show Solution

Solution (Java Program)

import java.util.Scanner; public class Factorial { public static void main(String[] args) { Scanner sn = new Scanner(System.in); System.out.print("Enter N: "); int n = sn.nextInt(); long factorial = 1; for (int i = 1; i <= n; i++) { factorial *= i; } System.out.println("Factorial of " + n + " = " + factorial); sn.close(); } }

Step-by-Step Explanation

  1. For loop simplicity for factorials: All three parts (init, condition, increment) are straightforward. For loops make factorial cleaner than while loops - everything's in one line.
  2. Use long for factorial results: Factorials grow explosively. 13! exceeds int's limit (2 billion). Using long extends range to 20!. For bigger, need BigInteger.
  3. Multiplication accumulator pattern: Like sum uses +=, factorial uses *=. Initialize to 1 (not 0!), then multiply by each number. This accumulate-by-operation pattern is universal.
  4. Could countdown instead: for (int i = n; i > 0; i--) works too. Count up or down - mathematically equivalent. Choose based on what feels more natural for the problem.
