Joins

JOIN is used to combine rows from two or more tables based on a related column between them.

INNER JOIN

🔹 Goal
Returns only the rows where there is a match in both tables.

🔹 Syntax
SELECT columns\nFROM table1\nINNER JOIN table2 ON table1.column = table2.column;
🔹 Example
SELECT e.emp_id, e.name, d.dept_name\nFROM employees e\nINNER JOIN departments d\nON e.dept_id = d.dept_id;
Result:
emp_idnamedept_name
1ArunIT
2RaviHR
3MeenaIT

❌ Sita excluded (no matching department in departments table).

LEFT JOIN (LEFT OUTER JOIN)

🔹 Goal
Returns all rows from the left table, and the matched rows from the right table. If no match, NULL is returned from the right side.

🔹 Example
SELECT e.name, d.dept_name\nFROM employees e\nLEFT JOIN departments d\nON e.dept_id = d.dept_id;
Result:
namedept_name
ArunIT
RaviHR
MeenaIT
SitaNULL

RIGHT JOIN (RIGHT OUTER JOIN)

🔹 Goal
Returns all rows from the right table, and the matched rows from the left table. If no match, NULL is returned from the left side.

🔹 Example
SELECT e.name, d.dept_name\nFROM employees e\nRIGHT JOIN departments d\nON e.dept_id = d.dept_id;
Result:
namedept_name
ArunIT
RaviHR
MeenaIT
NULLFinance

FULL JOIN (FULL OUTER JOIN)

🔹 Goal
Returns all rows when there is a match in either left or right table.
⚠️ Note: MySQL doesn't support FULL JOIN directly. We use UNION to achieve this.

🔹 Example
-- MySQL Alternative for FULL JOIN\nSELECT e.name, d.dept_name\nFROM employees e\nLEFT JOIN departments d ON e.dept_id = d.dept_id\nUNION\nSELECT e.name, d.dept_name\nFROM employees e\nRIGHT JOIN departments d ON e.dept_id = d.dept_id;
Result:
namedept_name
ArunIT
RaviHR
MeenaIT
SitaNULL

Practice Worksheet

Q1. Inner Join – Display students with their department names.
Query:
SELECT s.id, s.name, d.dept_name
FROM students s
INNER JOIN department d
ON s.dept_id = d.dept_id;

Output:
+----+--------+------------+
| id | name   | dept_name  |
+----+--------+------------+
| 1  | Arun   | IT         |
| 2  | Priya  | HR         |
| 3  | Kiran  | IT         |
+----+--------+------------+
Q2. Left Join – Display all students, including those without a department.
Query:
SELECT s.id, s.name, d.dept_name
FROM students s
LEFT JOIN department d
ON s.dept_id = d.dept_id;

Output:
+----+--------+------------+
| id | name   | dept_name  |
+----+--------+------------+
| 1  | Arun   | IT         |
| 2  | Priya  | HR         |
| 3  | Kiran  | IT         |
| 4  | Meena  | NULL       |
+----+--------+------------+
Q3. Right Join – Display all departments, including those without students.
Query:
SELECT s.id, s.name, d.dept_name
FROM students s
RIGHT JOIN department d
ON s.dept_id = d.dept_id;

Output:
+------+--------+------------+
| id   | name   | dept_name  |
+------+--------+------------+
| 1    | Arun   | IT         |
| 3    | Kiran  | IT         |
| 2    | Priya  | HR         |
| NULL | NULL   | Finance    |
+------+--------+------------+
Q4. Full Outer Join – Display all students and departments (MySQL uses UNION).
Query:
SELECT s.id, s.name, d.dept_name
FROM students s
LEFT JOIN department d ON s.dept_id = d.dept_id
UNION
SELECT s.id, s.name, d.dept_name
FROM students s
RIGHT JOIN department d ON s.dept_id = d.dept_id;

Output:
+------+--------+------------+
| id   | name   | dept_name  |
+------+--------+------------+
| 1    | Arun   | IT         |
| 2    | Priya  | HR         |
| 3    | Kiran  | IT         |
| 4    | Meena  | NULL       |
| NULL | NULL   | Finance    |
+------+--------+------------+
Q5. Self Join – Display students paired with students in the same department.
Query:
SELECT s1.name AS student1, s2.name AS student2, s1.dept_id
FROM students s1
JOIN students s2
ON s1.dept_id = s2.dept_id AND s1.id <> s2.id;

Output:
+----------+----------+---------+
| student1 | student2 | dept_id |
+----------+----------+---------+
| Arun     | Kiran    | 1       |
| Kiran    | Arun     | 1       |
+----------+----------+---------+
