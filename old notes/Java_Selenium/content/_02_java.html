02. Java Prerequisites for Selenium

📖 Introduction

Selenium supports multiple languages, but Java is the undisputed king of test automation. You do not need to be a full-stack Java developer to excel at Selenium. You only need to master "Core Java" concepts.

💡 What You'll Learn

  • Why Java is preferred for Selenium
  • Essential Java concepts: OOP, Collections, and Exceptions
  • How Java connects to WebDriver

🔍 Detailed Explanation

1. Object Oriented Programming (OOP)

Selenium itself is built on OOP.

  • Interface: WebDriver is an Interface.
  • Class: ChromeDriver is a Class implementing that interface.
  • Polymorphism: WebDriver driver = new ChromeDriver();

2. Collections Framework

We rarely use Arrays in Selenium because their size is fixed. We use Collections.

  • List (ArrayList): Storing a list of elements found by findElements().
  • Set (HashSet): Handling unique Window Handles.

💻 Code Examples

A Simple Java Class for Automation

public class BrowserFactory {
    
    // Encapsulation: Private Driver
    private WebDriver driver;

    // Method to return instance (Abstraction)
    public WebDriver initBrowser(String browserName) {
        if(browserName.equals("chrome")) {
            driver = new ChromeDriver();
        }
        return driver;
    }
}

⚠️ Common Pitfalls

NullPointerException: The most common Java error in Selenium. It happens when you try to use an object (like `driver`) that hasn't been initialized yet (is null).

📝 Interview Questions

Q1: Why do we use 'WebDriver driver = new ChromeDriver()' instead of 'ChromeDriver driver = ...'?

Answer: This is "Upcasting". If we use the interface WebDriver as the reference type, we can easily switch to new FirefoxDriver() later without changing the rest of our code. It allows for Cross-Browser Testing.
