19. Mouse Actions

📖 Introduction

While standard clicks are handled by the click() method, many modern web applications require complex mouse interactions like hovering, right-clicking, and double-clicking. Selenium's Actions class is designed to handle these low-level input device interactions.

💡 What You'll Learn

  • How to perform Mouse Hover (Move to Element)
  • How to Right-Click (Context Click)
  • How to Double-Click
  • Understanding the build() and perform() methods

🔍 Detailed Explanation

The Actions Class

To use mouse actions, you must instantiate the Actions class and pass the WebDriver instance to its constructor:

Actions act = new Actions(driver);

Common Methods

  • moveToElement(element): Hovers the mouse over an element.
  • contextClick(element): Performs a right-click on an element.
  • doubleClick(element): Performs a double-click on an element.
  • clickAndHold(element): Clicks without releasing.

Important: Most Actions methods won't execute until you call .perform() at the end of the chain.

💻 Code Examples

Mouse Hover Example

WebElement menu = driver.findElement(By.id("menu"));
Actions act = new Actions(driver);
act.moveToElement(menu).perform();

Right Click Example

WebElement box = driver.findElement(By.id("box"));
act.contextClick(box).perform();

⚠️ Common Pitfalls

Chaining Actions: If you are chaining multiple actions (e.g., move, then click), remember that perform() internally calls build().perform(). For single actions, perform() is sufficient.
