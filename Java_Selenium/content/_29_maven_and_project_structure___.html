29. Maven Project Structure

📖 Introduction

Professional Selenium projects are not just a collection of .java files. They need to manage dependencies (like Selenium jars, TestNG jars), compilation, and build lifecycles. Maven is the industry standard Build Automation Tool for Java.

💡 What You'll Learn

  • Why we need Maven (Dependency Management)
  • Understanding pom.xml
  • Correct folder structure (src/main vs src/test)
  • Running tests from Command Line

🔍 Detailed Explanation

The pom.xml

POM stands for Project Object Model. It is the heart of the project. You define valid dependencies here, and Maven automatically downloads them from the internet.

Standard Layout

MyProject
 ├── src
 │    ├── main
 │    │    └── java (Page Objects, Utilities)
 │    └── test
 │         ├── java (Test Cases)
 │         └── resources (Test Data, Config files)
 └── pom.xml

💻 Code Examples

pom.xml snippet

<dependencies>
    <!-- Selenium -->
    <dependency>
        <groupId>org.seleniumhq.selenium</groupId>
        <artifactId>selenium-java</artifactId>
        <version>4.10.0</version>
    </dependency>
    
    <!-- TestNG -->
    <dependency>
        <groupId>org.testng</groupId>
        <artifactId>testng</artifactId>
        <version>7.7.0</version>
        <scope>test</scope>
    </dependency>
</dependencies>

⚠️ Common Pitfalls

Scope Issues: If you mark a dependency as <scope>test</scope>, it will not be available in src/main/java. This is intentional. Test libraries should live in src/test.

🎯 Real-World Scenario

Practice Scenario

Goal: Update a Dependency.

  1. Check your current Selenium version in pom.xml.
  2. Go to mvnrepository.com.
  3. Find the latest version.
  4. Update the <version> tag.
  5. Reload Maven project to download new jars.

📝 Interview Questions

Q1: What is the command to run tests from Maven?

Answer: mvn test.
