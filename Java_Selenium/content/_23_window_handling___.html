23. Window Handling

📖 Introduction

Modern web applications often spawn new browser tabs or windows for things like Terms of Service, Login via Google, or Advertisements. When a new window opens, your WebDriver script does not automatically follow it. It stays stuck on the original page. You must manually "switch" the driver's focus to the new window to interact with it.

💡 What You'll Learn

  • What is a Window Handle?
  • How to Switch between multiple open windows/tabs
  • How to close a specific window and return to the parent
  • Iterating through window sets

🔍 Detailed Explanation

Window Handles

Every open browser tab has a unique identifier called a Window Handle (a String).

  • driver.getWindowHandle(): ID of current tab.
  • driver.getWindowHandles(): Set of IDs for ALL open tabs.

The Logic

  1. Store the ID of the Parent window.
  2. Click the link that opens a Child window.
  3. Get all Window Handles.
  4. Loop through them. If the handle is NOT the Parent handle, switch to it.
  5. Perform actions.
  6. Close Child window.
  7. Switch back to Parent window.

💻 Code Examples

// 1. Store Parent ID
String parentGUID = driver.getWindowHandle();

// 2. Click link (Opens new tab)
driver.findElement(By.id("new-tab-btn")).click();

// 3. Switch to Child
Set<String> allHandles = driver.getWindowHandles();

for(String handle : allHandles) {
    if(!handle.equals(parentGUID)) {
        driver.switchTo().window(handle);
        break; 
    }
}

// 4. Working in Child Tab
System.out.println("Child Title: " + driver.getTitle());
driver.close(); // Close Child only

// 5. Back to Parent
driver.switchTo().window(parentGUID);
System.out.println("Back to Parent");

⚠️ Common Pitfalls

NoSuchWindowException: This happens if you close a window (driver.close()) but forget to switch back to a valid window (driver.switchTo().window(...)). The driver is technically "floating" in a closed window context.

🎯 Real-World Scenario

Practice Scenario

Goal: Handle a Login Popup.

  1. Click "Login with Google".
  2. Switch to the new popup window.
  3. Type email credentials (just find element).
  4. Close the popup.
  5. Verify you are back on the homepage.

📝 Interview Questions

Q1: Can WebDriver handle Windows-based popups?

Answer: No. WebDriver handles Browser windows/tabs only. For OS-level popups (like Print Preview or File Upload), you need Robot class or AutoIT.
