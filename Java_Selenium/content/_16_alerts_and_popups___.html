16. Alerts & Popups

📖 Introduction

JavaScript Alerts are simple popups that block user interaction until handled. Unlike standard HTML modals (which you can inspect), JS Alerts are part of the browser's native interface. You cannot inspect them with DevTools, and you must use the Alert interface to control them.

💡 What You'll Learn

  • The 3 types of JS Alerts: Simple, Confirm, and Prompt
  • How to Switch to an Alert
  • Methods to Accept, Dismiss, and Type in alerts
  • Handling UnhandledAlertException

🔍 Detailed Explanation

The Alert Interface

Since the alert takes focus away from the DOM, you need to switch context:

Alert alert = driver.switchTo().alert();

Core Methods

  • alert.accept(): Simulates clicking "OK".
  • alert.dismiss(): Simulates clicking "Cancel".
  • alert.getText(): Retrieves the message text on the alert.
  • alert.sendKeys("text"): Enters text into a Prompt alert.

💻 Code Examples

Handling a Confirmation Box

// 1. Trigger the alert
driver.findElement(By.id("confirm-btn")).click();

// 2. Switch
Alert alert = driver.switchTo().alert();

// 3. Verify Text
System.out.println(alert.getText());

// 4. Click Cancel
alert.dismiss();

⚠️ Common Pitfalls

Synchronization: Sometimes alerts take a moment to appear. Use ExpectedConditions.alertIsPresent() in your Explicit Wait to avoid NoAlertPresentException.

🎯 Real-World Scenario

Practice Scenario

Goal: Handle a critical delete action.

  1. Click "Delete Account".
  2. Wait for the confirmation alert "Are you sure?".
  3. Read the text and click OK (Accept).
  4. Verify success message.

📝 Interview Questions

Q: Can you inspect an Alert?

A: No, it is a native browser window, not an HTML element.

Q: How to handle Authentication Popups (User/Pass)?

A: You can pass credentials in the URL: http://user:pass@example.com or use DevTools Protocol (CDP).
