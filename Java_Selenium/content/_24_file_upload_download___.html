24. File Upload & Download

📖 Introduction

Uploading resumes, profile pictures, or documents is a common user journey. Testing it automatically can be tricky because file dialogs ("Choose File" popup) are Native OS windows, not HTML. Selenium cannot click inside them. Fortunately, there is a simple workaround.

💡 What You'll Learn

  • The standard sendKeys trick for uploading files
  • Handling uploads when type="file" is missing (using Robot)
  • Verifying file downloads using Java File I/O

🔍 Detailed Explanation

1. The "sendKeys" Method (The Golden Path)

If the upload element in the DOM looks like <input type="file">, you are in luck. You do NOT click it. You simply send the Absolute Path of the file to it.

driver.findElement(By.id("upload")).sendKeys("C:\\users\\me\\photo.jpg");

This bypasses the system dialog entirely.

2. The Robot Class Method (The Hard Way)

If the element is not an input tag (e.g., a custom button triggering a hidden input), you might forced to click it. This opens the Windows File Explorer. You must use Java's Right class to simulate keyboard presses (Copy path to clipboard, Paste, Enter).

💻 Code Examples

We focus on the standard approach:

Standard Upload

// 1. Define file path (Use double backslashes for Windows)
String filePath = "C:\\test\\data.txt";

// 2. Identify the input element (hidden or visible)
WebElement uploadElement = driver.findElement(By.id("file-upload"));

// 3. Send path
uploadElement.sendKeys(filePath);

// 4. Submit
driver.findElement(By.id("submit")).click();

⚠️ Common Pitfalls

Invalid File Path: The most common error is a typo in the file path. Use Java's System.getProperty("user.dir") to make paths relative to your project folder for portability.

Clicking the Button: Beginners often click the upload button, the file dialog opens, and the script hangs because Selenium is waiting for the click logic to complete, but the OS dialog blocks the browser.

🎯 Real-World Scenario

Practice Scenario

Goal: Upload a Resume.

  1. Navigate to a Job Portal.
  2. Locate the upload input field.
  3. Upload a dummy text file.
  4. Assert the success message "File Uploaded Successfully".

📝 Interview Questions

Q1: How do you verify a file was downloaded successfully?

Answer: Selenium can't track downloads. Validation is done by checking the file system (Download folder) for the existence of the file with the expected name using Java's File class.
