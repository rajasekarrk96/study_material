14. Waits & Synchronization

📖 Introduction

Modern web applications are dynamic. Elements load at different speeds via AJAX, React, or Angular. If your script tries to find an element before it's ready, it fails with NoSuchElementException. Synchronization is the art of matching the speed of your WebDriver with the speed of the browser.

💡 What You'll Learn

  • Why simple Thread.sleep() is a bad practice
  • How to use Implicit Waits for global timeout
  • How to use Explicit Waits (WebDriverWait) for specific conditions
  • When to use Fluent Wait

🔍 Detailed Explanation

1. Implicit Wait

Tells WebDriver to wait for a certain amount of time when trying to find any element if it is not immediately available. It's set once globally.

driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));

Downside: It only waits for existence, not visibility or clickability.

2. Explicit Wait (Recommended)

Waits for a specific condition to apply to a specific element. It is more intelligent.

WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
wait.until(ExpectedConditions.visibilityOfElementLocated(By.id("loginBtn")));

3. Fluent Wait

Advanced version of explicit wait where you can define the polling interval (how often to check) and exceptions to ignore.

💻 Code Examples

Handling Slow Elements

// 1. Setup Explicit Wait
WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(15));

// 2. Clear conditions
// Wait until visible
WebElement banner = wait.until(ExpectedConditions.visibilityOfElementLocated(By.id("promo")));

// Wait until clickable
WebElement btn = wait.until(ExpectedConditions.elementToBeClickable(By.id("submit")));
btn.click();

// Wait until invisible (loading spinner)
wait.until(ExpectedConditions.invisibilityOfElementLocated(By.id("spinner")));

⚠️ Common Pitfalls

Don't Mix Waits: Avoid using Implicit and Explicit waits together. It can cause unpredictable wait times (e.g., 10s implicit + 10s explicit might result in 20s or more depending on browser implementation).

🎯 Real-World Scenario

Practice Scenario

Goal: Handle a slow-loading banking app.

  1. Click "Login" button.
  2. Wait for the loading spinner to disappear.
  3. Wait for the "Welcome User" message to be visible.
  4. Extract the text.

📝 Interview Questions

Q1: What is the default polling time of Selenium Explicit Wait?

Answer: 500 milliseconds (0.5 seconds).

Q2: Why is Thread.sleep() discouraged?

Answer: It's a static wait. It pauses execution for the full time even if the element loads in 1ms. Waits are dynamic; they proceed as soon as the condition is met.
