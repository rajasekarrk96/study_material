22. Find All Links on Page

📖 Introduction

So far, we've mostly used findElement() to grab a single item. But what if you want to count how many links are on the dashboard? Or verify that all 50 images on a page have a valid src attribute? This is where findElements() (plural) becomes your best friend.

💡 What You'll Learn

  • Difference between findElement and findElements
  • How to handle lists of WebElements
  • Iterating through elements to extract data
  • Handling "Empty Results" gracefully

🔍 Detailed Explanation

The Key Differences

Feature findElement() findElements()
Return Type WebElement List<WebElement>
If Found Returns first match Returns all matches
If Not Found Throws NoSuchElementException Returns Empty List (Size = 0)

💻 Code Examples

Scanning a page for links:

Find and Verify Links

// 1. Find all 'a' tags
List<WebElement> links = driver.findElements(By.tagName("a"));

// 2. Print Count
System.out.println("Total links: " + links.size());

// 3. Iterate and Print URLs
for(WebElement link : links) {
    String url = link.getAttribute("href");
    String text = link.getText();
    
    // Filter out empty links
    if(url != null && !url.isEmpty()) {
        System.out.println(text + " - " + url);
    }
}

⚠️ Common Pitfalls

StaleElementReferenceException: If you find a list of 100 elements, then the page refreshes (or DOM updates) while you are looping through them, the elements in your List become "stale" (dead). You cannot interact with them anymore. You must find the list again.

🎯 Real-World Scenario

Practice Scenario

Goal: Validate Footer Links.

  1. Locate the Footer section (e.g., a specific div).
  2. Find all links inside that footer only.
  3. Verify there are at least 5 links.
  4. Print the text of each link.

📝 Interview Questions

Q1: Does findElements() throw an exception if the element is not found?

Answer: No. It returns an empty list. This is often used to verify that an element does not exist (assert size is 0).
