27. Page Object Model

📖 Introduction

As your test suite grows, having code like driver.findElement(By.id("username")) scattered fast everywhere becomes a maintenance nightmare. If the developer changes the ID to "user_name", you have to fix it in 50 places.

Page Object Model (POM) is a design pattern that creates an "Object Repository" for your web elements. Each web page corresponds to a Java Class.

💡 What You'll Learn

  • The concept of separating Test Logic from Page Logic (Encapsulation)
  • Creating Page Classes
  • Using @FindBy annotations (PageFactory)
  • Benefits: Reusability, Maintainability, Readability

🔍 Detailed Explanation

Structure

  • LoginPage.java: Contains WebElements (username, password, loginBtn) and methods (enterUser, clickLogin).
  • LoginTest.java: Contains the test assertions (Open browser, call LoginPage methods, Verify success).

Page Factory

Selenium provides a PageFactory class to initialize elements defined with annotations.

💻 Code Examples

Page Class

public class LoginPage {
    WebDriver driver;

    // 1. Locators
    @FindBy(id="username")
    WebElement userField;

    @FindBy(id="password")
    WebElement passField;

    // 2. Constructor
    public LoginPage(WebDriver driver){
        this.driver = driver;
        PageFactory.initElements(driver, this);
    }

    // 3. Actions
    public void login(String u, String p){
        userField.sendKeys(u);
        passField.sendKeys(p);
    }
}

Test Class

@Test
public void validLoginTest(){
    LoginPage login = new LoginPage(driver);
    login.login("admin", "admin123");
}

⚠️ Common Pitfalls

InitElements: Beginners often forget PageFactory.initElements(driver, this) in the constructor. This leads to a NullPointerException when you try to use any element.

🎯 Real-World Scenario

Practice Scenario

Goal: Refactor legacy code.

  1. Create a SearchPage class.
  2. Move the search box locator and search button locator into it.
  3. Create a method doSearch(String keyword).
  4. Update your main test execution to use this new class.

📝 Interview Questions

Q1: Is POM a Selenium feature?

Answer: No, it is a Design Pattern. Selenium supports it via PageFactory, but you can implement POM without PageFactory too.
