04. WebDriver Commands

📖 Introduction

The WebDriver interface is the heart of Selenium. It controls the browser directly, just like a real user. Before we can interact with specific elements (like buttons or text boxes), we must first master how to control the browser window itself—opening pages, checking titles, and managing the session.

💡 What You'll Learn

  • How to launch a new browser window and navigate to URLs
  • Methods to fetch data from the current page (Title, URL, Page Source)
  • The critical difference between closing a window and quitting the session

🔍 Detailed Explanation

1. Browser Navigation

The most basic command is opening a webpage.

  • driver.get("url"): Loads a new web page in the current browser window. It waits for the page to load completely before proceeding.

2. Fetching Page Information

Once on a page, we often need to verify we are in the right place.

  • driver.getTitle(): Fetches the title of the current page. Essential for validation assertions.
  • driver.getCurrentUrl(): Returns the string representing the current URL. useful for redirect verification.
  • driver.getPageSource(): Returns the entire HTML source code of the page content.

3. Closing the Browser

Ending your test session properly is mandatory.

  • driver.close(): Closes only the current window that the driver is currently controlling.
  • driver.quit(): Closes all windows opened by the WebDriver session and safely terminates the process.

💻 Code Examples

Let's look at a script that navigates to a logical flow:

Basic Browser Operations

WebDriver driver = new ChromeDriver();

// 1. Open a website
driver.get("https://opensource-demo.orangehrmlive.com/");

// 2. Capture and print Title for validation
String title = driver.getTitle();
System.out.println("Page Title: " + title);

// 3. Validation Logic
if(title.contains("OrangeHRM")) {
    System.out.println("Test Passed: Title Matched");
} else {
    System.out.println("Test Failed: Title Mismatch");
}

// 4. Capture Current URL
System.out.println("Current URL: " + driver.getCurrentUrl());

// 5. Clean up
driver.quit();

⚠️ Common Pitfalls

Use `quit()` over `close()`: Beginners often use close() at the end of a test. If your test opened multiple tabs or popups, close() will leave them open, consuming memory (memory leaks). Always prefer quit() at the end of a test suite.

📝 Interview Questions

Q1: What is the difference between driver.get() and driver.navigate().to()?

Answer: Functionally they are similar. However, driver.get() waits for the page to load completely (DOM `readyState` to be complete), whereas navigate().to() does not wait and is used for browser history operations (back, forward, refresh).

Q2: What is the return type of driver.getTitle()?

Answer: It returns a String.
