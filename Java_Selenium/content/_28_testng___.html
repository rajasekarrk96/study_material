28. TestNG Testing Framework

📖 Introduction

Selenium is just a browser automation tool. It doesn't know what a "Test" is. It doesn't generate reports. It doesn't handle "Before" and "After" logic. That's why we pair Selenium with a Testing Framework like TestNG (Test Next Generation). It organizes your scripts into manageable test suites.

💡 What You'll Learn

  • Annotations: @Test, @BeforeMethod, @AfterMethod, @BeforeSuite
  • Grouping tests (Smoke, Regression)
  • Running tests in Parallel
  • Data Providers (Data Driven Testing)

🔍 Detailed Explanation

Key Annotations

  • @Test: Marks a method as a test case.
  • @BeforeMethod: Runs before every test method (e.g., Open Browser).
  • @AfterMethod: Runs after every test method (e.g., Close Browser).
  • @BeforeClass: Runs once before the first test method in the class.

xml Configuration

TestNG uses a testng.xml file to control execution (which tests to run, parallel settings, listeners).

💻 Code Examples

public class LoginTest {

    WebDriver driver;

    @BeforeMethod
    public void setup() {
        driver = new ChromeDriver();
        driver.get("https://app.com");
    }

    @Test(priority=1)
    public void verifyTitle() {
        Assert.assertEquals(driver.getTitle(), "My App");
    }

    @Test(priority=2)
    public void loginFlow() {
        // login code...
    }

    @AfterMethod
    public void tearDown() {
        driver.quit();
    }
}

⚠️ Common Pitfalls

Execution Order: By default, TestNG runs tests alphabetically, NOT in the order written in the file. Always use priority=X or dependsOnMethods="name" to control order.

🎯 Real-World Scenario

Practice Scenario

Goal: Create a Test Suite.

  1. Create two test classes: LoginTest and SearchTest.
  2. Create a testng.xml file.
  3. Configure it to run both classes.
  4. Run the XML suite.

📝 Interview Questions

Q1: Difference between @BeforeMethod and @BeforeClass?

Answer: @BeforeMethod runs before each @Test method (repeatedly). @BeforeClass runs once before any test in that class begins.
