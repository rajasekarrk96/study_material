11. Basic HTML Controls

📖 Introduction

Interacting with web elements is the daily bread and butter of any automation engineer. Whether it's typing a username, clicking a "Login" button, or selecting a checkbox, 90% of your test scripts will revolve around these basic HTML controls.

💡 What You'll Learn

  • How to type text into Input fields
  • How to clear existing text from fields
  • Clicking buttons and links
  • Handling Checkboxes and Radio buttons

🔍 Detailed Explanation

1. Input Fields (Text Box)

Used for entering data like names, emails, and passwords.

  • sendKeys("text"): Simulates typing keys into the element.
  • clear(): Erases any text currently in the field.

2. Buttons & Links

Used to trigger an action or navigation.

  • click(): The universal method to click anything that is clickable.

3. Checkboxes & Radio Buttons

Used for binary choices (Yes/No, Male/Female) or multiple selections.

  • click(): Toggles the state of a checkbox or selects a radio button.

💻 Code Examples

Here is how you control them:

Handling a Login Form

// 1. Enter Text
WebElement username = driver.findElement(By.id("user"));
username.clear(); // Good practice to clear first
username.sendKeys("admin");

// 2. Click Button
driver.findElement(By.id("loginBtn")).click();

// 3. Select Checkbox
WebElement rememberMe = driver.findElement(By.id("chkRemember"));
if(!rememberMe.isSelected()) {
    rememberMe.click();
}

⚠️ Common Pitfalls

Not Clearing First: If an input field has default text (like "Enter Name"), using sendKeys will append to it (e.g., "Enter NameJohn") instead of replacing it. Always clear() first.

ElementInterceptedException: Sometimes you try to click a button but another element (like a cookie banner) is covering it. You might need to close the banner first.

Practice Scenario

📝 Interview Questions

Q1: How do you submit a form without clicking the submit button?

Answer: You can use the submit() method on any element inside the form. It mimics pressing "Enter" on the keyboard.
