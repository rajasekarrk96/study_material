26. Common Exceptions

📖 Introduction

Exceptions are not your enemy; they are clues. They tell you exactly what went wrong. In Selenium, 90% of issues stem from a handful of common exceptions. Understanding them differentiates a beginner from an expert.

💡 What You'll Learn

  • The hierarchy of Selenium Exceptions
  • Root causes of NoSuchElementException, StaleElementReferenceException, etc.
  • How to debug and fix them

🔍 Detailed Explanation

1. NoSuchElementException

Cause: The locator (ID, XPath, etc.) you provided does not exist in the DOM at the exact moment the driver looked for it.

Fix: Check your locator logic. Check if the element is inside an iframe. Check if the page has fully loaded (Synchronization).

2. StaleElementReferenceException

Cause: The element you found was there, but the page refreshed (or DOM updated), and the reference is now dead ("stale").

Fix: You must findElement again to get a fresh reference.

3. ElementNotInteractableException

Cause: The element exists, but it might be hidden, overlapping with another element, or disabled.

Fix: Scroll the element into view. Wait for overlays to disappear.

💻 Code Examples

Handling exceptions gracefully:

Try-Catch Block

try {
    driver.findElement(By.id("logout")).click();
} catch (NoSuchElementException e) {
    System.out.println("Logout button not found. Must be already logged out.");
}

⚠️ Common Pitfalls

Catching Exception (Generic): Don't just catch Exception. Catch specific Selenium exceptions to handle them properly. Catching generic exceptions masks the real issue.

🎯 Real-World Scenario

Practice Scenario

Goal: Force a StaleElementReferenceException.

  1. Find the "Home" link.
  2. Refresh the page (driver.navigate().refresh()).
  3. Try to click the "Home" link variable you found in step 1.
  4. Observe the crash.
